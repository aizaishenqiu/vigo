# Vigo 应用部署文档

## 系统要求

- **操作系统**: CentOS 7.9 或更高版本
- **架构**: x86_64
- **内存**: 建议 4GB 以上（10 万并发建议 16GB+）
- **磁盘**: 建议 20GB 以上可用空间

---

## 一、快速部署

### 1.1 上传部署包

将以下文件上传到服务器 `/opt/vigo` 目录：

```
/opt/vigo/
├── vigo-linux-amd64    # 可执行文件（已嵌入模板）
├── config.yaml         # 配置文件
├── start.sh            # 启动脚本
├── stop.sh             # 停止脚本
└── restart.sh          # 重启脚本
```

### 1.2 设置文件权限

```bash
cd /opt/vigo
chmod +x vigo-linux-amd64
chmod +x start.sh
chmod +x stop.sh
chmod +x restart.sh
```

### 1.3 配置环境变量

编辑 `~/.bashrc` 或 `/etc/profile`，添加以下环境变量：

```bash
# 数据库配置（必需）
export DB_HOST=127.0.0.1
export DB_PORT=3306
export DB_USER=vigo
export DB_PASSWORD=your_password
export DB_NAME=vigo

# Redis 配置（必需）
export REDIS_HOST=127.0.0.1
export REDIS_PORT=6379
export REDIS_PASSWORD=

# JWT 配置（必需）
export JWT_SECRET=your_jwt_secret_key_at_least_32_characters

# 可选配置
export APP_ENV=prod
export APP_DEBUG=false
```

使配置生效：

```bash
source ~/.bashrc
```

### 1.4 启动应用

```bash
cd /opt/vigo
./start.sh
```

## 二、Systemd 服务部署（推荐）

### 2.1 创建服务用户

```bash
useradd -r -s /sbin/nologin www-data
chown -R www-data:www-data /opt/vigo
```

### 2.2 安装服务文件

```bash
cp /opt/vigo/vigo.service /etc/systemd/system/
```

编辑 `/etc/systemd/system/vigo.service`，修改路径为实际部署路径。

### 2.3 启动服务

```bash
# 重载 systemd 配置
systemctl daemon-reload

# 启动服务
systemctl start vigo

# 查看服务状态
systemctl status vigo

# 设置开机自启
systemctl enable vigo

# 查看日志
journalctl -u vigo -f
```

## 三、数据库配置

### 3.1 基础配置

```yaml
database:
  driver: mysql
  host: ${DB_HOST}
  port: ${DB_PORT:3306}
  name: ${DB_NAME}
  user: ${DB_USER}
  pass: ${DB_PASSWORD}
  charset: utf8mb4
```

### 3.2 连接池配置

| 配置项 | 说明 | 单机推荐 | 分布式推荐 |
|--------|------|----------|------------|
| max_open_conns | 最大数据库连接数 | 1000 | 300-500 |
| max_idle_conns | 最大空闲连接数 | 500 | 150-250 |
| conn_max_lifetime | 连接最大生命周期（秒） | 1800 | 1800 |
| conn_max_idletime | 空闲连接回收时间（秒） | 300 | 300 |

### 3.3 读写分离配置

```yaml
database:
  driver: mysql
  host: 192.168.1.10    # 主库（写库）
  port: 3306
  name: vigo
  user: root
  pass: password
  rw_split: true        # 开启读写分离

  # 多主库写入负载均衡
  writes:
    - host: 192.168.1.10
      port: 3306
      user: writer1
      pass: write-pass-1
    - host: 192.168.1.11
      port: 3306
      user: writer2
      pass: write-pass-2

  # 从库读取负载均衡
  reads:
    - host: 192.168.1.20
      port: 3306
      user: reader1
      pass: read-pass-1
    - host: 192.168.1.21
      port: 3306
      user: reader2
      pass: read-pass-2
```

### 3.4 多数据库配置

```yaml
databases:
  user_center:
    driver: mysql
    host: 192.168.2.10
    port: 3306
    name: user_center
    user: user_admin
    pass: user-pass
    max_open_conns: 500
    max_idle_conns: 100
    rw_split: true
    reads:
      - host: 192.168.2.20
        port: 3306
        user: reader
        pass: read-pass

  log_db:
    driver: postgres
    host: 192.168.4.10
    port: 5432
    name: log_db
    user: log_admin
    pass: log-pass
```

## 四、端口配置

| 服务 | 默认端口 | 说明 |
|------|----------|------|
| HTTP | 8080 | Web 服务端口 |
| gRPC | 9090 | 微服务端口（可选） |
| MySQL | 3306 | 数据库端口 |
| Redis | 6379 | 缓存端口 |
| RabbitMQ | 5672 | 消息队列端口（可选） |
| Nacos | 8848 | 配置中心端口（可选） |

修改端口：

```yaml
app:
  port: 8080    # HTTP 端口

grpc:
  enabled: true
  port: 9090    # gRPC 端口
```

## 五、系统优化（10 万并发）

### 5.1 内核参数优化

编辑 `/etc/sysctl.conf`：

```bash
# 文件描述符
fs.file-max = 6553500

# 网络优化
net.core.somaxconn = 65535
net.core.netdev_max_backlog = 65535
net.ipv4.tcp_max_syn_backlog = 65535
net.ipv4.tcp_syncookies = 1
net.ipv4.tcp_tw_reuse = 1
net.ipv4.tcp_fin_timeout = 30
net.ipv4.tcp_keepalive_time = 600
net.ipv4.ip_local_port_range = 1024 65535
```

使配置生效：

```bash
sysctl -p
```

### 5.2 文件描述符限制

编辑 `/etc/security/limits.conf`：

```bash
* soft nofile 65535
* hard nofile 65535
www-data soft nofile 65535
www-data hard nofile 65535
```

## 八、故障排查

### 8.1 端口被占用

```bash
# 查看端口占用
netstat -tlnp | grep 8080

# 杀掉占用进程
kill -9 <PID>

# 或配置自动杀掉
# config.yaml
app:
  auto_kill_port: true
```

### 8.2 数据库连接失败

检查：
- 数据库服务是否启动: `systemctl status mysql`
- 防火墙是否开放 3306 端口
- 环境变量是否正确配置: `echo $DB_HOST`
- 数据库用户权限: `mysql -u vigo -p`

### 8.3 Redis 连接失败

检查：
- Redis 服务是否启动: `redis-cli ping`
- 防火墙是否开放 6379 端口
- 密码是否正确

### 8.4 连接池耗尽

错误信息：`connection pool exhausted`

解决方案：
```yaml
database:
  max_open_conns: 2000
  max_idle_conns: 1000
```

## 九、目录结构

```
/opt/vigo/
├── vigo-linux-amd64    # 可执行文件（已嵌入模板）
├── config.yaml         # 配置文件
├── start.sh            # 启动脚本
├── stop.sh             # 停止脚本
├── restart.sh          # 重启脚本
├── vigo.service        # Systemd 服务文件
├── runtime/            # 运行时目录
│   ├── vigo.pid        # 进程 PID 文件
│   ├── app.log         # 应用日志
│   └── log/            # 日志目录
└── data/               # 数据目录（SQLite 等）
```

## 十一、安全建议

1. **使用非 root 用户运行**
2. **修改默认 JWT Secret**（至少 32 位）
3. **配置防火墙，只开放必要端口**
4. **定期备份数据库**
5. **启用 HTTPS**
6. **定期更新系统和依赖库**
7. **生产环境关闭 debug 模式**
