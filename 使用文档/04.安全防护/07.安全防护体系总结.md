# 安全防护体系总结

## 概述

Vigo 框架提供多层次的安全防护体系，确保应用安全：

### 三层防护架构

```
┌─────────────────────────────────────────┐
│  第一层：中间件自动拦截（兜底保护）      │
│  - 全局启用，自动拦截 SQL 注入、XSS 等攻击  │
│  - 配置文件控制：enable_security_middleware │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  第二层：Request 模块验证（推荐）         │
│  - 自动验证 + 数据净化                   │
│  - 请求方法验证                          │
│  - 简洁的助手函数                        │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  第三层：专业模块（文件上传等）          │
│  - Upload 模块：文件上传安全验证          │
│  - 独立的验证逻辑                        │
└─────────────────────────────────────────┘
```

## 各层防护详解

### 第一层：中间件自动拦截（必选）

**配置文件控制** (`config.yaml`)：

```yaml
security:
  # 是否启用全局安全中间件（强烈建议启用）
  enable_security_middleware: true
```

**防护内容**：
- ✅ SQL 注入检测
- ✅ XSS 攻击拦截
- ✅ 命令注入防护
- ✅ 路径穿越阻止
- ✅ 恶意 Header 检测

**特点**：
- **自动生效**：无需开发者干预
- **兜底保护**：即使忘记使用 request 模块也能拦截攻击
- **全局启用**：根据配置自动注册

**示例**：
```yaml
# config.yaml
security:
  enable_security_middleware: true  # 启用
```

### 第二层：Request 模块（推荐使用）

**安装使用**：

```go
import "vigo/framework/request"

// 方式 1：助手函数（推荐）
name := request.Input(c, "name", "")
age := request.InputInt(c, "age", 0)
email := request.Input(c, "email", "")

// 方式 2：对象调用
req := request.R(c)
name := req.Get("name", "")
age := req.GetInt("age", 0)

// 方式 3：请求方法验证
request.AllowMethods(c, request.MethodPOST, request.MethodPUT)
```

**防护内容**：
- ✅ SQL 注入检测（更严格）
- ✅ XSS 攻击过滤（自动净化）
- ✅ 命令注入检测
- ✅ 路径穿越防护
- ✅ 请求方法验证
- ✅ Header 安全验证
- ✅ 数据自动净化

**特点**：
- **简洁优雅**：一行代码获取安全数据
- **自动净化**：返回的数据已过滤恶意代码
- **类型安全**：支持字符串、整数、浮点数、布尔值、切片

**示例**：
```go
func (u *UserController) Create(c *mvc.Context) {
    // 获取数据（已自动验证和净化）
    username := request.Input(c, "username", "")
    email := request.Input(c, "email", "")
    age := request.InputInt(c, "age", 0)
    
    // 直接使用，无需担心安全问题
    user := &User{
        Username: username,
        Email:    email,
        Age:      age,
    }
    
    // 保存到数据库...
}
```

### 第三层：专业模块（按需使用）

#### 文件上传模块

```go
import "vigo/framework/upload"

// 快速上传
result, err := upload.Upload(
    c,
    "file",
    "./uploads",
    5*1024*1024,                    // 5MB
    []string{".jpg", ".png"},
)

// 或者使用对象
uploader := upload.NewUploader(
    "./uploads",
    5*1024*1024,
    []string{".jpg", ".png"},
)
result, err := uploader.ValidateAndUpload(c, "file")
```

**防护内容**：
- ✅ 文件类型白名单验证
- ✅ 文件大小限制
- ✅ MIME 类型检测
- ✅ 危险文件类型阻止
- ✅ 文件名安全检查
- ✅ 路径穿越防护
- ✅ 自动重命名（时间戳 + 哈希）
- ✅ SHA256 哈希计算

## 使用建议

### 推荐配置

```yaml
# config.yaml
security:
  # 必须启用全局中间件（兜底保护）
  enable_security_middleware: true
  
  # 可选：CSRF 保护
  enable_csrf_protection: false
  
  # 可选：速率限制
  enable_rate_limit: false
  rate_limit: 100
```

### 代码实践

```go
// ✅ 推荐：使用 request 模块 + 中间件兜底
func (u *UserController) Create(c *mvc.Context) {
    // Request 模块获取数据（自动验证和净化）
    username := request.Input(c, "username", "")
    email := request.Input(c, "email", "")
    
    // 中间件已经拦截了大部分攻击
    // request 模块再次验证并净化数据
    // 双重保护，安全可靠
}

// ✅ 推荐：文件上传使用独立模块
func (f *FileController) Upload(c *mvc.Context) {
    result, err := upload.Upload(c, "file", "./uploads", 5*1024*1024, []string{".jpg", ".png"})
    // 专业的文件上传验证
}

// ❌ 不推荐：直接使用原始数据
func (u *UserController) Create(c *mvc.Context) {
    // 危险：绕过安全验证
    username := c.Request.FormValue("username")
}
```

## 常见攻击防护示例

### 1. SQL 注入防护

**攻击示例**：
```
GET /user?id=1; DROP TABLE users--
POST /login
  username=admin' OR '1'='1
  password=anything
```

**防护效果**：
- 中间件：自动拦截，返回 400 错误
- Request：检测并阻止，数据被净化

### 2. XSS 攻击防护

**攻击示例**：
```html
POST /comment
  content=<script>alert('xss')</script>
  
GET /search?q=<img src=x onerror=alert('xss')>
```

**防护效果**：
- 中间件：检测并拦截恶意脚本
- Request：自动转义 HTML 特殊字符

### 3. 文件上传攻击防护

**攻击示例**：
```
上传文件：malicious.php
上传文件：../../../etc/passwd
上传文件：script.sh (MIME: application/x-sh)
```

**防护效果**：
- Upload 模块：文件类型白名单阻止.php
- Upload 模块：路径穿越检测阻止../
- Upload 模块：危险 MIME 类型阻止.sh

### 4. 命令注入防护

**攻击示例**：
```
GET /ping?ip=127.0.0.1; cat /etc/passwd
POST /exec
  command=ls | rm -rf /
```

**防护效果**：
- 中间件：检测命令分隔符并拦截
- Request：检测并阻止恶意命令

## 性能影响

| 防护层 | 性能影响 | 说明 |
|--------|---------|------|
| 中间件  | < 1ms   | 简单正则匹配，高性能 |
| Request | < 0.5ms | 数据净化，内存操作 |
| Upload  | 依赖文件大小 | 主要是 IO 操作 |

**总结**：安全防护对性能影响极小，可以忽略不计。

## 最佳实践清单

### ✅ 必须做的

1. **启用全局安全中间件**
   ```yaml
   security:
     enable_security_middleware: true
   ```

2. **使用 request 模块获取数据**
   ```go
   name := request.Input(c, "name", "")
   ```

3. **文件上传使用独立模块**
   ```go
   result, err := upload.Upload(c, "file", ...)
   ```

4. **设置合理的文件上传限制**
   ```go
   maxSize := 5 * 1024 * 1024 // 5MB
   allowedExts := []string{".jpg", ".png"}
   ```

### ⚠️ 避免做的

1. **不要直接使用原始数据**
   ```go
   // ❌ 危险
   username := c.Request.FormValue("username")
   
   // ✅ 安全
   username := request.Input(c, "username", "")
   ```

2. **不要禁用安全中间件**
   ```yaml
   # ❌ 危险
   enable_security_middleware: false
   ```

3. **不要信任客户端数据**
   ```go
   // ❌ 危险：直接使用客户端传来的文件名
   filename := c.Request.FormValue("filename")
   
   // ✅ 安全：使用上传模块生成的安全文件名
   result, _ := upload.Upload(c, "file", ...)
   safeFilename := result.Filename
   ```

## 故障排查

### 问题 1：正常请求被拦截

**原因**：输入中包含敏感关键词

**解决**：
- 检查日志：`[Security] 安全威胁检测...`
- 调整正则表达式（middleware.go）
- 使用白名单验证代替黑名单

### 问题 2：文件上传失败

**原因**：文件类型、大小或名称不符合要求

**解决**：
- 检查错误信息
- 放宽文件类型限制（谨慎）
- 增加文件大小限制

### 问题 3：中间件未生效

**原因**：配置未启用或路由未注册

**解决**：
- 检查 `config.yaml`：`enable_security_middleware: true`
- 检查 `route.go`：是否正确注册中间件
- 重启应用

## 总结

Vigo 框架的三层安全防护体系：

1. **中间件**：全局自动拦截，兜底保护
2. **Request 模块**：数据验证 + 净化，推荐使用
3. **专业模块**：文件上传等专项安全

**核心原则**：
- ✅ 所有输入都是危险的
- ✅ 多层防护，纵深防御
- ✅ 自动优先，手动补充
- ✅ 性能与安全并重

**记住**：安全不是一次性的工作，而是持续的过程。框架提供工具，但开发者需要正确使用！
