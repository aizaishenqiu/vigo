# 安全防护

框架内置了多层安全防护中间件，默认自动加载，保障应用安全。

---

## 1. 安全中间件一览

| 中间件 | 功能 | 默认状态 |
|:---|:---|:---|
| **Recovery** | 异常恢复，捕获 panic 防止进程崩溃 | 开启 |
| **Logger** | 请求日志，记录 IP、方法、路径、耗时 | 开启 |
| **SecurityHeaders** | 安全响应头 (X-Frame-Options 等) | 开启 |
| **XSS** | 跨站脚本过滤 | 开启 |
| **SQLInjection** | SQL 注入检测 | 开启 |
| **CSRF** | 跨站请求伪造防护 | 开启 |
| **RequestSizeLimit** | 请求体大小限制 (10MB) | 开启 |
| **RateLimit** | 速率限制 (DoS/DDoS 防护) | 可配置 |

## 2. 速率限制配置

```yaml
security:
  dos:
    enable: true
    limit: 100        # 窗口内最大请求数
    window: 60        # 窗口时长（秒）
    black_ip:         # IP 黑名单
      - 1.1.1.1
```

### 自动拉黑

当某 IP 请求频率超过限制的 **2 倍**，自动加入黑名单。

### 代理识别

自动识别 `X-Forwarded-For` 头部，防止代理绕过。

## 3. SQL 注入防护

自动检测请求参数中的注入特征：
- `UNION SELECT`, `OR 1=1`, `DROP TABLE` 等
- URL 参数和 POST Body 双重检测
- 匹配即返回 403

## 4. XSS 防护

自动对请求参数进行 HTML 转义，防止脚本注入。

## 5. 安全响应头

自动添加以下安全头部：
- `X-Frame-Options: DENY`
- `X-Content-Type-Options: nosniff`
- `X-XSS-Protection: 1; mode=block`

## 6. 内部路径白名单

框架自动跳过高频内部接口的中间件处理（如 `/health`、`/benchmark/stats`、`/rabbitmq/status`），避免性能损耗。

## 7. 自定义中间件

```go
func Auth() mvc.HandlerFunc {
    return func(c *mvc.Context) {
        token := c.Input("token")
        if token == "" {
            c.Error(401, "未授权")
            c.Abort()
            return
        }
        c.Next()
    }
}
```
