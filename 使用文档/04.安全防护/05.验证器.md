# ✅ 验证器 (Validator)

**Vigo** 内置了轻量级的验证器，支持链式调用，可以方便地对用户输入的数据进行合法性校验。

---

## 🚀 基础用法

在控制器中，你可以通过 `validate.New()` 创建一个验证器实例，并链式调用不同的规则。

```go
import "vigo/framework/validate"

func (c *UserController) Register(ctx *mvc.Context) {
    // 获取输入
    username := c.Input("username")
    email    := c.Input("email")
    age      := c.Input("age")
    mobile   := c.Input("mobile")

    // 链式验证
    v := validate.New().
        Required("username", username).
        Min("username", username, 4).
        Required("email", email).
        Email("email", email).
        Mobile("mobile", mobile)

    if !v.IsValid() {
        // 验证失败，返回错误信息
        c.Json(400, map[string]interface{}{
            "code":   400,
            "msg":    "验证失败",
            "errors": v.Errors, // 包含所有验证未通过的字段及原因
        })
        return
    }

    // 验证通过，继续业务逻辑...
}
```

---

## 🛠️ 内置验证规则

| 规则                   | 说明                       | 示例                                         |
| :--------------------- | :------------------------- | :------------------------------------------- |
| `Required(field, val)` | 必填项，不能为空字符串     | `.Required("name", name)`                    |
| `Email(field, val)`    | 必须是合法的邮箱格式       | `.Email("email", email)`                     |
| `Mobile(field, val)`   | 必须是合法的中国手机号格式 | `.Mobile("phone", phone)`                    |
| `Min(field, val, min)` | 字符串最小长度             | `.Min("pwd", pwd, 6)`                        |
| `Max(field, val, max)` | 字符串最大长度             | `.Max("desc", desc, 200)`                    |
| `In(field, val, list)` | 值必须在指定的范围内       | `.In("status", status, []interface{}{0, 1})` |

## 💡 进阶技巧

### 自定义错误信息

目前的验证器会自动生成英文错误信息。如果需要自定义或支持多语言，可以手动操作 `v.Errors` Map。

### 扩展验证规则

你可以在 `framework/validate/validate.go` 中参照现有规则轻松添加自己的验证逻辑，例如：

- `AlphaNum`: 仅允许字母和数字。
- `Date`: 日期格式校验。
- `Unique`: 数据库唯一性校验。

### 批量验证

验证器会一次性收集所有链式调用中的错误，并存储在 `v.Errors` 中。你可以直接将其返回给前端，方便用户一次性修正所有错误。
