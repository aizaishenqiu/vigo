# 缓存使用指南

Vigo 框架提供多种缓存实现，包括内存缓存、文件缓存和多级缓存。

## 一、快速开始

### 1.1 内存缓存

```go
package main

import (
    "time"
    "vigo/framework/cache"
)

func main() {
    // 创建内存缓存（最大1000条）
    c := cache.NewMemoryCache(1000)
    
    // 设置缓存
    c.Set("user:1", userData, 1*time.Hour)
    
    // 获取缓存
    value := c.Get("user:1")
    
    // 删除缓存
    c.Delete("user:1")
    
    // 检查是否存在
    if c.Has("user:1") {
        // ...
    }
    
    // 清空缓存
    c.Flush()
}
```

## 二、内存缓存

### 2.1 创建缓存

```go
// 创建内存缓存，指定最大条目数
c := cache.NewMemoryCache(1000) // 最多存储1000条

// 不限制数量（不推荐）
c := cache.NewMemoryCache(-1)
```

### 2.2 基础操作

```go
// 设置缓存（带过期时间）
c.Set("key", value, 10*time.Minute)

// 获取缓存
val := c.Get("key")
if val != nil {
    // 使用缓存值
}

// 删除单个缓存
c.Delete("key")

// 检查是否存在
exists := c.Has("key")

// 获取缓存数量
count := c.Count()

// 清空所有缓存
c.Flush()
```

### 2.3 缓存特性

- **自动过期**：支持设置过期时间，到期自动清理
- **LRU淘汰**：超过最大条目数时，自动淘汰最久未使用的缓存
- **线程安全**：并发安全，可在多协程中使用
- **零值处理**：过期或不存在时返回 nil

## 三、文件缓存

### 3.1 创建缓存

```go
// 创建文件缓存，指定存储目录
fc := cache.NewFileCache("./runtime/cache")

// 目录会自动创建
```

### 3.2 基础操作

```go
// 设置缓存
fc.Set("user:1", userData, 1*time.Hour)

// 获取缓存
val := fc.Get("user:1")

// 删除缓存
fc.Delete("user:1")

// 检查是否存在
exists := fc.Has("user:1")

// 清空所有缓存
fc.Flush()
```

### 3.3 文件缓存特性

- **持久化**：缓存数据保存在文件中，重启后仍然存在
- **自动清理**：过期文件会自动清理
- **适合场景**：配置缓存、大数据缓存、需要持久化的场景

---

## 四、多级缓存

### 4.1 创建多级缓存

```go
// 创建多级缓存（内存 + Redis）
mc := cache.NewMultiLevelCache(1000, 30*time.Minute)
```

### 4.2 基础操作

```go
// 设置缓存（同时写入内存和 Redis）
mc.Set("user:1", userData, 10*time.Minute)

// 获取缓存（先查内存，再查 Redis）
data := mc.Get("user:1")

// 删除缓存（同时删除内存和 Redis）
mc.Delete("user:1")

// 检查是否存在
exists := mc.Has("user:1")

// 清空缓存
c.Clear()
```

### 4.3 缓存穿透防护

```go
// 使用 GetWithProtection 防止缓存击穿
data, err := mc.GetWithProtection("user:1", func() (interface{}, error) {
    // 从数据库查询
    user, err := db.NewQuery(db.GetDB()).
        Table("users").
        Where("id", 1).
        Find()
    return user, err
})

if err != nil {
    log.Printf("查询失败: %v", err)
    return
}

// 使用数据
user := data.(*User)
```

**防护机制**：
- 使用互斥锁防止并发查询数据库（缓存击穿）
- 双重检查确保数据一致性
- 自动将查询结果写入多级缓存

## 六、最佳实践

### 6.1 缓存键命名规范

```go
// 推荐格式: 模块:类型:ID
c.Set("user:profile:1001", userProfile, time.Hour)
c.Set("article:content:500", articleContent, 30*time.Minute)
c.Set("config:system:database", dbConfig, 24*time.Hour)
```

### 6.2 缓存过期策略

```go
// 热点数据 - 短过期时间
c.Set("user:session:1001", session, 30*time.Minute)

// 配置数据 - 长过期时间
c.Set("config:app", appConfig, 24*time.Hour)

// 静态数据 - 不过期（或很长）
c.Set("dict:provinces", provinces, 7*24*time.Hour)
```

### 6.3 缓存更新策略

```go
// 方式1: 更新数据库后删除缓存（推荐）
func UpdateUser(userID int, data map[string]interface{}) error {
    // 1. 更新数据库
    _, err := db.NewQuery(db.GetDB()).
        Table("users").
        Where("id", userID).
        Update(data)
    if err != nil {
        return err
    }
    
    // 2. 删除缓存
    c.Delete(fmt.Sprintf("user:profile:%d", userID))
    
    return nil
}

// 方式2: 更新数据库后更新缓存
func UpdateUserAndCache(userID int, data map[string]interface{}) error {
    // 1. 更新数据库
    _, err := db.NewQuery(db.GetDB()).
        Table("users").
        Where("id", userID).
        Update(data)
    if err != nil {
        return err
    }
    
    // 2. 查询最新数据
    user, err := db.NewQuery(db.GetDB()).
        Table("users").
        Where("id", userID).
        Find()
    if err != nil {
        return err
    }
    
    // 3. 更新缓存
    c.Set(fmt.Sprintf("user:profile:%d", userID), user, time.Hour)
    
    return nil
}
```

### 6.4 防止缓存穿透

```go
// 对于可能不存在的数据，缓存空值
func GetUserWithNullCache(userID int) (*User, error) {
    key := fmt.Sprintf("user:profile:%d", userID)
    
    // 1. 查询缓存
    val := c.Get(key)
    if val != nil {
        if val == "__NULL__" {
            return nil, nil // 缓存的空值
        }
        return val.(*User), nil
    }
    
    // 2. 查询数据库
    user, err := db.NewQuery(db.GetDB()).
        Table("users").
        Where("id", userID).
        Find()
    if err != nil {
        if err == sql.ErrNoRows {
            // 3. 缓存空值（短期）
            c.Set(key, "__NULL__", 5*time.Minute)
            return nil, nil
        }
        return nil, err
    }
    
    // 4. 缓存数据
    c.Set(key, user, time.Hour)
    return user, nil
}
```

## 七、完整示例

### 7.1 用户缓存服务

```go
package service

import (
    "database/sql"
    "fmt"
    "time"
    "vigo/framework/cache"
    "vigo/framework/db"
)

type UserCacheService struct {
    cache *cache.MemoryCache
}

func NewUserCacheService() *UserCacheService {
    return &UserCacheService{
        cache: cache.NewMemoryCache(1000),
    }
}

// GetUser 获取用户（带缓存）
func (s *UserCacheService) GetUser(userID int) (*User, error) {
    key := fmt.Sprintf("user:%d", userID)
    
    // 1. 查询缓存
    val := s.cache.Get(key)
    if val != nil {
        return val.(*User), nil
    }
    
    // 2. 查询数据库
    var user User
    err := db.NewORM(db.GetDB()).
        Model(&User{}).
        Where("id", userID).
        First(&user)
    if err != nil {
        if err == sql.ErrNoRows {
            return nil, nil
        }
        return nil, err
    }
    
    // 3. 写入缓存
    s.cache.Set(key, &user, 30*time.Minute)
    
    return &user, nil
}

// UpdateUser 更新用户（更新缓存）
func (s *UserCacheService) UpdateUser(userID int, data map[string]interface{}) error {
    // 1. 更新数据库
    err := db.NewORM(db.GetDB()).
        Model(&User{}).
        Where("id", userID).
        Update(data)
    if err != nil {
        return err
    }
    
    // 2. 删除缓存
    s.cache.Delete(fmt.Sprintf("user:%d", userID))
    
    return nil
}

// DeleteUser 删除用户（删除缓存）
func (s *UserCacheService) DeleteUser(userID int) error {
    // 1. 删除数据库
    err := db.NewORM(db.GetDB()).
        Model(&User{}).
        Where("id", userID).
        Delete()
    if err != nil {
        return err
    }
    
    // 2. 删除缓存
    s.cache.Delete(fmt.Sprintf("user:%d", userID))
    
    return nil
}
```

## 八、注意事项

1. **缓存键要唯一**：避免不同数据使用相同键
2. **设置合理的过期时间**：避免缓存过期太早或太晚
3. **及时更新缓存**：数据变更时及时删除或更新缓存
4. **注意内存使用**：内存缓存要设置合理的上限
5. **处理缓存穿透**：对于不存在的数据也要缓存（短期）
6. **使用多级缓存**：高并发场景使用多级缓存提高性能
