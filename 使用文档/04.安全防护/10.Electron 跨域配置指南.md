# Electron/Tauri 桌面应用跨域配置指南

## 问题背景

在使用 Electron 或 Tauri 开发桌面应用时，由于浏览器的同源策略，前端访问后端 API 会遇到 CORS（跨域资源共享）问题。

## 解决方案

### 框架自动处理（推荐）

**框架已自动识别并放行桌面应用**，无需任何额外配置！

#### 自动识别的协议

框架会自动识别以下桌面应用协议并允许访问：

| 框架 | 协议 | 示例 |
|------|------|------|
| Electron | `file://` | `file:///C:/app/dist/index.html` |
| Electron | `app://` | `app://./index.html` |
| Electron | `http://localhost:*` | `http://localhost:3000` |
| Electron | `https://localhost:*` | `https://localhost:3000` |
| Tauri | `tauri://` | `tauri://localhost` |
| Tauri | `http://tauri.localhost` | `http://tauri.localhost` |

## Electron 配置

### 1. 主进程配置

```javascript
// main.js - Electron 主进程
const { app, BrowserWindow } = require('electron')
const path = require('path')

function createWindow() {
  const mainWindow = new BrowserWindow({
    width: 1200,
    height: 800,
    webPreferences: {
      // ✅ 保持 webSecurity 开启（安全）
      webSecurity: true,
      
      // ❌ 不建议开启 nodeIntegration（安全风险）
      nodeIntegration: false,
      
      // ✅ 使用 preload 脚本
      preload: path.join(__dirname, 'preload.js'),
      
      // ✅ 允许跨域请求
      webviewTag: true
    }
  })

  // 加载本地文件（生产环境）
  mainWindow.loadFile('dist/index.html')
  
  // 或加载本地开发服务器（开发环境）
  // mainWindow.loadURL('http://localhost:3000')
  
  // 打开开发者工具（开发环境）
  // mainWindow.webContents.openDevTools()
}

app.whenReady().then(createWindow)
```

### 2. 预加载脚本

```javascript
// preload.js - 预加载脚本
const { contextBridge, ipcRenderer } = require('electron')

// 暴露安全的 API 给渲染进程
contextBridge.exposeInMainWorld('electronAPI', {
  // 系统信息
  getPlatform: () => process.platform,
  
  // IPC 通信
  send: (channel, data) => ipcRenderer.send(channel, data),
  invoke: (channel, data) => ipcRenderer.invoke(channel, data),
  
  // 版本信息
  getVersion: () => ipcRenderer.invoke('app/getVersion')
})
```

### 3. 渲染进程配置

```javascript
// renderer.js - 渲染进程
// ✅ 正确的 API 调用方式
const API_BASE_URL = process.env.NODE_ENV === 'development'
  ? 'http://localhost:8080'  // 开发环境
  : 'https://api.example.com' // 生产环境

async function fetchUsers() {
  try {
    const response = await fetch(`${API_BASE_URL}/api/users`, {
      method: 'GET',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${token}`
      },
      credentials: 'include' // 包含 cookies
    })
    
    const data = await response.json()
    console.log('Users:', data)
  } catch (error) {
    console.error('Fetch error:', error)
  }
}

// 使用 Electron API
if (window.electronAPI) {
  const platform = await window.electronAPI.getPlatform()
  console.log('Running on:', platform)
}
```

## 常见问题

### Q1: 开发环境遇到 CORS 错误？

**A:** 确保 `app.mode = dev` 或 `app.debug = true`

### Q2: 生产环境 Electron 无法访问 API？

**A:** 检查：
1. API 地址是否正确
2. 是否使用 HTTPS（生产环境建议）
3. 防火墙是否阻止请求
4. 查看开发者工具的网络请求

### Q3: 如何查看 CORS 错误详情？

**A:** 打开 Electron 开发者工具（F12），查看 Console 和 Network 面板

### Q4: 需要关闭 webSecurity 吗？

**A:** **不需要！** 框架已自动处理 CORS，关闭 webSecurity 会带来安全风险

### Q5: 可以使用 nodeIntegration 吗？

**A:** **不建议！** 应该使用 preload 脚本暴露安全的 API

## 安全建议

### ✅ 推荐做法

1. **保持 webSecurity 开启**
   ```javascript
   webPreferences: {
     webSecurity: true  // ✅ 保持开启
   }
   ```

2. **使用 preload 脚本**
   ```javascript
   webPreferences: {
     preload: path.join(__dirname, 'preload.js')
   }
   ```

3. **最小化暴露 API**
   ```javascript
   contextBridge.exposeInMainWorld('api', {
     // 只暴露必要的功能
     getData: () => ipcRenderer.invoke('get-data')
   })
   ```

4. **使用 HTTPS（生产环境）**
   ```javascript
   const API_URL = 'https://api.example.com'  // ✅ 生产环境
   ```

### ❌ 不推荐做法

1. **关闭 webSecurity**
   ```javascript
   webPreferences: {
     webSecurity: false  // ❌ 危险！
   }
   ```

2. **开启 nodeIntegration**
   ```javascript
   webPreferences: {
     nodeIntegration: true  // ❌ 危险！
   }
   ```

3. **使用 enableRemoteModule**
   ```javascript
   webPreferences: {
     enableRemoteModule: true  // ❌ 已废弃，危险！
   }
   ```

## 完整示例

### Electron + Vue 3 + 框架

#### 1. 后端配置

```yaml
# config.yaml
app:
  mode: dev
  port: 8080

security:
  enable_cors_domain_check: true
```

#### 2. Electron 主进程

```javascript
// main.js
const { app, BrowserWindow } = require('electron')
const path = require('path')

function createWindow() {
  const mainWindow = new BrowserWindow({
    width: 1200,
    height: 800,
    webPreferences: {
      webSecurity: true,
      nodeIntegration: false,
      preload: path.join(__dirname, 'preload.js')
    }
  })

  if (process.env.NODE_ENV === 'development') {
    mainWindow.loadURL('http://localhost:3000')
    mainWindow.webContents.openDevTools()
  } else {
    mainWindow.loadFile(path.join(__dirname, '../dist/index.html'))
  }
}

app.whenReady().then(createWindow)
```

#### 3. Vue 组件

```vue
<template>
  <div>
    <h1>用户列表</h1>
    <ul>
      <li v-for="user in users" :key="user.id">{{ user.name }}</li>
    </ul>
  </div>
</template>

<script>
export default {
  data() {
    return {
      users: []
    }
  },
  async mounted() {
    const API_URL = process.env.NODE_ENV === 'development'
      ? 'http://localhost:8080'
      : 'https://api.example.com'
    
    const response = await fetch(`${API_URL}/api/users`)
    const data = await response.json()
    this.users = data.data
  }
}
</script>
```
