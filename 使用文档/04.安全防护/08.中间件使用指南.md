# 中间件使用指南

## 概述

Vigo 框架提供强大的中间件系统，支持：

- ✅ **全局中间件自动注册** - 配置文件控制
- ✅ **中间件管理器** - 统一管理中间件
- ✅ **中间件分组** - 灵活组合中间件
- ✅ **路由分组中间件** - 为不同路由组应用不同中间件

## 中间件架构

```
┌─────────────────────────────────────────┐
│  全局中间件（自动注册）                  │
│  - 安全中间件（配置控制）                │
│  - 日志中间件（始终启用）                │
│  - 恢复中间件（始终启用）                │
│  - 限流中间件（配置控制）                │
│  - CSRF 中间件（配置控制）                │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  路由分组中间件（按需使用）              │
│  - 认证中间件                            │
│  - 权限中间件                            │
│  - 自定义中间件                          │
└─────────────────────────────────────────┘
```

## 配置文件

在 `config.yaml` 中配置中间件：

```yaml
security:
  # 是否启用全局安全中间件（强烈建议启用）
  enable_security_middleware: true
  
  # 是否启用 CSRF 保护
  enable_csrf_protection: false
  
  # 是否启用速率限制
  enable_rate_limit: false
  rate_limit: 100  # 每秒请求数
```


### 2. RecoveryMiddleware - 错误恢复中间件

捕获 panic，防止服务崩溃：

```go
// 始终自动注册
```

### 3. RateLimitMiddleware - 速率限制中间件

限制每秒请求数：

```go
// 自动注册（根据配置）
// config.yaml: security.enable_rate_limit: true
//              security.rate_limit: 100
```

### 4. CSRF - CSRF 攻击防护

防止跨站请求伪造：

```go
// 自动注册（根据配置）
// config.yaml: security.enable_csrf_protection: true
```

验证 JWT Token：


## 中间件管理器

### 注册全局中间件

```go
import "vigo/framework/middleware"

// 方式 1：使用 Use 函数
middleware.Use(middleware.RecoveryMiddleware())
middleware.Use(middleware.LogMiddleware())

// 方式 2：使用管理器
manager := middleware.NewMiddlewareManager()
manager.Use(middleware.SecurityMiddleware())
manager.Use(middleware.RecoveryMiddleware())

// 获取所有中间件
middlewares := manager.GetMiddlewares()
```

### 中间件分组

```go
import "vigo/framework/middleware"

// 创建中间件分组
authGroup := middleware.AuthGroup()
logGroup := middleware.LogGroup()
securityGroup := middleware.SecurityGroup()

// 组合多个分组
fullStack := middleware.FullStack(true, 100) // 需要认证，限流 100req/s

// 在路由分组中使用
api := r.Group("/api")
api.Use(fullStack.GetMiddlewares()...)
```

### 自定义中间件分组

```go
// 创建自定义分组
customGroup := middleware.NewMiddlewareGroup(
    middleware.SecurityMiddleware(),
    middleware.RecoveryMiddleware(),
    myCustomMiddleware(),
)

// 添加更多中间件
customGroup.Use(middleware.JWTAuth())

// 使用
admin := r.Group("/admin")
admin.Use(customGroup.GetMiddlewares()...)
```

## 路由分组中间件

### 基本用法

```go
import "vigo/framework/mvc"

r := mvc.NewRouter()

// 创建路由分组并添加中间件
api := r.Group("/api", middleware.JWTAuth())
api.GET("/users", getUsers)
api.POST("/users", createUser)

// 嵌套分组
admin := api.Group("/admin", middleware.AdminAuth())
admin.GET("/dashboard", adminDashboard)

// 或使用 Use 方法
api2 := r.Group("/api")
api2.Use(middleware.JWTAuth())
api2.Use(middleware.RateLimitMiddleware(100))
api2.GET("/users", getUsers)
```

### 完整示例

```go
package route

import (
    "vigo/framework/middleware"
    "vigo/framework/mvc"
)

func Init(r *mvc.Router) {
    // 全局中间件（已自动注册）
    // middleware.Use(...)
    
    // 公开路由
    r.GET("/", indexHandler)
    r.POST("/login", loginHandler)
    
    // 需要认证的路由
    api := r.Group("/api", middleware.JWTAuth())
    api.GET("/users", getUsers)
    api.POST("/users", createUser)
    api.PUT("/users/:id", updateUser)
    api.DELETE("/users/:id", deleteUser)
    
    // 需要认证 + 限流的路由
    limited := r.Group("/api/limited")
    limited.Use(middleware.JWTAuth())
    limited.Use(middleware.RateLimitMiddleware(10)) // 10 req/s
    limited.GET("/sensitive", sensitiveHandler)
    
    // 管理员路由
    admin := r.Group("/admin")
    admin.Use(middleware.JWTAuth())
    admin.Use(middleware.AdminAuth())
    admin.Use(middleware.RateLimitMiddleware(50))
    admin.GET("/dashboard", adminDashboard)
    admin.POST("/users", adminCreateUser)
}
```

## 自定义中间件

### 创建中间件

```go
package middleware

import "vigo/framework/mvc"

// 自定义中间件
func MyCustomMiddleware() mvc.HandlerFunc {
    return func(c *mvc.Context) {
        // 前置处理
        println("请求前：", c.Request.URL.Path)
        
        // 继续处理
        c.Next()
        
        // 后置处理
        println("请求后：", c.Response.Status())
    }
}
```

### 带参数的中间件

```go
func RoleAuth(requiredRoles ...string) mvc.HandlerFunc {
    return func(c *mvc.Context) {
        // 获取用户角色
        userRole := c.Get("role")
        
        // 检查权限
        hasRole := false
        for _, role := range requiredRoles {
            if userRole == role {
                hasRole = true
                break
            }
        }
        
        if !hasRole {
            c.JSON(403, map[string]interface{}{
                "code":    403,
                "message": "权限不足",
            })
            c.Abort()
            return
        }
        
        c.Next()
    }
}

// 使用
admin := r.Group("/admin")
admin.Use(RoleAuth("admin", "super_admin"))
```

## 中间件执行顺序

```go
// 注册顺序决定执行顺序
r.Use(middleware.SecurityMiddleware())  // 1. 安全检查
r.Use(middleware.RecoveryMiddleware())  // 2. 错误恢复
r.Use(middleware.LogMiddleware())       // 3. 日志记录
r.Use(middleware.JWTAuth())             // 4. 身份认证

// 请求处理流程：
// Security -> Recovery -> Log -> Auth -> Handler
//                  <- Log <- Auth <- Handler
```

## 最佳实践

### 1. 全局中间件在配置文件中控制

```yaml
security:
  enable_security_middleware: true  # 必须启用
  enable_csrf_protection: false     # 按需启用
  enable_rate_limit: false          # 按需启用
  rate_limit: 100
```

### 2. 路由分组使用中间件

```go

r.GET("/", handler)

// 认证路由 - 使用 JWT
api := r.Group("/api", middleware.JWTAuth())

// 管理路由 - 多层中间件
admin := r.Group("/admin")
admin.Use(middleware.JWTAuth())
admin.Use(middleware.AdminAuth())
admin.Use(middleware.RateLimitMiddleware(50))
```

### 3. 自定义中间件分组

```go
// 定义中间件分组
var (
    AuthStack = middleware.AuthGroup()
    LogStack = middleware.LogGroup()
    AdminStack = middleware.NewMiddlewareGroup(
        middleware.JWTAuth(),
        middleware.AdminAuth(),
    )
)

// 使用
api := r.Group("/api", AuthStack.GetMiddlewares()...)
```

### 4. 中间件执行顺序

```go
// 重要：中间件按注册顺序执行
// 1. 安全 -> 2.恢复 -> 3.日志 -> 4.认证 -> 5.业务
r.Use(middleware.SecurityMiddleware())
r.Use(middleware.RecoveryMiddleware())
r.Use(middleware.LogMiddleware())
r.Use(middleware.JWTAuth())
```

## 常见问题

### Q1: 如何禁用某个中间件？

A: 在配置文件中设置：

```yaml
security:
  enable_security_middleware: false  # 禁用安全中间件
  enable_csrf_protection: false      # 禁用 CSRF
  enable_rate_limit: false           # 禁用限流
```

### Q2: 如何为特定路由禁用中间件？

A: 使用单独的路由分组：

```go
// 普通 API（需要认证）
api := r.Group("/api", middleware.JWTAuth())

// 公开 API（无需认证）
public := r.Group("/public")
public.Use(middleware.SecurityMiddleware()) // 只使用安全中间件
```

### Q3: 如何调试中间件？

A: 在中间件中添加日志：

```go
func DebugMiddleware() mvc.HandlerFunc {
    return func(c *mvc.Context) {
        fmt.Printf("[Debug] Before: %s %s\n", c.Request.Method, c.Request.URL.Path)
        c.Next()
        fmt.Printf("[Debug] After: %d\n", c.Response.Status())
    }
}
```

### Q4: 中间件性能影响？

A: 每个中间件约增加 0.1-0.5ms 延迟，建议：

- 全局中间件保持精简
- 特殊中间件按需使用
- 避免在中间件中进行耗时操作

## 总结

Vigo 框架的中间件系统提供：

- ✅ **自动注册** - 配置文件控制全局中间件

- ✅ **路由分组** - 为不同路由应用不同中间件
- ✅ **易于扩展** - 自定义中间件简单

**记住**：中间件按注册顺序执行，合理安排顺序很重要！
