# 自定义中间件开发指南

**更新日期**: 2026-03-06

---

## 📖 概述

Vigo 框架支持灵活扩展自定义中间件，开发者可以根据业务需求定义和注册中间件。本文档详细介绍自定义中间件的完整开发流程。

---

## 🎯 自定义中间件的三种注册方式

### 方式一：在路由初始化时注册（推荐）

**位置**: `route/route.go`

这是最常用和推荐的方式，适合全局中间件或特定路由分组的中间件。

#### 示例 1：注册全局自定义中间件

```go
package route

import (
    "vigo/framework/mvc"
    "vigo/framework/middleware"
    "log"
    "time"
)

// 自定义中间件：请求耗时统计
func RequestTimeMiddleware() mvc.HandlerFunc {
    return func(c *mvc.Context) {
        start := time.Now()
        c.Next() // 执行下一个中间件或控制器
        duration := time.Since(start)
        log.Printf("[RequestTime] 请求 %s 耗时：%v", c.Request.URL.Path, duration)
    }
}

func Init(r *mvc.Router) {
    // 注册全局中间件
    r.Use(RequestTimeMiddleware())
    r.Use(middleware.Logger())
    r.Use(middleware.Recovery())

    // 注册其他路由...
}
```

#### 示例 2：注册到特定路由分组

```go
func Init(r *mvc.Router) {
    // 自定义认证中间件
    authMiddleware := func() mvc.HandlerFunc {
        return func(c *mvc.Context) {
            token := c.Input("token")
            if token == "" {
                c.Error(401, "Unauthorized")
                c.Abort()
                return
            }
            // 验证 token 逻辑
            c.Next()
        }
    }

    // 仅对 /admin 路径生效
    adminGroup := r.Group("/admin", authMiddleware())
    adminGroup.GET("/dashboard", admin.Dashboard)
    adminGroup.POST("/user/create", admin.CreateUser)
}
```

### 方式二：使用中间件管理器注册

**位置**: `main.go` 或路由初始化前的任何地方

适合在应用启动时统一注册中间件。

#### 示例：在 main.go 中注册

```go
package main

import (
    "vigo/framework/app"
    "vigo/framework/middleware"
    "vigo/framework/mvc"
    "log"
)

// 自定义中间件：请求 ID 追踪
func RequestIDMiddleware() mvc.HandlerFunc {
    return func(c *mvc.Context) {
        // 生成或获取请求 ID
        requestID := c.Request.Header.Get("X-Request-ID")
        if requestID == "" {
            requestID = generateRequestID()
        }

        // 设置到响应头
        c.Writer.Header().Set("X-Request-ID", requestID)

        // 存储到 context 中供后续使用
        c.Set("requestID", requestID)

        c.Next()
    }
}

func generateRequestID() string {
    // 生成唯一请求 ID
    return fmt.Sprintf("req-%d", time.Now().UnixNano())
}

func main() {
    // 创建应用
    application := app.New(".")

    // 注册全局中间件（在路由初始化之前）
    middleware.Use(
        RequestIDMiddleware(),        // 自定义中间件
        middleware.Recovery(),        // 异常恢复
        middleware.Logger(),          // 日志记录
    )

    // 初始化路由
    application.Initialize()

    // 启动服务
    application.Run(":8080")
}
```

### 方式三：在框架中间件文件中扩展

**位置**: `framework/middleware/` 目录下创建新文件

适合复用性高的中间件，便于统一管理。

#### 步骤 1：创建中间件文件

在 `framework/middleware/` 目录下创建 `custom.go`：

```go
package middleware

import (
    "vigo/framework/mvc"
    "log"
    "strings"
)

// ==================== 自定义中间件示例 ====================

// TenantMiddleware 多租户中间件
// 从域名或 Header 中识别租户
func TenantMiddleware() mvc.HandlerFunc {
    return func(c *mvc.Context) {
        // 从域名提取租户 ID
        host := c.Request.Host
        parts := strings.Split(host, ".")

        var tenantID string
        if len(parts) > 2 {
            // subdomain.example.com -> subdomain
            tenantID = parts[0]
        } else {
            // 从 Header 获取
            tenantID = c.Request.Header.Get("X-Tenant-ID")
        }

        if tenantID == "" {
            c.Error(400, "Tenant ID required")
            c.Abort()
            return
        }

        // 存储租户信息到 context
        c.Set("tenantID", tenantID)

        log.Printf("[Tenant] 识别租户：%s", tenantID)

        c.Next()
    }
}

// RateLimitByUserMiddleware 基于用户的限流中间件
func RateLimitByUserMiddleware(limit int) mvc.HandlerFunc {
    // 实现基于用户 ID 的限流逻辑
    return func(c *mvc.Context) {
        userID := c.Get("userID")
        if userID == nil {
            c.Next()
            return
        }

        // 实现限流逻辑
        // ...

        c.Next()
    }
}

// CacheMiddleware 缓存中间件
func CacheMiddleware(ttl int) mvc.HandlerFunc {
    return func(c *mvc.Context) {
        // 实现缓存逻辑
        // 1. 检查缓存是否存在
        // 2. 存在则直接返回
        // 3. 不存在则继续执行，并缓存结果

        c.Next()
    }
}

// CompressionMiddleware 响应压缩中间件
func CompressionMiddleware() mvc.HandlerFunc {
    return func(c *mvc.Context) {
        // 实现响应压缩逻辑
        c.Next()
    }
}
```

#### 步骤 2：在路由中使用

```go
package route

import (
    "vigo/framework/middleware"
    "vigo/framework/mvc"
)

func Init(r *mvc.Router) {
    // 使用自定义中间件
    r.Use(middleware.TenantMiddleware())
    r.Use(middleware.RateLimitByUserMiddleware(100))

    // 对特定路由使用缓存中间件
    cacheGroup := r.Group("/api", middleware.CacheMiddleware(300))
    cacheGroup.GET("/data", controller.GetData)
    cacheGroup.GET("/list", controller.GetList)
}
```

## 🔧 实际业务场景示例

### 示例 1：用户认证中间件

```go
package middleware

import (
    "vigo/framework/mvc"
    "strings"
)

// UserAuthMiddleware 用户认证中间件
func UserAuthMiddleware() mvc.HandlerFunc {
    return func(c *mvc.Context) {
        // 1. 从 Header 获取 Token
        authHeader := c.Request.Header.Get("Authorization")
        if authHeader == "" {
            c.Error(401, "缺少认证信息")
            c.Abort()
            return
        }

        // 2. 解析 Token
        parts := strings.SplitN(authHeader, " ", 2)
        if len(parts) != 2 || parts[0] != "Bearer" {
            c.Error(401, "Token 格式错误")
            c.Abort()
            return
        }

        token := parts[1]

        // 3. 验证 Token（这里调用你的验证逻辑）
        userID, err := validateToken(token)
        if err != nil {
            c.Error(401, "Token 无效")
            c.Abort()
            return
        }

        // 4. 存储用户信息到 Context
        c.Set("userID", userID)
        c.Set("authenticated", true)

        c.Next()
    }
}
```

### 示例 2：请求日志中间件（增强版）

```go
package middleware

import (
    "vigo/framework/mvc"
    "log"
    "time"
)

// DetailedLogMiddleware 详细请求日志中间件
func DetailedLogMiddleware() mvc.HandlerFunc {
    return func(c *mvc.Context) {
        start := time.Now()

        // 记录请求信息
        log.Printf("[Request] %s %s from %s",
            c.Request.Method,
            c.Request.URL.Path,

        c.Next()

        // 记录响应信息

            c.Writer.Status(),
            duration,
        )
    }
}
```

### 示例 3：跨域中间件（自定义配置）

```go
package middleware

import (
    "vigo/framework/mvc"
    "net/http"
)

// CustomCORSMiddleware 自定义 CORS 中间件
func CustomCORSMiddleware(allowedOrigins []string) mvc.HandlerFunc {
    return func(c *mvc.Context) {
        origin := c.Request.Header.Get("Origin")

        // 检查是否允许该源
        allowed := false
        for _, o := range allowedOrigins {
            if o == "*" || o == origin {
                allowed = true
                break
            }
        }

        if !allowed {
            c.Next()
            return
        }

        // 设置 CORS 头
        c.Writer.Header().Set("Access-Control-Allow-Origin", origin)
        c.Writer.Header().Set("Access-Control-Allow-Methods", "GET, POST, PUT, DELETE, OPTIONS")
        c.Writer.Header().Set("Access-Control-Allow-Headers", "Origin, Content-Type, Authorization")

        // 处理预检请求
        if c.Request.Method == "OPTIONS" {
            c.Writer.WriteHeader(http.StatusNoContent)
            c.Abort()
            return
        }

        c.Next()
    }
}
```

## 📋 中间件开发最佳实践

### 1. 保持中间件单一职责

```go
// ✅ 好的做法：每个中间件只做一件事
func AuthMiddleware() mvc.HandlerFunc { /* 只做认证 */ }

func LogMiddleware() mvc.HandlerFunc { /* 只做日志 */ }

func RateLimitMiddleware() mvc.HandlerFunc { /* 只做限流 */ }

// ❌ 不好的做法：一个中间件做太多事
func EverythingMiddleware() mvc.HandlerFunc {
    // 认证 + 日志 + 限流 + 缓存...
}
```

### 2. 及时调用 Next() 或 Abort()

```go
// ✅ 正确的做法
func MyMiddleware() mvc.HandlerFunc {
    return func(c *mvc.Context) {
        if !isValid {
            c.Error(403, "Forbidden")
            c.Abort() // 终止处理
            return
        }
        c.Next() // 继续处理
    }
}

// ❌ 错误的做法：忘记调用 Next() 或 Abort()
func MyMiddleware() mvc.HandlerFunc {
    return func(c *mvc.Context) {
        // 没有调用 Next() 或 Abort()
        // 导致请求挂起
    }
}
```

### 3. 使用 Context 传递数据

```go
// ✅ 使用 Context 存储中间件产生的数据
func AuthMiddleware() mvc.HandlerFunc {
    return func(c *mvc.Context) {
        userID := validateToken()
        c.Set("userID", userID) // 存储数据
        c.Next()
    }
}

// 在控制器中获取
func (c *Controller) GetUser(ctx *mvc.Context) {
    userID := ctx.Get("userID").(int)
}
```

### 4. 避免在中间件中执行耗时操作

```go
// ✅ 好的做法：轻量级操作
func LogMiddleware() mvc.HandlerFunc {
    return func(c *mvc.Context) {
        go logAsync() // 异步记录日志
        c.Next()
    }
}

// ❌ 不好的做法：同步耗时操作
func LogMiddleware() mvc.HandlerFunc {
    return func(c *mvc.Context) {
        logToFile() // 同步写入文件，阻塞请求
        c.Next()
    }
}
```

### 5. 中间件命名规范

```go
// ✅ 清晰的命名
func UserAuthMiddleware() mvc.HandlerFunc { }
func RequestLogMiddleware() mvc.HandlerFunc { }
func RateLimitMiddleware() mvc.HandlerFunc { }

// ❌ 模糊的命名
func MyMiddleware() mvc.HandlerFunc { }
func TestMiddleware() mvc.HandlerFunc { }
```

## 🎯 总结

自定义中间件的三种注册方式：

| 方式                   | 位置                             | 适用场景                   | 推荐度     |
| ---------------------- | -------------------------------- | -------------------------- | ---------- |
| **路由初始化时注册**   | `route/route.go`                 | 全局中间件、路由分组中间件 | ⭐⭐⭐⭐⭐ |
| **中间件管理器注册**   | `main.go`                        | 应用启动时统一注册         | ⭐⭐⭐⭐   |
| **框架中间件文件扩展** | `framework/middleware/custom.go` | 可复用的通用中间件         | ⭐⭐⭐⭐⭐ |

开发自定义中间件的关键点：

1. **明确职责**：每个中间件只做一件事
2. **正确调用**：记得调用 `Next()` 或 `Abort()`
3. **数据传递**：使用 Context 传递中间件产生的数据
4. **性能优先**：避免同步耗时操作
5. **规范命名**：使用清晰的命名约定
