# 🛡️ 中间件 (Middleware)

中间件用于在请求处理前后执行逻辑，如日志记录、权限验证、异常捕获等。

## 定义中间件

中间件本质上是一个返回 `mvc.HandlerFunc` 的函数。

```go
package middleware

import (
    "vigo/framework/mvc"
    "log"
)

func Auth() mvc.HandlerFunc {
    return func(c *mvc.Context) {
        token := c.Input("token")
        if token == "" {
            c.Error(401, "Unauthorized")
            c.Abort() // 终止后续处理
            return
        }

        // 继续执行下一个中间件或控制器
        c.Next()
    }
}
```

## 注册中间件

在 `route/route.go` 中注册。

### 全局中间件

对所有路由生效。

```go
func Init(r *mvc.Router) {
    r.Use(middleware.Logger())
    r.Use(middleware.Recovery())
    // ...
}
```

### 路由中间件 (需自行扩展 Router 支持)

_当前版本暂未内置路由组级别的中间件支持，建议在 Controller 的 Init 方法中手动调用或在 Handler 内部调用。_

## 内置安全中间件

框架内置了多项安全中间件，保障系统的基础防御能力：

- **Logger**: 记录请求耗时、IP、方法、路径等日志。
- **Recovery**: 自动捕获运行时 Panic 异常，防止进程崩溃并返回 500 响应。
- **XSS**: 跨站脚本过滤，自动对 URL 参数和表单数据进行 HTML 转义。
- **RateLimit**: 速率限制 (DoS/DDoS 防护)，基于 IP 限制请求频率，支持黑名单和自动拉黑。

## 安全配置 (config.yaml)

你可以通过配置文件灵活调整安全中间件的行为：

```yaml
security:
  dos:
    enable: true # 是否开启 DoS 防护，默认开启
    limit: 100 # 限制窗口内的最大请求数
    window: 60 # 限制窗口时长（秒）
    black_ip: # 初始黑名单 IP 列表
      - 1.1.1.1
```

### 增强特性

1. **自动拉黑**：当某个 IP 的请求频率超过设定限制的 2 倍时，系统将自动将其加入黑名单并持久化拦截（内存级）。
2. **代理识别**：自动识别 `X-Forwarded-For` 头部，防止攻击者通过代理绕过限流。
3. **黑名单管理**：支持在 `config.yaml` 中预设黑名单，也支持运行时动态拉黑。

## 自动加载

框架在 `App.Run` 时会自动加载以上安全中间件，开发者无需手动干预即可获得基础防护。
