# 🛡️ 中间件 (Middleware)

中间件用于在请求处理前后执行逻辑，如日志记录、权限验证、异常捕获等。

## 中间件系统架构

Vigo 框架提供完整的中间件管理系统，包括：

- **全局中间件管理器**：统一管理全局中间件
- **路由组中间件**：针对特定路由分组的中间件
- **中间件分组**：预定义的中间件组合

## 定义中间件

中间件本质上是一个返回 `mvc.HandlerFunc` 的函数。

```go
package middleware

import (
    "vigo/framework/mvc"
    "log"
    "time"
)

// 示例：请求耗时统计中间件
func RequestTime() mvc.HandlerFunc {
    return func(c *mvc.Context) {
        start := time.Now()
        c.Next()
        duration := time.Since(start)
        log.Printf("请求耗时：%v", duration)
    }
}
```

## 注册中间件

### 方式一：全局中间件管理器

对所有路由生效，通过中间件管理器统一注册。

```go
package main

import (
    "vigo/framework/middleware"
    "vigo/framework/app"
)

func main() {
    // 创建应用
    application := app.New(".")
    
    // 注册全局中间件（在路由初始化之前）
    middleware.Use(
        middleware.Logger(),      // 日志记录
        middleware.Recovery(),    // 异常恢复
        middleware.CORS(),        // 跨域支持
    )
    
    // 初始化路由
    application.Initialize()
    
    // 启动服务
    application.Run(":8080")
}
```

### 方式二：路由注册时直接添加

在 `route/route.go` 中直接注册全局中间件。

```go
func Init(r *mvc.Router) {
    // 全局安全中间件
    if config.App.Security.EnableSecurityMiddleware {
        r.Use(middleware.SecurityMiddleware())
    }
    
    // 全局日志中间件
    r.Use(middleware.Logger())
    
    // 全局异常恢复中间件
    r.Use(middleware.Recovery())
    
    // 注册其他路由...
}
```

### 方式三：路由组中间件

针对特定路由分组的中间件。

```go
func Init(r *mvc.Router) {
    // 带中间件的路由分组
    // 限制每个 IP 每秒最多 10 次请求
    loginGroup := r.Group("/api", middleware.RateLimitMiddleware(10))
    loginGroup.POST("/login", auth.Login)
    loginGroup.GET("/login", auth.Login)
    
    // 带多个中间件的路由分组
    adminGroup := r.Group("/admin", 
        middleware.JWTAuth(),              // JWT 认证
        middleware.SecurityMiddleware(),   // 安全验证
    )
    adminGroup.GET("/dashboard", admin.Dashboard)
    adminGroup.POST("/user/create", admin.CreateUser)
}
```

### 方式四：排除特定中间件

对某些路由排除已注册的中间件。

```go
func Init(r *mvc.Router) {
    // 创建路由分组
    group := r.Group("/api")
    
    // 排除某些中间件（如果中间件支持名称匹配）
    group.WithoutMiddleware("AuthMiddleware")
    
    // 注册公开接口（不需要认证）
    group.GET("/public/data", controller.PublicData)
}
```

## 内置中间件列表

框架内置了丰富的中间件，满足各种场景需求：

### 安全防护类

| 中间件 | 说明 | 使用示例 |
|--------|------|----------|
| **SecurityMiddleware** | 统一安全验证（SQL 注入、XSS、命令注入等） | `middleware.SecurityMiddleware()` |
| **SecurityHeaders** | 设置安全响应头 | `middleware.SecurityHeaders()` |
| **CSRF** | CSRF Token 验证 | `middleware.CSRF()` |
| **XSS** | XSS 跨站脚本过滤 | `middleware.XSS()` |

### 性能保护类

| 中间件 | 说明 | 使用示例 |
|--------|------|----------|
| **RateLimit** | 速率限制（分片锁 + 自动清理 + 黑名单） | `middleware.RateLimit(100, time.Second)` |
| **RequestSizeLimit** | 请求体大小限制 | `middleware.RequestSizeLimit(10 * 1024 * 1024)` |

### 认证授权类

| 中间件 | 说明 | 使用示例 |
|--------|------|----------|
| **JWTAuth** | JWT Token 认证 | `middleware.JWTAuth()` |
| **HMACSign** | API 签名验证（HMAC-SHA256） | `middleware.HMACSign(secrets, 300)` |

### 功能增强类

| 中间件 | 说明 | 使用示例 |
|--------|------|----------|
| **Logger** | 日志记录（敏感信息过滤） | `middleware.Logger()` |
| **Recovery** | 异常恢复（防止 Panic 崩溃） | `middleware.Recovery()` |
| **CORS** | 跨域资源共享 | `middleware.CORS(cfg)` |

## 中间件组合

框架提供了便捷的中间件组合函数：

```go
// 安全防护组合
middleware.SecurityGroup()

// 日志组合
middleware.LogGroup()

// 认证组合
middleware.AuthGroup()

// 限流组合
middleware.RateLimitGroup(100)

// 完整中间件栈
middleware.FullStack(true, 100) // 需要认证，限流 100 请求/秒
```

## 中间件配置

### 安全中间件配置

在 `config.yaml` 中配置安全相关参数：

```yaml
security:
  enable_security_middleware: true  # 是否启用安全中间件
  
  # 速率限制配置
  rate_limit:
    enable: true
    requests_per_second: 100
    window_seconds: 60
  
  # 黑名单 IP
  black_ips:
    - 1.1.1.1
    - 2.2.2.2
```

### CORS 配置

```go
// 自定义 CORS 配置
corsCfg := middleware.CORSConfig{
    AllowOrigins:     []string{"https://example.com", "https://api.example.com"},
    AllowMethods:     []string{"GET", "POST", "PUT", "DELETE"},
    AllowHeaders:     []string{"Origin", "Content-Type", "Authorization"},
    ExposeHeaders:    []string{"Content-Length"},
    AllowCredentials: true,
    MaxAge:           86400,
}

// 使用自定义配置
r.Use(middleware.CORS(corsCfg))
```

## 完整示例

### 示例 1：API 服务中间件配置

```go
package main

import (
    "vigo/framework/app"
    "vigo/framework/middleware"
)

func main() {
    application := app.New(".")
    
    // 注册全局中间件
    middleware.Use(
        middleware.Recovery(),              // 异常恢复（必须）
        middleware.Logger(),                // 日志记录
        middleware.SecurityMiddleware(),    // 安全验证
        middleware.CORS(),                  // 跨域支持
    )
    
    application.Initialize()
    application.Run(":8080")
}
```

### 示例 2：管理后台中间件配置

```go
func Init(r *mvc.Router) {
    // 公开接口
    r.POST("/api/login", auth.Login)
    
    // 需要认证的管理接口
    adminGroup := r.Group("/admin",
        middleware.JWTAuth(),              // JWT 认证
        middleware.SecurityMiddleware(),   // 安全验证
        middleware.RateLimitMiddleware(50), // 限流 50 请求/秒
    )
    
    adminGroup.GET("/dashboard", admin.Dashboard)
    adminGroup.GET("/users", admin.ListUsers)
    adminGroup.POST("/user/create", admin.CreateUser)
    
    // 超级管理员接口（额外权限验证）
    superGroup := adminGroup.Group("/super", middleware.RoleCheck("super_admin"))
    superGroup.DELETE("/user/:id", admin.DeleteUser)
}
```

### 示例 3：公开 API 中间件配置

```go
func Init(r *mvc.Router) {
    // 公开 API（仅基础保护）
    publicGroup := r.Group("/public",
        middleware.Recovery(),           // 异常恢复
        middleware.RateLimitMiddleware(200), // 限流 200 请求/秒
    )
    
    publicGroup.GET("/data", controller.GetData)
    publicGroup.GET("/list", controller.GetList)
    
    // 需要签名的 API
    apiGroup := r.Group("/api",
        middleware.Recovery(),
        middleware.HMACSign(apiSecrets, 300), // API 签名验证
    )
    
    apiGroup.POST("/order/create", order.Create)
    apiGroup.GET("/order/:id", order.Detail)
}
```

## 中间件执行顺序

中间件按照注册顺序依次执行：

```
请求 → 中间件 1 → 中间件 2 → 中间件 3 → 控制器 → 响应
         ↓           ↓           ↓
      前置逻辑   前置逻辑   前置逻辑
         
         ↑           ↑           ↑
      后置逻辑   后置逻辑   后置逻辑
```

示例：

```go
// 注册顺序
r.Use(
    middleware.Logger(),      // 1. 记录请求开始
    middleware.Recovery(),    // 2. 异常捕获
    middleware.SecurityMiddleware(), // 3. 安全验证
)

// 执行流程：
// 1. Logger 记录请求开始时间
// 2. Recovery 设置 defer 捕获异常
// 3. SecurityMiddleware 验证请求安全性
// 4. 控制器处理请求
// 5. 返回响应
// 6. Logger 记录请求耗时
```

## 注意事项

1. **中间件顺序很重要**：先注册的中间件先执行， Recovery 应该放在最前面
2. **调用 Next()**：中间件中必须调用 `c.Next()` 才能继续执行
3. **终止处理**：调用 `c.Abort()` 可以终止后续处理
4. **性能考虑**：避免在中间件中执行耗时操作
5. **错误处理**：中间件中的错误要及时处理或返回

## 最佳实践

1. **全局中间件**：日志、异常恢复、安全验证等基础保护
2. **路由组中间件**：认证、限流等特定场景的中间件
3. **中间件组合**：使用预定义的中间件组合简化配置
4. **配置化**：通过配置文件控制中间件的启用和参数
5. **性能监控**：监控中间件的性能开销
