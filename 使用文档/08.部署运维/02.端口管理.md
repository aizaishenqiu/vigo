# 端口管理

框架支持启动时自动检测端口占用，并可选择性地释放被占用的端口。

---

## 1. 配置

```yaml
app:
  port: 8080
  auto_kill_port: true  # 启动时自动释放被占用的端口
```

## 2. 工作原理

应用启动时 (`App.Initialize()`)：

1. **读取配置**: 收集所有需要使用的端口（HTTP、gRPC）
2. **检测占用**: 逐个检测端口是否被其他进程占用
3. **自动释放**: 如果 `auto_kill_port: true`，自动终止占用端口的进程
4. **友好提示**: 在控制台输出端口状态和操作结果

```
[端口检测] HTTP 端口 8080 可用 ✓
[端口检测] gRPC 端口 9090 被 PID 12345 占用，正在释放...
[端口检测] gRPC 端口 9090 已释放 ✓
```

## 3. 检测范围

自动检测的端口包括：
- HTTP 服务端口 (`app.port`)
- gRPC 服务端口 (`grpc.port`，仅在 `grpc.enabled: true` 时）

## 4. 跨平台支持

- **Windows**: 使用 `netstat` + `taskkill`
- **Linux/macOS**: 使用 `lsof` + `kill`

## 5. 手动使用

```go
import "vigo/framework/port"

// 检查单个端口
port.CheckAndKill(8080)

// 检查所有配置的端口
port.CheckAll(map[string]int{
    "HTTP": 8080,
    "gRPC": 9090,
})
```
