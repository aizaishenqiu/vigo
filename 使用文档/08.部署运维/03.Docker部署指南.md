# Docker 部署指南

本文档介绍如何使用 Docker 部署 Vigo 应用程序及其相关服务（Nacos、RabbitMQ、Redis、MySQL）。

---

## 1. 准备工作

### 1.1 Docker 和 Docker Compose

确保已安装 Docker 和 Docker Compose：

- Docker 版本 ≥ 19.03
- Docker Compose 版本 ≥ 1.25

### 1.2 配置文件准备

在Docker环境中，Vigo框架使用YAML配置文件而不是.env文件。您需要准备适用于Docker环境的配置文件。

创建 `config.docker.yaml` 配置文件：

```yaml
# 应用配置
app:
  name: vigo
  version: 1.0.1
  port: 8080
  debug: false
  mode: prod
  console: true
  auto_kill_port: true

# 数据库配置
database:
  driver: mysql
  host: mysql
  port: 3306
  name: jw_platform_db
  user: root
  pass: shine20
  charset: utf8mb4
  max_open_conns: 200
  max_idle_conns: 100
  conn_max_lifetime: 3600
  rw_split: false
  reads: []

# Nacos 配置
nacos:
  host: nacos
  port: 8848
  namespace: ""
  data_id: vigo-config
  group: DEFAULT_GROUP
  install_path: "" # 在容器中通常不使用自动启动
  # 服务发现配置
  discovery:
    enabled: true
    service_name: vigo-app
    auto_register: true
    heartbeat: 5
  # 管理界面配置
  admin:
    enabled: true
    username: ""
    password: ""
    port: 8848

# RabbitMQ 配置
rabbitmq:
  enabled: true
  host: rabbitmq
  port: 5672
  user: guest
  pass: guest
  vhost: /
  conn_timeout: 5
  heartbeat: 10
  reconnect_delay: 3
  max_retries: 3
  # 管理界面配置
  admin:
    enabled: true
    username: ""
    password: ""
    port: 15672

# Redis 配置
redis:
  host: redis
  port: 6379
  password: "123456"
  db: 0
  pool_size: 100

# gRPC 微服务配置
grpc:
  enabled: false
  port: 9090
  service_name: vigo
  enable_recovery: true
  enable_logger: true
  max_recv_msg_size: 4
  max_send_msg_size: 4

# 安全配置
security:
  jwt:
    secret: prod-jwt-secret-key-for-production-only-32chars
    expire: 7200
    issuer: vigo
  session:
    lifetime: 120
    secure: true
    httponly: true
    samesite: strict
  password:
    min_length: 8
    require_uppercase: true
    require_lowercase: true
    require_number: true
    require_special: false
  cors:
    allow_origins: []
    allow_methods: ["GET", "POST", "PUT", "DELETE", "OPTIONS"]
    allow_headers: ["Content-Type", "Authorization", "X-Requested-With"]
    max_age: 86400
  dos:
    enable: true
    limit: 200
    window: 60
    black_ip: []
```

## 2. Dockerfile

创建 `Dockerfile` 构建应用镜像：

```dockerfile
FROM golang:1.21-alpine AS builder

# 安装构建依赖
RUN apk add --no-cache git ca-certificates

WORKDIR /app

# 复制 go mod 文件
COPY go.mod go.sum ./
RUN go mod download

# 复制源代码
COPY . .

# 构建应用
RUN CGO_ENABLED=0 GOOS=linux go build -a -installsuffix cgo -o main .

# 最终镜像
FROM alpine:latest

# 安装 ca-certificates
RUN apk --no-cache add ca-certificates tzdata

WORKDIR /root/

# 从 builder 阶段复制二进制文件
COPY --from=builder /app/main .

# 创建配置目录
RUN mkdir -p config

# 复制配置文件
COPY config.yaml ./config/
COPY config.docker.yaml ./config/

# 暴露端口
EXPOSE 8080

# 启动命令
CMD ["./main"]
```

## 3. Docker Compose 配置

创建 `docker-compose.yml` 配置多服务：

```yaml
version: "3.8"

services:
  # 应用服务
  app:
    build:
      context: .
      dockerfile: Dockerfile
    container_name: vigo-app
    ports:
      - "8080:8080"
    depends_on:
      - mysql
      - redis
      - rabbitmq
      - nacos
    environment:
      - APP_ENV=docker
    volumes:
      - ./config.docker.yaml:/root/config/config.docker.yaml
    networks:
      - vigo-network
    restart: unless-stopped
    healthcheck:
      test:
        [
          "CMD",
          "wget",
          "--quiet",
          "--tries=1",
          "--spider",
          "http://localhost:8080/health",
        ]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 60s

  # MySQL 数据库
  mysql:
    image: mysql:8.0
    container_name: vigo-mysql
    ports:
      - "3306:3306"
    environment:
      MYSQL_ROOT_PASSWORD: shine20
      MYSQL_DATABASE: jw_platform_db
    volumes:
      - mysql-data:/var/lib/mysql
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql
    networks:
      - vigo-network
    restart: unless-stopped
    healthcheck:
      test:
        [
          "CMD",
          "mysqladmin",
          "ping",
          "-h",
          "localhost",
          "-u",
          "root",
          "-p$$MYSQL_ROOT_PASSWORD",
        ]
      timeout: 20s
      retries: 10

  # Redis 缓存
  redis:
    image: redis:7-alpine
    container_name: vigo-redis
    ports:
      - "6379:6379"
    command: redis-server --requirepass 123456 --appendonly yes
    volumes:
      - redis-data:/data
    networks:
      - vigo-network
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 3s
      retries: 3

  # RabbitMQ 消息队列
  rabbitmq:
    image: rabbitmq:3-management
    container_name: vigo-rabbitmq
    ports:
      - "5672:5672"
      - "15672:15672"
    environment:
      RABBITMQ_DEFAULT_USER: guest
      RABBITMQ_DEFAULT_PASS: guest
    volumes:
      - rabbitmq-data:/var/lib/rabbitmq
    networks:
      - vigo-network
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "rabbitmqctl", "status"]
      interval: 30s
      timeout: 10s
      retries: 3

  # Nacos 服务发现与配置中心
  nacos:
    image: nacos/nacos-server:v2.3.0
    container_name: vigo-nacos
    ports:
      - "8848:8848"
      - "9848:9848"
    environment:
      MODE: standalone
      NACOS_APPLICATION_PORT: 8848
      JVM_XMS: 512m
      JVM_XMX: 512m
      JVM_XMN: 256m
      NACOS_SERVER_IP: nacos
    volumes:
      - nacos-data:/home/nacos/data
      - nacos-logs:/home/nacos/logs
    networks:
      - vigo-network
    restart: unless-stopped
    healthcheck:
      test:
        [
          "CMD",
          "wget",
          "--quiet",
          "--tries=1",
          "--spider",
          "http://localhost:8848/nacos/v1/console/health/liveness",
        ]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 120s

networks:
  vigo-network:
    driver: bridge

volumes:
  mysql-data:
  redis-data:
  rabbitmq-data:
  nacos-data:
  nacos-logs:
```

### 3.1 Docker Compose 环境变量说明

在Docker环境中，Vigo框架会按以下顺序加载配置：

1. `config.yaml` - 基础配置
2. `config.${APP_ENV}.yaml` - 环境特定配置（如 `config.docker.yaml`）
3. `config.local.yaml` - 本地覆盖配置

通过设置 `APP_ENV=docker`，框架会自动加载 `config.docker.yaml` 配置文件。

## 4. 部署步骤

### 4.1 准备配置文件

首先，确保已准备好 `config.docker.yaml` 配置文件，并根据您的环境调整配置项。

### 4.2 构建并启动服务

```bash
# 构建并启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f app

# 查看特定服务日志
docker-compose logs -f nacos
docker-compose logs -f mysql
```

### 4.3 验证部署

等待所有服务启动完成（可能需要2-3分钟），然后验证部署：

```bash
# 检查服务健康状态
docker-compose ps

# 访问以下地址验证服务状态：
# 应用主页: http://localhost:8080
# Nacos 控制台: http://localhost:8848/nacos (用户名: nacos, 密码: nacos)
# RabbitMQ 控制台: http://localhost:15672 (用户名: guest, 密码: guest)
# MySQL: 通过应用或客户端连接验证
# Redis: 通过应用连接验证
```

### 4.2 初始数据库设置

首次部署时需要初始化数据库：

```bash
# 进入 MySQL 容器
docker exec -it vigo-mysql mysql -uroot -p

# 创建数据库和表结构
CREATE DATABASE IF NOT EXISTS jw_platform_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE jw_platform_db;

# 执行你的 SQL 初始化脚本
```

### 4.3 验证部署

访问以下地址验证服务状态：

- 应用主页: http://localhost:8080
- Nacos 控制台: http://localhost:8848/nacos (用户名: nacos, 密码: nacos)
- RabbitMQ 控制台: http://localhost:15672 (用户名: guest, 密码: guest)

## 5. 生产环境最佳实践

### 5.1 安全配置

```yaml
# docker-compose.prod.yml
version: "3.8"

services:
  app:
    # ... 其他配置
    environment:
      # 使用密钥管理服务或外部配置
      - JWT_SECRET_FILE=/run/secrets/jwt_secret
      - DB_PASSWORD_FILE=/run/secrets/db_password
    secrets:
      - jwt_secret
      - db_password

secrets:
  jwt_secret:
    file: ./secrets/jwt_secret.txt
  db_password:
    file: ./secrets/db_password.txt
```

### 5.2 监控和日志

```yaml
# 添加监控服务
prometheus:
  image: prom/prometheus
  ports:
    - "9090:9090"
  volumes:
    - ./prometheus.yml:/etc/prometheus/prometheus.yml
  networks:
    - vigo-network

grafana:
  image: grafana/grafana
  ports:
    - "3000:3000"
  environment:
    - GF_SECURITY_ADMIN_PASSWORD=admin
  volumes:
    - grafana-storage:/var/lib/grafana
  networks:
    - vigo-network
```

### 5.3 健康检查

在应用 Dockerfile 中添加健康检查：

```dockerfile
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD curl -f http://localhost:8080/health || exit 1
```

## 6. 故障排除

### 6.1 常见问题

1. **端口冲突**

   ```bash
   # 检查端口占用
   docker-compose logs
   # 修改 docker-compose.yml 中的端口映射
   ```

2. **数据库连接失败**

   ```bash
   # 等待 MySQL 完全启动后再启动应用
   # 检查 MySQL 健康状态
   docker-compose exec mysql mysqladmin ping -h localhost -u root -p
   # 检查应用日志
   docker-compose logs app
   ```

3. **配置文件加载失败**

   ```bash
   # 确保配置文件路径正确
   # 检查挂载的配置文件
   docker-compose exec app ls -la /root/config/
   # 检查应用启动日志
   docker-compose logs app
   ```

4. **Nacos 服务不可用**

   ```bash
   # 检查 Nacos 日志
   docker-compose logs nacos
   # 等待 Nacos 完全启动（通常需要2分钟）
   # 检查 Nacos 健康端点
   docker-compose exec nacos wget --quiet --tries=1 --spider http://localhost:8848/nacos/v1/console/health/liveness
   ```

5. **应用启动失败**
   ```bash
   # 检查应用日志
   docker-compose logs app
   # 确认依赖服务已启动
   docker-compose ps
   # 检查配置文件语法
   docker-compose exec app cat /root/config/config.docker.yaml
   ```

### 6.2 性能优化

- 使用 Docker 多阶段构建减少镜像大小
- 合理配置各服务的内存限制
- 使用 Docker 的 build cache 加速构建
- 配置合适的容器重启策略

## 7. CI/CD 集成

创建 `.github/workflows/deploy.yml`：

```yaml
name: Deploy to Production

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Setup Docker Buildx
        uses: docker/setup-buildx-action@v2

      - name: Login to Registry
        uses: docker/login-action@v2
        with:
          registry: your-registry.com
          username: ${{ secrets.REGISTRY_USERNAME }}
          password: ${{ secrets.REGISTRY_PASSWORD }}

      - name: Build and Push
        uses: docker/build-push-action@v4
        with:
          context: .
          file: ./Dockerfile
          push: true
          tags: your-registry/vigo-app:${{ github.sha }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

      - name: Deploy to Server
        uses: appleboy/ssh-action@v0.1.5
        with:
          host: ${{ secrets.HOST }}
          username: ${{ secrets.USERNAME }}
          key: ${{ secrets.KEY }}
          script: |
            # 拉取最新镜像
            docker pull your-registry/vigo-app:${{ github.sha }}

            # 停止当前服务
            docker-compose down

            # 启动新版本服务
            docker-compose up -d

            # 清理旧镜像
            docker image prune -f
```

### 7.1 配置文件管理

在生产环境中，建议使用安全的方式管理配置文件：

1. 使用 Docker Secrets 管理敏感信息
2. 使用环境变量覆盖配置文件中的敏感信息
3. 在 CI/CD 流程中动态生成配置文件

这样就可以实现完整的 Docker 部署方案。
