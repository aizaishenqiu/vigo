# 打包发布指南

本文档介绍如何打包和发布 Vigo 应用，包括 Windows 和 Linux 平台的打包方法。

---

## 一、打包概述

### 1.1 打包方式

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| 手动编译 | 使用 go build 命令 | 开发测试 |
| 脚本打包 | 使用打包脚本 | 自动化部署 |
| Makefile | 使用 make 命令 | 标准化构建 |

### 1.2 打包产物

```
dist/
├── vigo-1.0.1-linux-amd64/      # Linux 部署包
│   ├── vigo-linux-amd64         # 可执行文件（已嵌入模板）
│   ├── config.yaml              # 配置文件
│   ├── start.sh                 # 启动脚本
│   ├── stop.sh                  # 停止脚本
│   ├── restart.sh               # 重启脚本
│   ├── vigo.service             # Systemd 服务文件
│   ├── 部署说明.md               # 部署文档
│   └── runtime/log/             # 日志目录
├── vigo-1.0.1-linux-amd64.zip   # Linux 部署包（压缩）
├── vigo-1.0.1-windows-amd64/    # Windows 部署包
│   ├── vigo.exe                 # 可执行文件
│   ├── config.yaml              # 配置文件
│   └── runtime/log/             # 日志目录
└── vigo-1.0.1-windows-amd64.zip # Windows 部署包（压缩）
```

## 二、手动编译

### 2.1 编译 Linux 版本

```bash
# Linux amd64
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -a -ldflags '-extldflags "-static"' -o vigo-linux-amd64 main.go

# Linux arm64
CGO_ENABLED=0 GOOS=linux GOARCH=arm64 go build -a -ldflags '-extldflags "-static"' -o vigo-linux-arm64 main.go
```

### 2.2 编译 Windows 版本

```bash
# Windows amd64
GOOS=windows GOARCH=amd64 go build -o vigo.exe main.go

# Windows 386
GOOS=windows GOARCH=386 go build -o vigo.exe main.go
```

### 2.3 编译参数说明

| 参数 | 说明 |
|------|------|
| CGO_ENABLED=0 | 禁用 CGO，生成纯静态二进制 |
| GOOS | 目标操作系统 |
| GOARCH | 目标架构 |
| -a | 强制重新构建 |
| -ldflags | 链接参数 |
| -extldflags "-static" | 静态链接 |

## 三、使用 Makefile 打包

### 3.1 Makefile 配置

```makefile
# Makefile
APP_NAME := vigo
VERSION := 1.0.1
BUILD_DIR := dist

# 编译
build:
	go build -o $(APP_NAME) main.go

# 编译 Linux
build-linux:
	CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -a -ldflags '-extldflags "-static"' -o $(BUILD_DIR)/$(APP_NAME)-linux-amd64 main.go

# 编译 Windows
build-windows:
	GOOS=windows GOARCH=amd64 go build -o $(BUILD_DIR)/$(APP_NAME).exe main.go

# 打包 Linux
package-linux: build-linux
	mkdir -p $(BUILD_DIR)/$(APP_NAME)-$(VERSION)-linux-amd64
	cp $(BUILD_DIR)/$(APP_NAME)-linux-amd64 $(BUILD_DIR)/$(APP_NAME)-$(VERSION)-linux-amd64/
	cp config.yaml $(BUILD_DIR)/$(APP_NAME)-$(VERSION)-linux-amd64/
	cp deploy/start.sh $(BUILD_DIR)/$(APP_NAME)-$(VERSION)-linux-amd64/
	cp deploy/stop.sh $(BUILD_DIR)/$(APP_NAME)-$(VERSION)-linux-amd64/
	cp deploy/restart.sh $(BUILD_DIR)/$(APP_NAME)-$(VERSION)-linux-amd64/
	cp deploy/vigo.service $(BUILD_DIR)/$(APP_NAME)-$(VERSION)-linux-amd64/
	cp deploy/部署说明.md $(BUILD_DIR)/$(APP_NAME)-$(VERSION)-linux-amd64/
	cd $(BUILD_DIR) && zip -r $(APP_NAME)-$(VERSION)-linux-amd64.zip $(APP_NAME)-$(VERSION)-linux-amd64

# 打包 Windows
package-windows: build-windows
	mkdir -p $(BUILD_DIR)/$(APP_NAME)-$(VERSION)-windows-amd64
	cp $(BUILD_DIR)/$(APP_NAME).exe $(BUILD_DIR)/$(APP_NAME)-$(VERSION)-windows-amd64/
	cp config.yaml $(BUILD_DIR)/$(APP_NAME)-$(VERSION)-windows-amd64/

# 打包所有
package-all: package-linux package-windows

# 清理
clean:
	rm -rf $(BUILD_DIR)

# 帮助
help:
	@echo "可用命令:"
	@echo "  make build          - 编译当前平台"
	@echo "  make build-linux    - 编译 Linux 版本"
	@echo "  make build-windows  - 编译 Windows 版本"
	@echo "  make package-linux  - 打包 Linux 部署包"
	@echo "  make package-windows- 打包 Windows 部署包"
	@echo "  make package-all    - 打包所有平台"
	@echo "  make clean          - 清理构建目录"
```

### 3.2 使用 Makefile

```bash
# 查看帮助
make help

# 编译 Linux 版本
make build-linux

# 打包 Linux 部署包
make package-linux

# 打包所有平台
make package-all

# 清理构建目录
make clean
```

## 四、使用打包脚本

### 4.1 Linux 打包脚本

创建 `deploy/package.sh`：

```bash
#!/bin/bash

APP_NAME="vigo"
VERSION="1.0.1"
BUILD_DIR="dist"

echo "开始打包 ${APP_NAME} ${VERSION}..."

# 清理旧文件
rm -rf ${BUILD_DIR}/${APP_NAME}-${VERSION}-linux-amd64
rm -f ${BUILD_DIR}/${APP_NAME}-${VERSION}-linux-amd64.zip

# 创建目录
mkdir -p ${BUILD_DIR}/${APP_NAME}-${VERSION}-linux-amd64/runtime/log

# 编译
echo "编译 Linux amd64..."
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -a -ldflags '-extldflags "-static"' -o ${BUILD_DIR}/${APP_NAME}-${VERSION}-linux-amd64/${APP_NAME}-linux-amd64 main.go

# 复制文件
echo "复制配置文件..."
cp config.yaml ${BUILD_DIR}/${APP_NAME}-${VERSION}-linux-amd64/
cp deploy/start.sh ${BUILD_DIR}/${APP_NAME}-${VERSION}-linux-amd64/
cp deploy/stop.sh ${BUILD_DIR}/${APP_NAME}-${VERSION}-linux-amd64/
cp deploy/restart.sh ${BUILD_DIR}/${APP_NAME}-${VERSION}-linux-amd64/
cp deploy/vigo.service ${BUILD_DIR}/${APP_NAME}-${VERSION}-linux-amd64/
cp deploy/部署说明.md ${BUILD_DIR}/${APP_NAME}-${VERSION}-linux-amd64/

# 设置权限
chmod +x ${BUILD_DIR}/${APP_NAME}-${VERSION}-linux-amd64/${APP_NAME}-linux-amd64
chmod +x ${BUILD_DIR}/${APP_NAME}-${VERSION}-linux-amd64/*.sh

# 压缩
echo "压缩部署包..."
cd ${BUILD_DIR}
zip -r ${APP_NAME}-${VERSION}-linux-amd64.zip ${APP_NAME}-${VERSION}-linux-amd64

echo "打包完成: ${BUILD_DIR}/${APP_NAME}-${VERSION}-linux-amd64.zip"
```

### 4.2 Windows 打包脚本

创建 `deploy/package.ps1`：

```powershell
$APP_NAME = "vigo"
$VERSION = "1.0.1"
$BUILD_DIR = "dist"

Write-Host "开始打包 ${APP_NAME} ${VERSION}..."

# 清理旧文件
if (Test-Path "${BUILD_DIR}\${APP_NAME}-${VERSION}-linux-amd64") {
    Remove-Item -Recurse -Force "${BUILD_DIR}\${APP_NAME}-${VERSION}-linux-amd64"
}
if (Test-Path "${BUILD_DIR}\${APP_NAME}-${VERSION}-linux-amd64.zip") {
    Remove-Item -Force "${BUILD_DIR}\${APP_NAME}-${VERSION}-linux-amd64.zip"
}

# 创建目录
New-Item -ItemType Directory -Force -Path "${BUILD_DIR}\${APP_NAME}-${VERSION}-linux-amd64\runtime\log"

# 编译
Write-Host "编译 Linux amd64..."
$env:CGO_ENABLED = "0"
$env:GOOS = "linux"
$env:GOARCH = "amd64"
go build -a -ldflags '-extldflags "-static"' -o "${BUILD_DIR}\${APP_NAME}-${VERSION}-linux-amd64\${APP_NAME}-linux-amd64" main.go

# 复制文件
Write-Host "复制配置文件..."
Copy-Item "config.yaml" "${BUILD_DIR}\${APP_NAME}-${VERSION}-linux-amd64\"
Copy-Item "deploy\start.sh" "${BUILD_DIR}\${APP_NAME}-${VERSION}-linux-amd64\"
Copy-Item "deploy\stop.sh" "${BUILD_DIR}\${APP_NAME}-${VERSION}-linux-amd64\"
Copy-Item "deploy\restart.sh" "${BUILD_DIR}\${APP_NAME}-${VERSION}-linux-amd64\"
Copy-Item "deploy\vigo.service" "${BUILD_DIR}\${APP_NAME}-${VERSION}-linux-amd64\"
Copy-Item "deploy\部署说明.md" "${BUILD_DIR}\${APP_NAME}-${VERSION}-linux-amd64\"

# 压缩
Write-Host "压缩部署包..."
Compress-Archive -Path "${BUILD_DIR}\${APP_NAME}-${VERSION}-linux-amd64" -DestinationPath "${BUILD_DIR}\${APP_NAME}-${VERSION}-linux-amd64.zip"

Write-Host "打包完成: ${BUILD_DIR}\${APP_NAME}-${VERSION}-linux-amd64.zip"
```

### 4.3 运行打包脚本

```bash
# Linux/Mac
chmod +x deploy/package.sh
./deploy/package.sh

# Windows PowerShell
.\deploy\package.ps1
```

## 五、版本管理

### 5.1 版本号规范

使用语义化版本号：`主版本号.次版本号.修订号`

- **主版本号**：不兼容的 API 修改
- **次版本号**：向下兼容的功能性新增
- **修订号**：向下兼容的问题修正

### 5.2 版本信息注入

```bash
# 编译时注入版本信息
VERSION=$(git describe --tags --always --dirty)
BUILD_TIME=$(date +%Y%m%d%H%M%S)
GIT_COMMIT=$(git rev-parse --short HEAD)

go build -ldflags "-X main.Version=${VERSION} -X main.BuildTime=${BUILD_TIME} -X main.GitCommit=${GIT_COMMIT}" -o vigo main.go
```

### 5.3 main.go 版本信息

```go
package main

var (
    Version   = "dev"
    BuildTime = "unknown"
    GitCommit = "unknown"
)

func main() {
    fmt.Printf("Version: %s\n", Version)
    fmt.Printf("Build Time: %s\n", BuildTime)
    fmt.Printf("Git Commit: %s\n", GitCommit)
    // ...
}
```

## 六、发布流程

### 6.1 发布前检查

- [ ] 更新版本号
- [ ] 更新 CHANGELOG
- [ ] 运行测试
- [ ] 编译检查

### 6.2 发布步骤

```bash
# 1. 创建标签
git tag -a v1.0.1 -m "Release v1.0.1"

# 3. 打包
make package-all

# 4. 上传到发布平台

```

## 七、持续集成

### 7.1 GitHub Actions

创建 `.github/workflows/release.yml`：

```yaml
name: Release

on:
  push:
    tags:
      - 'v*'

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Set up Go
        uses: actions/setup-go@v4
        with:
          go-version: '1.21'
      
      - name: Build Linux
        run: |
          CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -a -ldflags '-extldflags "-static"' -o vigo-linux-amd64 main.go
      
      - name: Build Windows
        run: |
          GOOS=windows GOARCH=amd64 go build -o vigo.exe main.go
      
      - name: Create Release
        uses: softprops/action-gh-release@v1
        with:
          files: |
            vigo-linux-amd64
            vigo.exe
```

## 八、注意事项

1. 生产环境编译时禁用 CGO（`CGO_ENABLED=0`）
2. 使用静态链接避免运行时依赖
3. 打包前确保配置文件正确
4. 版本号与 Git 标签保持一致
5. 发布前进行充分的测试
6. 保留历史版本以便回滚
