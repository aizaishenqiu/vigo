# Vigo 压力测试与性能指南

本框架内置了全模块压力测试工具，帮助开发者实时监控系统在极端并发下的表现。

---

## 1. 压力测试面板

访问路径：`/benchmark`

该面板集成了以下压测维度：
- **QPS (纯逻辑响应)**：测试 Web 引擎、Context 对象池及路由分发器的极限吞吐量。
- **Redis 延迟**：测试网络 IO 与 Redis 读写在高并发下的响应速度（微秒级）。
- **MySQL 延迟**：测试数据库连接池与 SQL 执行效率。
- **RabbitMQ 延迟**：测试消息队列投递的稳定性。

---

## 2. 压测方法

### 2.1 自动化面板压测
直接在浏览器打开 `/benchmark` 页面，点击 **“开始全模块压测”**。页面会以每秒多次的频率模拟并发请求，并通过 ECharts 实时绘制延迟曲线。

### 2.2 专业工具压测 (推荐)
对于更严苛的测试，建议使用 `wrk` 或 `ab`：

**测试纯 QPS (框架引擎性能)：**
```bash
wrk -t12 -c400 -d30s http://127.0.0.1:8080/benchmark/qps
```

**测试 Redis 读写性能：**
```bash
wrk -t12 -c400 -d30s http://127.0.0.1:8080/benchmark/redis
```

## 3. 性能指标参考

| 模块 | 理想延迟 | 预警延迟 | 优化建议 |
| :--- | :--- | :--- | :--- |
| **QPS (纯逻辑)** | < 1ms | > 5ms | 检查中间件是否过多，开启 Context 复用 |
| **Redis** | < 500us | > 2ms | 检查网络延迟，增加 Redis 连接池大小 |
| **MySQL** | < 2ms | > 10ms | 优化 SQL 索引，调整数据库最大连接数 |
| **RabbitMQ** | < 1ms | > 5ms | 检查磁盘 IO，开启异步确认模式 |

---

## 4. 性能调优建议

1. **启用 Context 池**：框架默认已开启 `sync.Pool`，请勿手动销毁 Context。
2. **减少反射**：在 Controller 中尽量避免大规模使用 `reflect` 库。
3. **异步日志**：生产环境建议将 `Logger` 中间件配置为异步写入，以消除磁盘 IO 对 QPS 的影响。
4. **多环境配置**：压测前请确保 `APP_ENV=prod`，生产模式会关闭冗余的调试日志。
