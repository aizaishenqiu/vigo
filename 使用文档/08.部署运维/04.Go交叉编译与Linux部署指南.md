# Go交叉编译与Linux部署指南

本文档介绍如何使用Go交叉编译功能将Vigo应用编译为Linux二进制文件，并部署到Linux服务器。

---

## 1. 交叉编译简介

Go语言支持交叉编译，无需在目标平台上编译即可生成对应平台的可执行文件。这对于部署到Linux服务器非常有用。

### 1.1 支持的目标平台

Go支持以下常见的Linux发行版架构：

- linux/amd64 (x86-64)
- linux/arm64 (ARM 64-bit)
- linux/arm (ARM 32-bit)
- linux/386 (x86-32)

## 2. 交叉编译到Linux

### 2.1 编译到Linux amd64

```bash
# 设置目标操作系统和架构
GOOS=linux GOARCH=amd64 go build -o vigo-linux-amd64 main.go
```

### 2.2 编译到Linux arm64

```bash
# 适用于树莓派4、AWS Graviton处理器等
GOOS=linux GOARCH=arm64 go build -o vigo-linux-arm64 main.go
```

### 2.3 优化编译参数

为了生成更小的二进制文件并移除调试信息：

```bash
# Linux amd64
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -a -ldflags '-extldflags "-static"' -o vigo-linux-amd64 main.go

# Linux arm64
CGO_ENABLED=0 GOOS=linux GOARCH=arm64 go build -a -ldflags '-extldflags "-static"' -o vigo-linux-arm64 main.go
```

参数说明：

- `CGO_ENABLED=0`: 禁用CGO，生成纯静态链接的二进制文件
- `-a`: 强制重新构建所有包
- `-ldflags '-extldflags "-static"'`: 生成静态链接的二进制文件

### 2.4 创建编译脚本

创建 `build-linux.sh` 脚本：

```bash
#!/bin/bash

echo "开始交叉编译到Linux..."

# 编译Linux amd64版本
echo "编译Linux amd64版本..."
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -a -ldflags '-extldflags "-static"' -o bin/vigo-linux-amd64 .

# 编译Linux arm64版本
echo "编译Linux arm64版本..."
CGO_ENABLED=0 GOOS=linux GOARCH=arm64 go build -a -ldflags '-extldflags "-static"' -o bin/vigo-linux-arm64 .

echo "交叉编译完成！"
echo "生成的文件:"
ls -la bin/vigo-linux-*
```

## 3. Linux服务器部署

### 3.1 服务器准备

在Linux服务器上准备部署环境：

```bash
# 创建部署目录
sudo mkdir -p /opt/vigo-app
sudo chown $USER:$USER /opt/vigo-app
cd /opt/vigo-app

# 创建必要的目录结构
mkdir -p logs config
```

### 3.2 传输二进制文件

将编译好的二进制文件传输到服务器：

```bash
# 使用scp传输
scp bin/vigo-linux-amd64 user@server:/opt/vigo-app/vigo

# 或使用rsync
rsync -avz bin/vigo-linux-amd64 user@server:/opt/vigo-app/vigo
```

### 3.3 配置文件部署

将配置文件传输到服务器：

```bash
# 传输配置文件
scp config.prod.yaml user@server:/opt/vigo-app/config/config.prod.yaml

# 或者在服务器上创建配置文件
nano /opt/vigo-app/config/config.prod.yaml
```

### 3.4 设置执行权限

```bash
chmod +x /opt/vigo-app/vigo
```

## 4. 系统服务配置

### 4.1 创建Systemd服务

创建 `/etc/systemd/system/vigo.service`：

```ini
[Unit]
Description=Vigo Application
After=network.target
Wants=network-online.target

[Service]
Type=simple
User=www-data
Group=www-data
WorkingDirectory=/opt/vigo-app
ExecStart=/opt/vigo-app/vigo
Restart=always
RestartSec=10
Environment=APP_ENV=prod
Environment=CONFIG_FILE=/opt/vigo-app/config/config.prod.yaml

# 日志配置
StandardOutput=journal
StandardError=journal
SyslogIdentifier=vigo

# 安全配置
NoNewPrivileges=true
PrivateTmp=true

[Install]
WantedBy=multi-user.target
```

### 4.2 启动服务

```bash
# 重新加载systemd配置
sudo systemctl daemon-reload

# 启用服务（开机自启）
sudo systemctl enable vigo

# 启动服务
sudo systemctl start vigo

# 查看服务状态
sudo systemctl status vigo

# 查看服务日志
sudo journalctl -u vigo -f
```

## 5. 环境变量配置

### 5.1 创建环境配置文件

创建 `/opt/vigo-app/.env` 或使用系统环境变量：

```bash
# 在 /etc/environment 中添加
export APP_ENV=prod
export DB_HOST=mysql-server
export REDIS_HOST=redis-server
export JWT_SECRET=your-production-secret-here
```

### 5.2 在Systemd服务中配置环境变量

在服务文件中添加多个环境变量：

```ini
[Service]
# ... 其他配置
Environment=APP_ENV=prod
Environment=DB_HOST=mysql-server
Environment=DB_PORT=3306
Environment=DB_USER=app_user
Environment=DB_PASS=secure_password
Environment=REDIS_HOST=redis-server
Environment=REDIS_PORT=6379
Environment=JWT_SECRET=your-production-secret-here
```

## 6. 进程管理与监控

### 6.1 使用Supervisor（可选）

如果不想使用systemd，也可以使用supervisor：

```bash
# 安装supervisor
sudo apt-get install supervisor

# 创建配置文件 /etc/supervisor/conf.d/vigo.conf
[program:vigo]
command=/opt/vigo-app/vigo
directory=/opt/vigo-app
user=www-data
autostart=true
autorestart=true
redirect_stderr=true
stdout_logfile=/opt/vigo-app/logs/supervisor.log
environment=APP_ENV="prod",DB_HOST="mysql-server"
```

### 6.2 启动Supervisor管理

```bash
# 重新加载配置
sudo supervisorctl reread
sudo supervisorctl update

# 启动应用
sudo supervisorctl start vigo

# 查看状态
sudo supervisorctl status
```

## 7. 部署脚本自动化

### 7.1 创建部署脚本

创建 `deploy.sh`：

```bash
#!/bin/bash

set -e  # 遇到错误立即退出

APP_NAME="vigo"
REMOTE_HOST="your-server.com"
REMOTE_USER="deploy"
REMOTE_DIR="/opt/vigo-app"

echo "开始部署 $APP_NAME..."

# 1. 交叉编译
echo "1. 交叉编译到Linux amd64..."
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -a -ldflags '-extldflags "-static"' -o bin/vigo-linux-amd64 main.go

# 2. 停止远程服务
echo "2. 停止远程服务..."
ssh $REMOTE_USER@$REMOTE_HOST "sudo systemctl stop $APP_NAME"

# 3. 备份当前版本
echo "3. 备份当前版本..."
ssh $REMOTE_USER@$REMOTE_HOST "cd $REMOTE_DIR && sudo cp vigo vigo.backup.$(date +%Y%m%d_%H%M%S) || true"

# 4. 传输新版本
echo "4. 传输新版本..."
scp bin/vigo-linux-amd64 $REMOTE_USER@$REMOTE_HOST:$REMOTE_DIR/vigo.new

# 5. 替换并设置权限
ssh $REMOTE_USER@$REMOTE_HOST "cd $REMOTE_DIR && sudo mv vigo.new vigo && sudo chmod +x vigo"

# 6. 启动服务
echo "5. 启动服务..."
ssh $REMOTE_USER@$REMOTE_HOST "sudo systemctl start $APP_NAME"

# 7. 验证服务状态
sleep 5
STATUS=$(ssh $REMOTE_USER@$REMOTE_HOST "sudo systemctl is-active $APP_NAME")
if [ "$STATUS" = "active" ]; then
    echo "部署成功！服务状态: $STATUS"
else
    echo "部署失败！服务状态: $STATUS"
    exit 1
fi

echo "部署完成！"
```

## 8. 性能优化建议

### 8.1 系统级优化

```bash
# 调整文件描述符限制
echo "www-data soft nofile 65536" | sudo tee -a /etc/security/limits.conf
echo "www-data hard nofile 65536" | sudo tee -a /etc/security/limits.conf

# 调整网络参数
sudo tee -a /etc/sysctl.conf << EOF
net.core.somaxconn = 65535
net.ipv4.tcp_max_syn_backlog = 65535
net.ipv4.ip_local_port_range = 1024 65535
EOF

sudo sysctl -p
```

### 8.2 应用级优化

在配置文件中调整连接池和其他参数：

```yaml
database:
  max_open_conns: 100 # 根据服务器性能调整
  max_idle_conns: 25 # 根据服务器性能调整
  conn_max_lifetime: 3600

redis:
  pool_size: 50 # 根据服务器性能调整

app:
  port: 8080
  debug: false # 生产环境设为false
  mode: prod
```

## 9. 安全注意事项

### 9.1 文件权限

```bash
# 设置正确的文件权限
sudo chown -R www-data:www-data /opt/vigo-app
sudo chmod -R 755 /opt/vigo-app
sudo chmod 600 /opt/vigo-app/config/*  # 配置文件仅所有者可读
```

### 9.2 防火墙配置

```bash
# 使用ufw配置防火墙
sudo ufw allow ssh
sudo ufw allow 8080  # 应用端口
sudo ufw enable
```

## 10. 监控与日志

### 10.1 日志轮转

创建日志轮转配置 `/etc/logrotate.d/vigo`：

```bash
/opt/vigo-app/logs/*.log {
    daily
    missingok
    rotate 52
    compress
    delaycompress
    notifempty
    create 644 www-data www-data
    postrotate
        systemctl reload vigo.service > /dev/null 2>&1 || true
    endscript
}
```

### 10.2 健康检查

在应用中添加健康检查端点，便于监控：

```bash
# 检查服务是否运行
curl -f http://localhost:8080/health

# 或使用systemctl检查状态
systemctl is-active vigo
```

通过以上步骤，您可以成功将Vigo应用交叉编译到Linux并部署到生产环境。
