# Nacos 配置中心使用指南

## 概述

Vigo 框架集成了 Nacos 配置中心，提供动态配置管理、实时更新、多格式支持等能力。

## 功能特性

- ✅ 动态配置加载和更新
- ✅ 支持 JSON、YAML、Properties 格式
- ✅ 配置变更监听
- ✅ 配置缓存
- ✅ 多环境配置
- ✅ 配置加密

## 快速开始

### 1. 安装 Nacos

```bash
# Docker 安装
docker run -d \
  -p 8848:8848 \
  -p 9848:9848 \
  -p 9849:9849 \
  --name nacos \
  -e MODE=standalone \
  nacos/nacos-server:2.3.0
```

访问 http://localhost:8848/nacos，默认账号密码：nacos/nacos

### 2. 配置 Nacos

在 Nacos 控制台创建配置：

- **Data ID**: `app.yaml`
- **Group**: `DEFAULT_GROUP`
- **配置格式**: YAML
- **配置内容**:

```yaml
database:
  driver: mysql
  dsn: root:password@tcp(127.0.0.1:3306)/dbname?charset=utf8mb4
  max_idle_conns: 10
  max_open_conns: 100

redis:
  addr: 127.0.0.1:6379
  password: ""
  db: 0
  pool_size: 100

app:
  name: myapp
  version: 1.0.0
  debug: true
```

### 3. 代码中使用

```go
package main

import (
    "context"
    "fmt"
    "log"
    "vigo/framework/config"
)

func main() {
    // 创建 Nacos 配置
    nacosCfg, err := config.NewNacosConfig(&config.NacosConfigOptions{
        ServerAddr:  "127.0.0.1:8848",
        NamespaceId: "public",
        DataId:      "app.yaml",
        Group:       "DEFAULT_GROUP",
        Format:      "yaml",
        TimeoutMs:   5000,
        LogDir:      "logs/nacos",
        CacheDir:    "cache/nacos",
        LogLevel:    "info",
    })
    if err != nil {
        log.Fatal(err)
    }

    // 加载配置
    ctx := context.Background()
    if err := nacosCfg.Load(ctx); err != nil {
        log.Fatal(err)
    }

    // 监听配置变化
    go func() {
        if err := nacosCfg.Watch(ctx); err != nil {
            log.Printf("Watch error: %v", err)
        }
    }()

    // 获取配置
    dbDSN, err := nacosCfg.Get("database.dsn")
    if err != nil {
        log.Fatal(err)
    }
    fmt.Println("Database DSN:", dbDSN)

    // 获取整个配置段
    dbConfig, err := nacosCfg.Get("database")
    if err != nil {
        log.Fatal(err)
    }
    fmt.Printf("Database Config: %+v\n", dbConfig)

    // 保持运行
    select {}
}
```

## 配置选项

### NacosConfigOptions

```go
type NacosConfigOptions struct {
    // Nacos 服务器地址
    ServerAddr string `yaml:"server_addr"`
    
    // 命名空间 ID
    NamespaceId string `yaml:"namespace_id"`
    
    // 配置 Data ID
    DataId string `yaml:"data_id"`
    
    // 配置 Group
    Group string `yaml:"group"`
    
    // 配置格式：yaml, json, properties
    Format string `yaml:"format"`
    
    // 超时时间（毫秒）
    TimeoutMs uint64 `yaml:"timeout_ms"`
    
    // 日志目录
    LogDir string `yaml:"log_dir"`
    
    // 缓存目录
    CacheDir string `yaml:"cache_dir"`
    
    // 日志级别：debug, info, warn, error
    LogLevel string `yaml:"log_level"`
    
    // 用户名（可选）
    Username string `yaml:"username"`
    
    // 密码（可选）
    Password string `yaml:"password"`
}
```

## 高级用法

### 1. 获取配置值

```go
// 获取字符串
value, err := nacosCfg.Get("app.name")

// 获取整数
port, err := nacosCfg.GetInt("server.port")

// 获取布尔值
debug, err := nacosCfg.GetBool("app.debug")

// 获取浮点数
rate, err := nacosCfg.GetFloat("rate.limit")

// 获取字符串数组
hosts, err := nacosCfg.GetStringSlice("cluster.hosts")

// 获取整个配置段
config, err := nacosCfg.Get("database")

// 获取配置并转换为结构体
type DBConfig struct {
    Driver       string `json:"driver"`
    DSN          string `json:"dsn"`
    MaxIdleConns int    `json:"max_idle_conns"`
    MaxOpenConns int    `json:"max_open_conns"`
}

var dbCfg DBConfig
err := nacosCfg.GetStruct("database", &dbCfg)
```

### 2. 监听配置变化

```go
// 监听特定 key 的变化
nacosCfg.AddListener("database.dsn", func(key string, value interface{}) {
    fmt.Printf("Config changed: %s = %v\n", key, value)
    
    // 重新初始化数据库连接
    ReinitDB(value.(string))
})

// 监听整个配置的变化
nacosCfg.AddListener("database", func(key string, value interface{}) {
    fmt.Printf("Database config changed: %+v\n", value)
})
```

### 3. 发布配置

```go
// 发布配置更新
err := nacosCfg.Publish(&config.PublishConfig{
    DataId:  "app.yaml",
    Group:   "DEFAULT_GROUP",
    Content: `
app:
  name: myapp
  version: 2.0.0
`,
    Format: "yaml",
})
if err != nil {
    log.Fatal(err)
}
```

### 4. 删除配置

```go
err := nacosCfg.Delete(&config.DeleteConfig{
    DataId: "app.yaml",
    Group:  "DEFAULT_GROUP",
})
```

### 5. 多环境配置

```go
// 开发环境
devCfg, _ := config.NewNacosConfig(&config.NacosConfigOptions{
    ServerAddr:  "127.0.0.1:8848",
    NamespaceId: "dev",  // 开发环境命名空间
    DataId:      "app.yaml",
    Group:       "DEFAULT_GROUP",
    Format:      "yaml",
})

// 生产环境
prodCfg, _ := config.NewNacosConfig(&config.NacosConfigOptions{
    ServerAddr:  "192.168.1.100:8848",
    NamespaceId: "prod",  // 生产环境命名空间
    DataId:      "app.yaml",
    Group:       "DEFAULT_GROUP",
    Format:      "yaml",
})
```

### 6. 配置加密

```go
// Nacos 支持配置加密
// 在配置内容中使用加密值
// app.password: ENC(AES256_CFB,encrypted_value)

// 框架会自动解密
password, _ := nacosCfg.Get("app.password")
```

## 配置示例

### 数据库配置

```yaml
# app.yaml
database:
  driver: mysql
  dsn: ${DB_USER}:${DB_PASS}@tcp(${DB_HOST}:${DB_PORT})/${DB_NAME}?charset=utf8mb4
  max_idle_conns: 10
  max_open_conns: 100
  conn_max_lifetime: 1h
  
  # 读写分离
  read_replicas:
    - dsn: root:pass@tcp(192.168.1.101:3306)/dbname
    - dsn: root:pass@tcp(192.168.1.102:3306)/dbname
```

### Redis 配置

```yaml
# app.yaml
redis:
  addr: 127.0.0.1:6379
  password: ${REDIS_PASSWORD}
  db: 0
  pool_size: 100
  min_idle_conns: 10
  max_conn_age: 1h
  dial_timeout: 5s
  read_timeout: 3s
  write_timeout: 3s
```

### 应用配置

```yaml
# app.yaml
app:
  name: myapp
  version: 1.0.0

  debug: true
  
server:
  port: 8080

  level: debug
```

### 微服务配置


## 最佳实践

### 1. 配置分层

```yaml
# 基础配置（所有环境共享）
# Data ID: app-common.yaml
database:
  driver: mysql
  max_idle_conns: 10

# 开发环境配置

# Data ID: app-prod.yaml
database:
  dsn: ${DB_USER}:${DB_PASS}@tcp(${DB_HOST}:3306)/prod_db

  debug: false
```

### 2. 使用环境变量

```yaml
# 在配置中使用环境变量

  dsn: ${DB_USER}:${DB_PASS}@tcp(${DB_HOST}:${DB_PORT})/${DB_NAME}
```

### 3. 配置热更新

```go
// 监听配置变化并热更新
nacosCfg.AddListener("database.dsn", func(key string, value interface{}) {
    // 1. 关闭旧连接
    oldDB.Close()
    
    // 2. 创建新连接
    newDB, _ = db.NewDB(&db.DBConfig{
        DSN: value.(string),
    })
    
    // 3. 替换旧连接
    oldDB = newDB
    
    log.Println("Database connection updated")
})
```

### 4. 配置备份

```go
// 定期备份配置
func backupConfig(cfg *config.NacosConfig) {
    ticker := time.NewTicker(24 * time.Hour)
    defer ticker.Stop()
    
    for range ticker.C {
        // 获取当前配置
        allConfig, _ := cfg.GetAll()
        
        // 序列化并保存
        data, _ := json.MarshalIndent(allConfig, "", "  ")
        ioutil.WriteFile("backup/config_"+time.Now().Format("20060102")+".json", data, 0644)
    }
}
```

## 故障排查

### 1. 连接失败

```bash
# 检查 Nacos 服务状态
curl http://127.0.0.1:8848/nacos/

# 查看 Nacos 日志
tail -f nacos/logs/start.out

# 检查网络连通性
telnet 127.0.0.1 8848
```

### 2. 配置加载失败

```go
// 启用详细日志
&config.NacosConfigOptions{
    LogLevel: "debug",  // 设置为 debug 查看详细信息
    LogDir:   "logs/nacos",
}
```

### 3. 配置不更新

```go
// 检查监听器是否注册
nacosCfg.AddListener("key", callback)

// 检查 Watch 是否运行
go nacosCfg.Watch(ctx)
```

## 性能优化

### 1. 配置缓存

```go
// Nacos 客户端会自动缓存配置到本地
// 缓存目录：cache/nacos
// 即使 Nacos 服务器不可用，也能读取缓存的配置
```

### 2. 长轮询优化

```go
// Nacos 使用长轮询机制
// 配置变更时才会推送，减少无效请求
&config.NacosConfigOptions{
    TimeoutMs: 5000,  // 长轮询超时时间
}
```
