# 代码生成工具 (vigoctl)

Vigo 框架提供了强大的代码生成工具 `vigoctl`，可以帮助开发者快速生成 API、模型、服务和中间件代码，大大提高开发效率。

## 安装

### 从源码安装

```bash
go install github.com/yourorg/vigo/cmd/vigoctl@latest
```

### 直接编译

```bash
cd M:\www\qiuye-saas
go build -o vigoctl.exe ./cmd/vigoctl
```

## 命令概览

```bash
vigoctl <command> [options]
```

### 可用命令

| 命令 | 说明 |
|------|------|
| `generate api` | 从 IDL 生成 API 代码 |
| `generate model` | 生成模型代码 |
| `generate service` | 生成服务代码 |
| `generate middleware` | 生成中间件代码 |
| `init idl` | 初始化 IDL 项目 |
| `validate idl` | 验证 IDL 文件 |
| `upgrade` | 升级 vigoctl |

## API 代码生成

### 基本用法

```bash
vigoctl generate api -i apis/user.api -o app/api -style mvc
```

### 参数说明

| 参数 | 别名 | 说明 | 默认值 |
|------|------|------|--------|
| `--input` | `-i` | IDL 文件路径（必需） | - |
| `--output` | `-o` | 输出目录（必需） | - |
| `--style` | `-s` | 代码风格：mvc, rpc | mvc |

### 示例

#### 生成 MVC 风格的 API 代码

```bash
vigoctl generate api -i apis/user.api -o app/api --style mvc
```

生成的文件结构：

```
app/api/
├── user_api.go        # API 处理器
├── user_request.go    # 请求结构
└── user_response.go   # 响应结构
```

#### 生成 RPC 风格的 API 代码

```bash
vigoctl generate api -i apis/user.api -o app/rpc --style rpc
```

生成的文件结构：

```
app/rpc/
├── user_service.go    # RPC 服务实现
├── user_pb.go         # Protobuf 生成的代码
└── user_client.go     # RPC 客户端
```

## 模型代码生成

### 基本用法

```bash
vigoctl generate model -i apis/user.api -o app/model
```

### 参数说明

| 参数 | 别名 | 说明 | 默认值 |
|------|------|------|--------|
| `--input` | `-i` | IDL 文件路径（必需） | - |
| `--output` | `-o` | 输出目录（必需） | - |
| `--table-prefix` | `-p` | 数据库表前缀 | - |

### 示例

```bash
vigoctl generate model -i apis/user.api -o app/model --table-prefix tp_
```

生成的文件结构：

```
app/model/
├── user.go            # User 模型
├── user_base.go       # User 基础模型（不要修改）
└── user_custom.go     # User 自定义模型（可修改）
```

| 参数 | 别名 | 说明 | 默认值 |

## 中间件代码生成

### 基本用法

```bash
vigoctl generate middleware -n Auth -o app/middleware
```

### 参数说明

| 参数 | 别名 | 说明 | 默认值 |
|------|------|------|--------|
| `--name` | `-n` | 中间件名称（必需） | - |
| `--output` | `-o` | 输出目录（必需） | - |
| `--type` | `-t` | 中间件类型：auth, cors, rate, log | custom |

### 示例

#### 生成认证中间件

```bash
vigoctl generate middleware -n Auth -o app/middleware --type auth
```

生成的文件结构：

```
app/middleware/
└── auth.go              # 认证中间件
```

生成的中间件代码示例：

```go
package middleware

import (
    "vigo/framework/mvc"
)

// Auth 认证中间件
func Auth() mvc.HandlerFunc {
    return func(ctx *mvc.Context) error {
        // 获取 Token
        token := ctx.GetHeader("Authorization")
        
        // 验证 Token
        if token == "" {
            return ctx.JSON(401, map[string]string{
                "error": "Unauthorized",
            })
        }
        
        // 解析 Token 并设置用户信息
        // ...
        
        return ctx.Next()
    }
}
```

#### 生成自定义中间件

```bash

```

## IDL 语法详解

### 基本结构

```yaml
# API 名称
name: user-api

# API 版本
version: v1

# 基础路径
basePath: /api/v1

# 支持的协议
schemes:
  - http
  - https

# 请求格式
consumes:
  - application/json

# 响应格式
produces:
  - application/json

# 标签分类
tags:
  - name: user
    description: 用户管理

# API 路径定义
paths:
  - path: /users
    method: GET
    summary: 获取用户列表
    operationId: listUsers
    tags:
      - user
    parameters:
      - name: page
        in: query
        type: integer
        default: 1
      - name: pageSize
        in: query
        type: integer
        default: 10
    responses:
      - statusCode: "200"
        description: 成功
        schema:
          type: array
          items:
            $ref: "#/models/User"

# 数据模型定义
models:
  - name: User
    description: 用户信息
    properties:
      id:
        type: string
        format: uuid
        description: 用户 ID
      username:
        type: string
        description: 用户名
        required: true
      email:
        type: string
        format: email
        description: 邮箱
        required: true
      createdAt:
        type: string
        format: date-time
        description: 创建时间
```

### 参数位置

| 位置 | 说明 | 示例 |
|------|------|------|
| `path` | 路径参数 | `/users/{id}` |
| `query` | 查询参数 | `?page=1` |
| `header` | 请求头 | `Authorization: Bearer xxx` |
| `body` | 请求体 | JSON 对象 |

### 数据类型

| 类型 | 说明 | 格式 |
|------|------|------|
| `string` | 字符串 | - |
| `integer` | 整数 | int32, int64 |
| `number` | 数字 | float, double |
| `boolean` | 布尔值 | - |
| `array` | 数组 | items: { type: string } |
| `object` | 对象 | properties: { ... } |

### 数据验证

```yaml
parameters:
  - name: username
    in: query
    type: string
    required: true
    min: 3          # 最小长度
    max: 32         # 最大长度
    pattern: "^[a-zA-Z0-9_]+$"  # 正则匹配
    enum:           # 枚举值
      - admin
      - user
      - guest
```

## 最佳实践

### 1. 项目结构组织

```
project/
├── apis/                    # IDL 定义
│   ├── user.api
│   ├── order.api
│   └── product.api
├── app/
│   ├── api/                # 生成的 API 代码
│   ├── model/              # 生成的模型代码
│   ├── service/            # 生成的服务代码
│   └── middleware/         # 生成的中间件代码
└── vigoctl.yaml            # vigoctl 配置
```

### 2. 版本控制

- IDL 文件应该纳入版本控制
- 生成的代码也应该纳入版本控制
- 手动修改的代码放在 `*_custom.go` 文件中

### 3. 命名规范

- API 文件名：`{module}.api`
- 模型名：大驼峰 `User`
- 操作 ID：小驼峰 `listUsers`
- 路径：复数形式 `/users`

### 4. 代码定制

```go
// user_custom.go - 手动修改的文件
package model

// CustomMethod 自定义方法
func (u *User) CustomMethod() error {
    // 自定义业务逻辑
    return nil
}
```

## 常见问题

### Q: 生成的代码可以修改吗？

A: 可以，但建议：
- 不要修改 `*_base.go` 文件（会被覆盖）
- 在 `*_custom.go` 文件中添加自定义逻辑
- 重新生成前备份修改

### Q: 如何添加自定义字段？

A: 在 IDL 的 models 中添加字段定义，然后重新生成：

```yaml
models:
  - name: User
    properties:
      customField:
        type: string
        description: 自定义字段
```

### Q: 支持哪些数据库？

A: 目前支持：
- MySQL
- SQL Server
- PostgreSQL（即将支持）

### Q: 如何自定义模板？

A: 可以通过配置文件自定义代码模板：

```yaml
# vigoctl.yaml
templates:
  model: templates/model.tmpl
  service: templates/service.tmpl
```
