# config.yaml 配置文件完整说明

## 概述

`config.yaml` 是 Vigo 框架的主配置文件，采用 YAML 格式，支持环境变量占位符。

### 环境变量语法

```yaml
# 基本格式
${ENV_VAR}

# 带默认值格式
${ENV_VAR:default_value}
```

**示例**：
```yaml
database:
  host: ${DB_HOST}           # 从 DB_HOST 环境变量读取
  port: ${DB_PORT:3306}      # 从 DB_PORT 读取，未设置则使用 3306
  name: ${DB_NAME:myapp}     # 从 DB_NAME 读取，未设置则使用 myapp
```

## 应用基础配置

### 配置路径
```yaml
app:
```

### 参数说明

| 参数 | 类型 | 默认值 | 必填 | 说明 |
|------|------|--------|------|------|
| `name` | string | `vago` | 是 | 应用名称，用于日志、监控等 |
| `version` | string | `1.0.0` | 是 | 应用版本号 |
| `port` | int | `8080` | 是 | HTTP 服务监听端口 |
| `debug` | bool | `false` | 是 | 是否开启调试模式 |
| `mode` | string | `prod` | 是 | 运行模式：`dev` \| `test` \| `prod` |
| `console` | bool | `true` | 是 | 是否显示控制台输出 |
| `auto_kill_port` | bool | `true` | 是 | 启动时是否自动杀掉占用端口的进程 |

### 完整示例

```yaml
app:
  name: my-app              # 应用名称
  version: 1.0.0            # 版本号
  port: 8080                # 监听端口
  debug: false              # 生产环境关闭调试
  mode: prod                # 生产环境
  console: true             # 显示控制台输出
  auto_kill_port: true      # 自动处理端口占用
```

### 环境模式说明

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| `dev` | 开发环境 | 本地开发，自动允许 CORS |
| `test` | 测试环境 | 测试服务器，部分安全限制 |
| `prod` | 生产环境 | 线上服务器，严格安全限制 |

### 注意事项

⚠️ **生产环境必须设置**：
- `debug: false` - 关闭调试模式
- `mode: prod` - 生产环境模式

## 安全防护配置

### 配置路径
```yaml
security:
```

### 参数说明

| 参数 | 类型 | 默认值 | 必填 | 说明 |
|------|------|--------|------|------|
| `enable_security_middleware` | bool | `true` | 是 | 是否启用全局安全中间件 |
| `enable_csrf_protection` | bool | `false` | 是 | 是否启用 CSRF 保护 |
| `enable_rate_limit` | bool | `false` | 是 | 是否启用速率限制 |
| `rate_limit` | int | `100` | 是 | 每秒请求数限制 |
| `enable_cors_domain_check` | bool | `true` | 是 | 是否启用 CORS 域名验证 |
| `allowed_domains` | []string | `[]` | 是 | 允许的域名列表 |
| `ip_whitelist` | []string | `[]` | 是 | IP 白名单 |
| `ip_blacklist` | []string | `[]` | 是 | IP 黑名单 |

### 完整示例

```yaml
security:
  # 启用安全中间件（强烈建议启用）
  enable_security_middleware: true
  
  # CSRF 保护（Web 应用建议启用）
  enable_csrf_protection: false
  
  # 速率限制（防刷）
  enable_rate_limit: false
  rate_limit: 100  # 每秒 100 个请求
  
  # CORS 域名验证
  enable_cors_domain_check: true
  allowed_domains:
    - "https://www.example.com"
    - "https://api.example.com"
    - "https://admin.example.com"
  
  # IP 白名单（可选）
  ip_whitelist:
    - "192.168.1.100"
    - "10.0.0.0/8"        # CIDR 格式
    - "172.16.*.*"        # 通配符格式
  
  # IP 黑名单
  ip_blacklist:
    - "1.2.3.4"
    - "5.6.7.0/24"
```

### 安全中间件功能

启用 `enable_security_middleware: true` 后，自动拦截：

- ✅ **SQL 注入攻击**
- ✅ **XSS 跨站脚本攻击**
- ✅ **命令注入攻击**
- ✅ **路径遍历攻击**
- ✅ **其他常见 Web 攻击**

### CORS 配置说明

#### 开发环境（mode: dev）
- 自动允许所有 CORS 请求
- 无需配置 `allowed_domains`
- Electron/Tauri 自动放行

#### 生产环境（mode: prod）
- 严格验证域名
- 只允许 `allowed_domains` 中的域名
- Electron/Tauri 自动放行（无需配置）

### IP 规则格式

支持三种格式：

1. **精确 IP**
   ```yaml
   ip_whitelist:
     - "192.168.1.100"
   ```

2. **CIDR 格式**
   ```yaml
   ip_whitelist:
     - "192.168.1.0/24"    # 256 个 IP
     - "10.0.0.0/8"        # 1600 万个 IP
   ```

3. **通配符格式**
   ```yaml
   ip_whitelist:
     - "192.168.1.*"       # 256 个 IP
     - "172.16.*.*"        # 65536 个 IP
   ```

| `app_id` | string | `""` | 条件 | 应用 ID（支付宝开放平台申请） |

| `is_provider` | bool | `false` | 是 | 是否服务商模式 |
| `provider_pid` | string | `""` | 条件 | 服务商 PID（服务商模式必填） |

```yaml
payment:
  alipay:
    enabled: true

    return_url: "https://www.example.com/pay/return"
    is_provider: false
    provider_pid: ""  # 服务商模式填写服务商 PID
```

## 数据库配置

### 主数据库配置

#### 配置路径
```yaml
database:
```

#### 参数说明

| 参数 | 类型 | 默认值 | 必填 | 说明 |
|------|------|--------|------|------|
| `driver` | string | `mysql` | 是 | 数据库驱动：`mysql` \| `postgres` \| `sqlite` \| `mssql` |
| `host` | string | - | 条件 | 数据库主机（SQLite 不需要） |
| `port` | int | `3306` | 条件 | 数据库端口（SQLite 不需要） |
| `name` | string | - | 是 | 数据库名称（SQLite 为文件路径） |
| `user` | string | - | 条件 | 数据库用户名（SQLite 不需要） |
| `pass` | string | - | 条件 | 数据库密码（SQLite 不需要） |
| `charset` | string | `utf8mb4` | 是 | 字符集 |
| `max_open_conns` | int | `1000` | 是 | 最大连接数 |
| `max_idle_conns` | int | `500` | 是 | 最大空闲连接数 |
| `conn_max_lifetime` | int | `1800` | 是 | 连接最大生命周期（秒） |
| `conn_max_idletime` | int | `300` | 是 | 空闲连接回收时间（秒） |
| `rw_split` | bool | `false` | 是 | 是否启用读写分离 |
| `writes` | []object | `[]` | 是 | 写库配置（多主库） |
| `reads` | []object | `[]` | 是 | 读库配置（从库） |

#### 完整示例（MySQL）

```yaml
database:
  driver: mysql
  host: ${DB_HOST}              # 从环境变量读取
  port: ${DB_PORT:3306}         # 默认 3306
  name: ${DB_NAME}
  user: ${DB_USER}
  pass: ${DB_PASSWORD}
  charset: utf8mb4
  
  # 连接池配置
  max_open_conns: 1000          # 最大连接数
  max_idle_conns: 500           # 空闲连接数
  conn_max_lifetime: 1800       # 30 分钟
  conn_max_idletime: 300        # 5 分钟
  
  # 读写分离
  rw_split: false
  writes: []
  reads: []
```

#### SQLite 示例

```yaml
database:
  driver: sqlite3
  name: ./data/app.db          # 数据库文件路径
  max_open_conns: 10
  max_idle_conns: 5
  conn_max_lifetime: 3600
  conn_max_idletime: 600
```

#### PostgreSQL 示例

```yaml
database:
  driver: postgres
  host: ${DB_HOST}
  port: ${DB_PORT:5432}
  name: ${DB_NAME}
  user: ${DB_USER}
  pass: ${DB_PASSWORD}
  charset: utf8
  
  max_open_conns: 500
  max_idle_conns: 200
  conn_max_lifetime: 3600
  conn_max_idletime: 600
```

### 读写分离配置

#### 配置多个写库（写入负载均衡）

```yaml
database:
  driver: mysql
  host: ${DB_HOST}
  port: 3306
  name: ${DB_NAME}
  user: ${DB_USER}
  pass: ${DB_PASSWORD}
  charset: utf8mb4
  
  rw_split: true
  
  # 多个写库（轮询）
  writes:
    - host: 192.168.1.10
      port: 3306
      user: writer1
      pass: write-pass-1
      charset: utf8mb4
    - host: 192.168.1.11
      port: 3306
      user: writer2
      pass: write-pass-2
      charset: utf8mb4
  
  # 多个读库（轮询）
  reads:
    - host: 192.168.1.20
      port: 3306
      user: reader1
      pass: read-pass-1
      charset: utf8mb4
    - host: 192.168.1.21
      port: 3306
      user: reader2
      pass: read-pass-2
      charset: utf8mb4
    - host: 192.168.1.22
      port: 3306
      user: reader3
      pass: read-pass-3
      charset: utf8mb4
```

### 连接池优化建议

#### 单机部署（10 万并发）

```yaml
max_open_conns: 1000    # 1000 个连接
max_idle_conns: 500     # 50% 空闲
conn_max_lifetime: 1800 # 30 分钟
conn_max_idletime: 300  # 5 分钟
```

#### 分布式部署（每节点）

```yaml
max_open_conns: 300     # 300 个连接
max_idle_conns: 150     # 50% 空闲
conn_max_lifetime: 3600 # 1 小时
conn_max_idletime: 600  # 10 分钟
```

## 多数据库配置

### 配置路径
```yaml
databases:
```

### 使用场景

- 不同业务使用独立数据库
- 数据库分库分表
- 日志独立存储
- 多租户数据隔离

### 完整示例

```yaml
databases:
  # 用户中心数据库
  user_center:
    driver: mysql
    host: 192.168.2.10
    port: 3306
    name: user_center
    user: user_admin
    pass: user-pass
    charset: utf8mb4
    max_open_conns: 500
    max_idle_conns: 100
    conn_max_lifetime: 1800
    conn_max_idletime: 300
    rw_split: true
    reads:
      - host: 192.168.2.20
        port: 3306
        user: user_reader
        pass: user-read-pass
        charset: utf8mb4

  # 订单数据库
  order_db:
    driver: mysql
    host: 192.168.3.10
    port: 3306
    name: order_db
    user: order_admin
    pass: order-pass
    charset: utf8mb4
    max_open_conns: 800
    max_idle_conns: 200
    conn_max_lifetime: 1800
    conn_max_idletime: 300
    rw_split: true
    writes:
      - host: 192.168.3.11
        port: 3306
        user: order_writer
        pass: order-write-pass
        charset: utf8mb4
    reads:
      - host: 192.168.3.20
        port: 3306
        user: order_reader
        pass: order-read-pass
        charset: utf8mb4

  # 日志数据库（PostgreSQL）
  log_db:
    driver: postgres
    host: 192.168.4.10
    port: 5432
    name: log_db
    user: log_admin
    pass: log-pass
    charset: utf8
    max_open_conns: 200
    max_idle_conns: 50
    conn_max_lifetime: 3600
    conn_max_idletime: 600
    rw_split: false
    reads: []

  # 配置数据库（SQLite）
  config_db:
    driver: sqlite3
    name: ./data/config.db
    max_open_conns: 10
    max_idle_conns: 5
    conn_max_lifetime: 3600
    conn_max_idletime: 600
```

### 代码中使用

```go
// 使用默认数据库
db := database.GetDB()

// 使用指定数据库
userDB := database.GetDB("user_center")
orderDB := database.GetDB("order_db")
logDB := database.GetDB("log_db")
```

## 配置文件加载顺序

框架按以下顺序加载配置文件：

1. `config.yaml` - 基础配置
2. `config.{mode}.yaml` - 环境配置（如 `config.dev.yaml`）
3. `config.local.yaml` - 本地配置（优先级最高）
4. 环境变量 - 覆盖配置文件

### 示例

```bash
# 开发环境
mode: dev

# 加载顺序：
# 1. config.yaml
# 2. config.dev.yaml
# 3. config.local.yaml（如果存在）
# 4. 环境变量
```

## 最佳实践

### 1. 使用环境变量

```yaml
# ✅ 推荐：敏感信息使用环境变量
database:
  host: ${DB_HOST}
  user: ${DB_USER}
  pass: ${DB_PASSWORD}

# ❌ 不推荐：明文密码
database:
  host: localhost
  user: root
  pass: root123
```

### 2. 环境分离

```yaml
# ✅ 推荐：不同环境使用不同配置
# config.dev.yaml - 开发环境
# config.test.yaml - 测试环境
# config.prod.yaml - 生产环境
```

### 3. 连接池优化

```yaml
# ✅ 推荐：根据部署方式调整
# 单机部署
max_open_conns: 1000
max_idle_conns: 500

# 分布式部署（每节点）
max_open_conns: 300
max_idle_conns: 150
```

### 4. 安全配置

```yaml
# ✅ 推荐：生产环境必须配置
security:
  enable_security_middleware: true
  enable_cors_domain_check: true
  allowed_domains:
    - "https://www.example.com"
  enable_rate_limit: true
  rate_limit: 100
```
