# Vigo 极速上手指南

Vigo 是一款高性能、企业级的 Go 语言开发框架，提供完整的 MVC 架构、ThinkPHP/Laravel 风格 ORM、微服务支持等能力。

> "大道至简，开发由我"

---

## 1. 核心优势

### 🚀 极致性能

- **10 万 + 并发**: 经过压力测试验证，单机支持 10 万 + 并发连接
- **QPS 5 万+**: 优化的路由匹配和上下文复用机制
- **连接池优化**: 数据库、Redis 连接池深度优化
- **内存复用**: `sync.Pool` 复用上下文，减少 GC 压力

### 🏗️ 完整架构

- **MVC 分层**: Controller、Model、View 清晰分层
- **ThinkPHP/Laravel 风格 ORM**: 熟悉的链式操作，降低学习成本
- **依赖注入**: IoC 容器支持，代码更灵活
- **中间件链**: 灵活的请求/响应中间件机制

### 🔒 企业级安全

- **XSS 防护**: 自动转义危险字符
- **SQL 注入防护**: 预处理语句，拦截 99% 注入攻击
- **CSRF 防护**: Token 验证机制
- **速率限制**: 防 DDoS 攻击
- **JWT 认证**: 无状态身份验证

### 🌐 微服务就绪

- **gRPC**: 高性能 RPC 框架
- **Nacos**: 服务注册与发现、配置中心
- **RabbitMQ**: 消息队列支持
- **API 网关**: 统一入口管理

### 📊 监控与管理

- **Web 管理界面**: 可视化管理工具
- **压力测试**: 内置压测工具
- **性能监控**: CPU、内存、磁盘实时监控
- **日志系统**: 分级日志、按日期分割

## 3. 架构模式

### 3.1 单体模式 (Monolith)

适合初创项目和中小型应用。将所有功能打包在一个应用中。

**特点**:

- 开发简单，部署方便
- 所有数据库和视图启用
- 在 `route/route.go` 中注册所有业务路由

**配置文件**:

```yaml
# config.yaml
app:
  name: vigo
  port: 8080
  debug: true

database:
  host: 127.0.0.1
  name: vigo
  # ... 其他配置
```

### 3.2 微服务模式 (Microservices)

适合中大型项目。按业务拆分服务，通过 Nacos 注册发现，gRPC 通信。

**特点**:

- 服务独立部署，可扩展性强
- 通过 Nacos 实现服务注册与发现
- 使用 gRPC 进行服务间通信
- 每个服务有独立的配置文件

**服务提供者模式**:

```go
// 实现 ServiceProvider 接口
type UserServiceProvider struct{}

func (p *UserServiceProvider) Register() {
    // 只加载用户服务所需的模块
    // 如：Redis、RabbitMQ，不加载视图引擎
}
```

## 4. 核心功能演示

### 4.1 路由与控制器

```go
// route/route.go
r := mvc.NewRouter()

// 定义路由
r.GET("/user/info", controller.GetUserInfo)
r.POST("/user/create", controller.CreateUser)
r.PUT("/user/update", controller.UpdateUser)
r.DELETE("/user/delete", controller.DeleteUser)

// 路由组
api := r.Group("/api")
{
    api.GET("/users", controller.ListUsers)
    api.POST("/users", controller.CreateUser)
}
```

### 4.2 数据库操作 (ThinkPHP 风格)

```go
import "vigo/framework/db"

// 查询多个记录
users, err := db.Table("users").
    Field("id", "name", "email").
    Where("status", 1).
    WhereLike("name", "%张%").
    WhereIn("role", []string{"admin", "user"}).
    Order("id DESC").
    Page(1, 10).
    Select()

// 查询单条记录
user, err := db.Table("users").
    Where("id", 1).
    Find()

// 插入数据
id, err := db.Table("users").InsertGetId(map[string]interface{}{
    "name":  "张三",
    "email": "zhangsan@example.com",
    "status": 1,
})

// 更新数据
affected, err := db.Table("users").
    Where("id", 1).
    Update(map[string]interface{}{
        "name": "李四",
    })

// 删除数据
affected, err := db.Table("users").
    Where("id", 1).
    Delete()

// 聚合查询
count, err := db.Table("users").
    Where("status", 1).
    Count()

sum, err := db.Table("orders").
    Where("user_id", 1).
    Sum("amount")
```

### 4.3 事务操作

```go
// 自动事务
err := db.Transaction(func(q *db.Query) error {
    // 插入订单
    _, err := q.Table("orders").Insert(orderData)
    if err != nil {
        return err
    }

    // 扣减库存
    _, err = q.Table("products").
        Where("id", productId).
        Dec("stock", 1)
    if err != nil {
        return err
    }

    return nil
})

if err != nil {
    log.Println("事务失败:", err)
}
```

### 4.4 读写分离

```yaml
# config.yaml
database:
  host: 192.168.1.10 # 主库（写入）
  port: 3306
  name: vigo
  user: root
  pass: password

  # 启用读写分离
  rw_split: true

  # 从库配置（读取）
  reads:
    - host: 192.168.1.20
      port: 3306
    - host: 192.168.1.21
      port: 3306
```

框架会自动将查询路由到从库，写入路由到主库。

### 4.5 Redis 缓存

```go
import "vigo/framework/redis"

client := redis.GetClient()

// 设置缓存
err := client.Set("key", "value", 3600) // 1 小时过期

// 获取缓存
val, err := client.Get("key")

// 删除缓存
err := client.Del("key")

// 哈希操作
err := client.HSet("user:1", "name", "张三")
name, err := client.HGet("user:1", "name")
```

### 4.6 消息队列

```go
import "vigo/framework/rabbitmq"

mq := rabbitmq.GetClient()

// 发布消息
err := mq.PublishToQueue("order.created", []byte(`{"order_id": 123}`))

// 消费消息
mq.Consume("order.created", func(body []byte) {
    log.Println("收到订单消息:", string(body))
    // 处理业务逻辑
})
```

## 5. 性能优化

### 5.1 路由匹配

使用 **Map + Prefix** 匹配算法，实现毫秒级精确匹配。

```go
// 精确匹配，无需正则
r.GET("/user/info", controller.GetInfo)
r.GET("/user/profile", controller.GetProfile)
```

### 5.2 内存复用

框架自动处理上下文池化，开发者无需介入。

```go
// 框架内部实现
c := r.pool.Get().(*Context)
defer r.pool.Put(c)
```

### 5.3 连接池配置

```yaml
# 数据库连接池
database:
  max_open_conns: 200 # 最大连接数
  max_idle_conns: 100 # 最大空闲连接
  conn_max_lifetime: 3600 # 连接生命周期（秒）

# Redis 连接池
redis:
  pool_size: 200 # 连接池大小
  min_idle_conns: 50 # 最小空闲连接
  max_idle_conns: 100 # 最大空闲连接
```

## 6. 安全最佳实践

### 6.1 输入验证

使用内置验证器进行输入验证：

```go
import "vigo/framework/mvc"

func CreateUser(c *mvc.Context) {
    v := mvc.NewValidator(c)

    // 必填验证
    username := v.Required("username", "用户名必填")

    // 邮箱验证
    email := v.Email("email", "邮箱格式不正确")

    // 手机号验证
    phone := v.Mobile("phone", "手机号格式不正确")

    // 长度验证
    password := v.Length("password", 6, 20, "密码长度 6-20 位")

    if v.HasError() {
        c.Error(400, v.GetError())
        return
    }

    // 业务逻辑
}
```

### 6.2 SQL 注入防护

框架默认开启 SQL 注入检测中间件：

```go
// 自动拦截危险输入
// 使用参数化查询，避免拼接 SQL
db.Table("users").Where("name", userInput) // ✅ 安全
// db.Query("SELECT * FROM users WHERE name = '" + userInput + "'") // ❌ 危险
```

### 6.3 XSS 防护

```go
// 自动转义输出
c.HTML(200, "index.html", map[string]interface{}{
    "content": "<script>alert('xss')</script>", // 自动转义
})
```

### 6.4 JWT 认证

```go
import "vigo/framework/jwt"

// 生成 Token
token, err := jwt.GenerateToken(map[string]interface{}{
    "user_id": 1,
    "username": "admin",
})

// 验证 Token
claims, err := jwt.ParseToken(token)
userID := claims["user_id"]
```
