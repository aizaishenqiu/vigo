**版本**: 1.0  
**更新时间**: 2026-03-02

## 一、错误代码体系概述

Vigo 框架采用统一的错误代码体系，便于错误追踪、日志分析和问题定位。

### 错误代码结构

```
错误代码 = 错误类型前缀 + 模块编号 + 具体错误编号
```

**示例**: `USER_001` 表示用户模块的第 1 个错误

---

## 二、错误类型前缀

| 前缀 | 类型 | 说明 | HTTP 状态码 |
|------|------|------|-------------|
| `COMMON` | 通用错误 | 通用业务错误 | 400 |
| `VALIDATION` | 验证错误 | 参数验证失败 | 422 |
| `AUTH` | 认证错误 | 身份认证失败 | 401 |
| `FORBIDDEN` | 授权错误 | 权限不足 | 403 |
| `NOT_FOUND` | 未找到 | 资源不存在 | 404 |
| `INTERNAL` | 内部错误 | 服务器内部错误 | 500 |
| `DATABASE` | 数据库错误 | 数据库操作失败 | 500 |
| `CACHE` | 缓存错误 | 缓存操作失败 | 500 |
| `FILE` | 文件错误 | 文件操作失败 | 500 |
| `NETWORK` | 网络错误 | 网络请求失败 | 502 |

## 四、错误代码使用示例

### 4.1 创建错误

```go
import "vigo/framework/errors"

// 创建通用错误
err := errors.NewError("COMMON_001", "请求失败")

// 创建带详细信息的错误
err := errors.NewError("VALIDATION_001", "参数不能为空").
    WithDetails("field", "username")

// 创建带堆栈跟踪的错误
err := errors.InternalError("服务器内部错误").
    WithStack()
```

### 4.2 使用预定义错误

```go
// 用户相关错误
err := errors.UserNotFound("用户不存在")
err := errors.UserDisabled("账号被禁用")

// 验证相关错误
err := errors.RequiredField("username", "用户名")
err := errors.InvalidEmail("邮箱格式不正确")
err := errors.InvalidMobile("手机号格式不正确")

// 认证相关错误
err := errors.Unauthorized("未登录")
err := errors.TokenExpired("Token 已过期")

// 数据库相关错误
err := errors.DatabaseError("数据库连接失败")
err := errors.QueryFailed("查询失败")
```

### 4.3 错误处理

```go
// 在控制器中处理错误
func (c *UserController) Show(ctx *mvc.Context) {
    user, err := userService.FindByID(ctx.Param("id"))
    if err != nil {
        if appErr, ok := err.(*errors.AppError); ok {
            ctx.Error(appErr.Code, appErr.Message)
        } else {
            ctx.Error(500, "服务器内部错误")
        }
        return
    }
    
    ctx.Json(200, user)
}

// 在服务层处理错误
func (s *UserService) FindByID(id string) (*User, error) {
    if id == "" {
        return nil, errors.RequiredField("id", "用户 ID")
    }
    
    user, err := s.repo.FindByID(id)
    if err != nil {
        if err == sql.ErrNoRows {
            return nil, errors.UserNotFound("用户不存在")
        }
        return nil, errors.DatabaseError("查询失败：" + err.Error())
    }
    
    return user, nil
}
```

### 4.4 错误日志

```go
// 记录错误日志
appErr := errors.InternalError("处理失败").
    WithDetails("user_id", userID).
    WithStack()

log.Printf("[ERROR] %v", appErr)
// 输出：[ERROR] INTERNAL_001: 服务器内部错误 | user_id: 123 | stack: ...
```

## 五、错误代码最佳实践

### 5.1 错误分类原则

1. **按模块分类**: 每个业务模块使用独立的前缀
2. **按严重程度**: 区分客户端错误和服务器错误
3. **唯一性**: 每个错误代码对应唯一的错误场景

### 5.2 错误信息规范

1. **简洁明了**: 错误信息应简洁明了，便于理解
2. **避免敏感信息**: 错误信息不应包含敏感数据
3. **提供解决方案**: 尽可能提供解决方案提示

### 5.3 错误处理原则

1. **及时捕获**: 及时捕获并处理错误
2. **统一处理**: 使用统一的错误处理机制
3. **记录日志**: 记录详细错误信息用于排查
4. **用户友好**: 向用户展示友好的错误信息

### 5.4 错误监控

1. **错误统计**: 统计各类错误发生频率
2. **告警机制**: 对严重错误设置告警
3. **趋势分析**: 分析错误趋势，提前预防

## 六、错误代码扩展

### 6.1 添加新错误代码

1. 确定错误类型前缀
2. 分配错误编号（按顺序递增）
3. 定义错误信息
4. 添加到此文档
5. 更新错误工厂函数

### 6.2 自定义错误代码

可以在项目中定义自己的错误代码：

```go
// 定义业务错误代码
const (
    ErrOrderNotFound = "ORDER_001"
    ErrOrderPaid = "ORDER_002"
    ErrOrderCancelled = "ORDER_003"
)

// 创建自定义错误
func OrderNotFound() *errors.AppError {
    return errors.NewError(ErrOrderNotFound, "订单不存在")
}
```

## 八、常见问题

### Q1: 如何选择错误代码？

**A**: 根据错误类型和场景选择：
- 参数验证失败 → VALIDATION
- 用户未登录 → AUTH
- 数据不存在 → NOT_FOUND
- 服务器异常 → INTERNAL

### Q2: 错误代码需要国际化吗？

**A**: 建议支持国际化：
- 错误代码保持不变
- 错误信息根据语言环境显示

### Q3: 如何处理未知错误？

**A**: 使用通用错误代码：
```go
err := errors.InternalError("未知错误：" + err.Error())
```
