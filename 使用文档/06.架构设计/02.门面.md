# 门面 (Facade)

门面（Facade）为容器中的服务提供了一个简洁的、全局可访问的接口。在框架设计中，门面模式是实现“极致开发体验”的关键。

## 门面模式的意义

1.  **代码简洁性**：不需要在每个函数或方法中传递长长的参数或手动从容器获取实例。
2.  **开发效率**：模仿 `Facade::method()` 的静态调用感，符合 PHP 开发者的直觉。
3.  **解耦与抽象**：门面内部封装了从容器获取实例的逻辑，业务代码不需要知道服务是如何创建和定位的。
4.  **避免循环引用**：在 Go 中，门面可以作为第三方中转，帮助打破包与包之间的循环引用。

## 使用门面

框架在 `framework/facade` 包中定义了常用门面。

### Redis 门面

```go
import "vigo/framework/facade"

// 直接获取 Redis 实例并调用方法
res, err := facade.Redis().Get(ctx, "key").Result()
```

### Config 门面

```go
import "vigo/framework/facade"

// 获取全局配置对象
config := facade.Config()
```

### App 门面

```go
import "vigo/framework/facade"

// 获取应用核心容器
container := facade.App()
instance := container.Make("some_service")
```

## 核心实现

在 Go 中，由于没有静态类，我们通过**函数包装**模拟门面：

```go
// framework/facade/facade.go
func Redis() *redis.Client {
    // 从容器中获取单例
	instance := container.App().Make("redis")
	if instance == nil {
		return nil
	}
	return instance.(*redis.Client)
}
```
