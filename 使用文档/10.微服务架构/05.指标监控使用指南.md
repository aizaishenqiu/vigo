# Vigo 框架 - 指标监控使用指南

## 1. 概述

Vigo 框架集成了 Prometheus 指标监控系统，提供全面的性能指标收集、可视化展示和告警功能。

### 特性

- ✅ HTTP 请求指标（QPS、延迟、状态码）
- ✅ 缓存指标（命中率、内存使用）
- ✅ 数据库指标（连接池、查询延迟）
- ✅ RPC 指标（调用次数、错误率）
- ✅ 系统指标（CPU、内存、Goroutine）
- ✅ Grafana 可视化集成
- ✅ 告警规则配置

## 2. 快速开始

### 2.1 安装依赖

```bash
go get github.com/prometheus/client_golang/prometheus
go get github.com/prometheus/client_golang/prometheus/promhttp
```

### 2.2 初始化指标系统

```go
package main

import (
    "log"
    "net/http"
    "time"
    
    "github.com/yourorg/vigo/framework/metrics"
)

func main() {
    // 初始化指标系统
    err := metrics.Init("user-service")
    if err != nil {
        log.Fatalf("Failed to initialize metrics: %v", err)
    }

    // 启动指标服务器（默认端口 9090）
    go func() {
        if err := metrics.StartServer(); err != nil {
            log.Printf("Metrics server error: %v", err)
        }
    }()

    // 等待指标服务器启动
    time.Sleep(100 * time.Millisecond)

    // 启动应用
    app.Run()
}
```

### 2.3 配置说明

```yaml
# config.yaml
metrics:
  enabled: true
  service_name: user-service
  port: 9090                    # 指标暴露端口
  path: /metrics                # 指标路径
  read_timeout: 10s             # 读取超时
  write_timeout: 10s            # 写入超时
  idle_timeout: 30s             # 空闲超时
```

## 3. 指标类型

### 3.1 HTTP 请求指标

#### 自动收集的指标

```prometheus
# HTTP 请求总数
http_requests_total{method="GET", path="/api/users", status="200"}

# HTTP 请求延迟（秒）
http_request_duration_seconds{method="GET", path="/api/users"}

# HTTP 请求体大小
http_request_size_bytes{method="POST", path="/api/users"}

# HTTP 响应体大小
http_response_size_bytes{method="GET", path="/api/users"}

# 活跃请求数
http_requests_in_progress{method="GET", path="/api/users"}
```

#### 使用示例

```go
import (
    "github.com/yourorg/vigo/framework/metrics"
    "github.com/yourorg/vigo/framework/mvc"
)

func (c *UserController) GetUser(ctx *mvc.Context) error {
    // 框架自动记录指标，无需手动处理
    userID := ctx.Param("id")
    user, err := c.userService.GetUser(ctx.Request.Context(), userID)
    if err != nil {
        return ctx.Status(500).JSON(map[string]string{"error": err.Error()})
    }
    return ctx.JSON(user)
}
```

### 3.2 缓存指标

```prometheus
# 缓存命中次数
cache_hits_total{cache_type="memory"}
cache_hits_total{cache_type="redis"}

# 缓存未命中次数
cache_misses_total{cache_type="memory"}
cache_misses_total{cache_type="redis"}

# 缓存命中率
cache_hit_ratio{cache_type="memory"}

# 缓存键数量
cache_keys_count{cache_type="memory"}

# 缓存内存使用（字节）
cache_memory_bytes{cache_type="memory"}
```

#### 使用示例

```go
import (
    "github.com/yourorg/vigo/framework/cache"
    "github.com/yourorg/vigo/framework/metrics"
)

func (s *UserService) GetUserWithCache(ctx context.Context, userID string) (*User, error) {
    cacheKey := "user:" + userID
    
    // 尝试从缓存获取
    if user, err := s.cache.Get(ctx, cacheKey); err == nil {
        // 记录缓存命中
        metrics.RecordCacheHit("memory")
        return user, nil
    }
    
    // 记录缓存未命中
    metrics.RecordCacheMiss("memory")
    
    // 从数据库查询
    user, err := s.db.QueryUser(ctx, userID)
    if err != nil {
        return nil, err
    }
    
    // 写入缓存
    s.cache.Set(ctx, cacheKey, user, 5*time.Minute)
    
    return user, nil
}
```

### 3.3 数据库指标

```prometheus
# 数据库连接池状态
db_pool_connections{db_name="master", state="open"}
db_pool_connections{db_name="master", state="in_use"}
db_pool_connections{db_name="master", state="idle"}

# 数据库查询延迟
db_query_duration_seconds{db_name="master", operation="SELECT"}

# 数据库查询总数
db_query_total{db_name="master", operation="SELECT"}

# 数据库错误总数
db_errors_total{db_name="master", error_type="timeout"}
```

#### 使用示例

```go
import (
    "github.com/yourorg/vigo/framework/db"
    "github.com/yourorg/vigo/framework/metrics"
)

func (r *UserRepository) QueryUsers(ctx context.Context) ([]*User, error) {
    start := time.Now()
    
    query := "SELECT * FROM users"
    rows, err := r.db.QueryContext(ctx, query)
    duration := time.Since(start).Seconds()
    
    if err != nil {
        // 记录数据库错误
        metrics.RecordDBError("master", "query")
        return nil, err
    }
    defer rows.Close()
    
    // 记录查询指标
    metrics.RecordDBQuery("master", "SELECT", duration)
    
    // 处理结果
    var users []*User
    for rows.Next() {
        var user User
        if err := rows.Scan(&user.ID, &user.Name, &user.Email); err != nil {
            return nil, err
        }
        users = append(users, &user)
    }
    
    return users, nil
}
```

### 3.4 RPC 指标

```prometheus
# RPC 调用总数

# RPC 错误率
rpc_error_rate{service="order-service"}

# 活跃 RPC 连接数
rpc_active_connections{service="order-service"}
```

#### 使用示例

```go
import (
    "github.com/yourorg/vigo/framework/metrics"
)

func (c *RPCClient) CallOrderService(ctx context.Context, req *CreateOrderRequest) (*CreateOrderResponse, error) {
    start := time.Now()
    
    resp, err := c.client.CreateOrder(ctx, req)
    duration := time.Since(start).Seconds()
    
    if err != nil {
        // 记录 RPC 错误
        metrics.RecordRPCCall("order-service", "CreateOrder", "error", duration)
        return nil, err
    }
    
    // 记录成功调用
    metrics.RecordRPCCall("order-service", "CreateOrder", "success", duration)
    
    return resp, nil
}
```

## 4. 自定义指标

### 4.1 创建 Counter

```go
import (
    "github.com/prometheus/client_golang/prometheus"
    "github.com/yourorg/vigo/framework/metrics"
)

// 注册自定义指标
var (
    orderCounter = prometheus.NewCounterVec(
        prometheus.CounterOpts{
            Name:        "orders_total",
            Help:        "Total number of orders",
            ConstLabels: prometheus.Labels{"service": "order-service"},
        },
        []string{"order_type", "status"},
    )
)

func init() {
    // 注册到 Prometheus
    metrics.GetRegistry().MustRegister(orderCounter)
}

// 使用
func CreateOrder(orderType string) error {
    // ...
    orderCounter.WithLabelValues(orderType, "created").Inc()
    return nil
}
```

### 4.2 创建 Gauge

```go
var (
    inventoryGauge = prometheus.NewGaugeVec(
        prometheus.GaugeOpts{
            Name:        "inventory_count",
            Help:        "Current inventory count",
            ConstLabels: prometheus.Labels{"service": "inventory-service"},
        },
        []string{"product_id", "warehouse"},
    )
)

func init() {
    metrics.GetRegistry().MustRegister(inventoryGauge)
}

// 使用
func UpdateInventory(productID, warehouse string, count int) {
    inventoryGauge.WithLabelValues(productID, warehouse).Set(float64(count))
}
```

### 4.3 创建 Histogram

```go
var (
    processingTime = prometheus.NewHistogramVec(
        prometheus.HistogramOpts{
            Name:        "processing_time_seconds",
            Help:        "Processing time distribution",
            ConstLabels: prometheus.Labels{"service": "processing-service"},
            Buckets:     prometheus.DefBuckets,
        },
        []string{"task_type"},
    )
)

func init() {
    metrics.GetRegistry().MustRegister(processingTime)
}

// 使用
func ProcessTask(taskType string) error {
    start := time.Now()
    defer func() {
        duration := time.Since(start).Seconds()
        processingTime.WithLabelValues(taskType).Observe(duration)
    }()
    
    // 处理任务
    return nil
}
```

### 4.4 创建 Summary

```go
var (
    responseSize = prometheus.NewSummaryVec(
        prometheus.SummaryOpts{
            Name:        "response_size_bytes",
            Help:        "Response size distribution",
            ConstLabels: prometheus.Labels{"service": "api-service"},
            Objectives:  map[float64]float64{0.5: 0.05, 0.9: 0.01, 0.99: 0.001},
        },
        []string{"endpoint"},
    )
)

func init() {
    metrics.GetRegistry().MustRegister(responseSize)
}

// 使用
func SendResponse(endpoint string, data []byte) {
    responseSize.WithLabelValues(endpoint).Observe(float64(len(data)))
}
```

### 5.4 查询指标

使用 PromQL 查询指标：

```promql
# 查询 QPS
rate(http_requests_total[5m])

# 查询平均延迟
rate(http_request_duration_seconds_sum[5m]) / rate(http_request_duration_seconds_count[5m])

# 查询错误率
rate(http_requests_total{status=~"5.."}[5m]) / rate(http_requests_total[5m])

# 查询缓存命中率
rate(cache_hits_total[5m]) / (rate(cache_hits_total[5m]) + rate(cache_misses_total[5m]))
```

## 7. 告警配置

### 7.1 告警规则

```yaml
# alert_rules.yml
groups:
  - name: user-service-alerts
    interval: 30s
    rules:
      # 高错误率告警
      - alert: HighErrorRate
        expr: rate(http_requests_total{service="user-service",status=~"5.."}[5m]) / rate(http_requests_total{service="user-service"}[5m]) > 0.05
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "高错误率：{{ $labels.instance }}"
          description: "错误率：{{ $value | humanizePercentage }}"

      # 高延迟告警
      - alert: HighLatency
        expr: histogram_quantile(0.99, rate(http_request_duration_seconds_bucket{service="user-service"}[5m])) > 1
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "高延迟：{{ $labels.instance }}"
          description: "P99 延迟：{{ $value }}s"

      # 服务宕机告警
      - alert: ServiceDown
        expr: up{job="user-service"} == 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "服务宕机：{{ $labels.instance }}"
          description: "{{ $labels.instance }} 已宕机超过 1 分钟"

      # 缓存命中率低告警
      - alert: LowCacheHitRate
        expr: rate(cache_hits_total{cache_type="memory"}[5m]) / (rate(cache_hits_total{cache_type="memory"}[5m]) + rate(cache_misses_total{cache_type="memory"}[5m])) < 0.8
        for: 10m
        labels:
          severity: warning
        annotations:
          summary: "缓存命中率低：{{ $labels.instance }}"
          description: "缓存命中率：{{ $value | humanizePercentage }}"

      # 数据库连接池耗尽告警
      - alert: DBConnectionPoolExhausted
        expr: db_pool_connections{db_name="master",state="in_use"} / db_pool_connections{db_name="master",state="open"} > 0.9
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "数据库连接池耗尽：{{ $labels.instance }}"
          description: "连接池使用率：{{ $value | humanizePercentage }}"
```

## 8. 最佳实践

### 8.1 指标命名规范

```promql
# 使用下划线分隔

httpRequestsTotal        # 错误

# 使用有意义的名称
db_query_duration_seconds    # 正确
db_time                      # 错误

# 包含单位
cache_memory_bytes           # 正确
cache_memory                 # 错误
```

### 8.2 标签使用规范

```promql
# 使用低基数字标签
http_requests_total{method="GET", path="/api/users"}  # 正确
http_requests_total{user_id="123"}                    # 错误（高基数）

# 标签值有限集合
http_requests_total{status="200"}   # 正确
http_requests_total{status="123"}   # 正确
http_requests_total{status="abc"}   # 错误
```

### 8.3 性能优化

```go
// 1. 预注册指标
var metrics = prometheus.NewCounterVec(...)
func init() {
    prometheus.MustRegister(metrics)
}

// 2. 使用 WithLabelValues 缓存
labelValues := metrics.WithLabelValues("label1", "label2")
labelValues.Inc()  // 直接使用，避免重复查找

// 3. 避免高基数字标签
// 错误示例：user_id 作为标签
metrics.WithLabelValues(userID)  // userID 可能有百万级

// 正确示例：使用 Counter
totalUsers.Inc()
```

## 9. 常见问题

### Q1: 指标没有暴露？

**检查项：**
1. 指标服务器是否启动
2. 端口是否正确配置
3. 防火墙是否阻止访问
4. 访问 /metrics 路径查看指标

### Q2: 指标数据不准确？

**排查方法：**
1. 检查指标注册是否正确
2. 验证标签值是否正确
3. 确认采集间隔是否合理
4. 查看 Prometheus 日志

### Q3: 内存占用过高？

**优化建议：**
1. 减少高基数字标签
2. 降低指标采集频率
3. 使用合理的采样率
4. 定期清理无用指标

### Q4: 告警误报？

**优化方法：**
1. 调整告警阈值
2. 增加持续时间（for）
3. 使用滑动窗口
4. 添加告警抑制规则
