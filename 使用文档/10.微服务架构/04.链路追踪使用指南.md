# Vigo 框架 - 链路追踪使用指南

## 1. 概述

Vigo 框架集成了 OpenTelemetry 链路追踪系统，支持分布式追踪、性能分析和故障诊断。

### 特性

- ✅ 支持 OpenTelemetry 标准
- ✅ Jaeger 后端集成
- ✅ 自动上下文传播
- ✅ 自定义 Span 创建
- ✅ 高性能异步导出
- ✅ 采样率配置

## 2. 快速开始

### 2.1 安装依赖

```bash
go get go.opentelemetry.io/otel
go get go.opentelemetry.io/otel/exporters/jaeger
go get go.opentelemetry.io/otel/sdk/trace
```

### 2.2 初始化链路追踪

```go
package main

import (
    "log"
    "os"
    "os/signal"
    "syscall"

    "github.com/yourorg/vigo/framework/trace"
)

func main() {
    // 初始化链路追踪
    tp, err := trace.Init(&trace.Config{
        ServiceName:    "user-service",
        Endpoint:       "http://localhost:14268/api/traces",
        SampleRate:     1.0,
        MaxQueueSize:   10000,
        BatchTimeout:   5000,
        MaxExportBatch: 512,
    })
    if err != nil {
        log.Fatalf("Failed to initialize trace: %v", err)
    }
    defer func() {
        if err := tp.Shutdown(context.Background()); err != nil {
            log.Printf("Error shutting down tracer provider: %v", err)
        }
    }()

    // 启动应用
    app.Run()
}
```

### 2.3 配置说明

```yaml
# config.yaml
trace:
  enabled: true
  service_name: user-service
  jaeger_endpoint: http://localhost:14268/api/traces
  sample_rate: 1.0 # 采样率 (0.0-1.0)
  max_queue_size: 10000 # 最大队列大小
  batch_timeout: 5000 # 批量导出超时 (毫秒)
  max_export_batch: 512 # 最大导出批次大小
```

## 3. 使用示例

### 3.1 自动追踪 HTTP 请求

Vigo 框架自动为所有 HTTP 请求创建 Span：

```go
// 无需手动创建，框架自动处理
func (c *UserController) GetUser(ctx *mvc.Context) error {
    // 这个函数已经在 Span 上下文中
    userID := ctx.Param("id")
    user, err := c.userService.GetUser(ctx.Request.Context(), userID)
    if err != nil {
        return err
    }
    return ctx.JSON(user)
}
```

### 3.2 手动创建 Span

```go
import (
    "github.com/yourorg/vigo/framework/trace"
    "go.opentelemetry.io/otel/attribute"
    "go.opentelemetry.io/otel/trace"
)

func (s *UserService) GetUser(ctx context.Context, userID string) (*User, error) {
    // 创建自定义 Span
    ctx, span := trace.StartSpan(ctx, "UserService.GetUser",
        trace.WithAttributes(
            attribute.String("user.id", userID),
        ),
    )
    defer span.End()

    // 业务逻辑
    user, err := s.db.QueryUser(ctx, userID)
    if err != nil {
        span.RecordError(err)
        return nil, err
    }

    return user, nil
}
```

### 3.3 数据库操作追踪

```go
func (r *UserRepository) QueryUser(ctx context.Context, userID string) (*User, error) {
    ctx, span := trace.StartSpan(ctx, "DB.QueryUser",
        trace.WithAttributes(
            attribute.String("db.table", "users"),
            attribute.String("db.operation", "SELECT"),
        ),
    )
    defer span.End()

    query := "SELECT * FROM users WHERE id = ?"
    row := r.db.QueryRowContext(ctx, query, userID)

    var user User
    err := row.Scan(&user.ID, &user.Name, &user.Email)
    if err != nil {
        span.RecordError(err)
        return nil, err
    }

    return &user, nil
}
```

### 3.4 RPC 调用追踪

```go
func (c *RPCClient) CallRemoteService(ctx context.Context, req *Request) (*Response, error) {
    ctx, span := trace.StartSpan(ctx, "RPC.CallRemoteService",
        trace.WithAttributes(
            attribute.String("rpc.service", "order-service"),
            attribute.String("rpc.method", "CreateOrder"),
        ),
    )
    defer span.End()

    // 创建请求
    httpReq, err := http.NewRequestWithContext(ctx, "POST", url, body)
    if err != nil {
        span.RecordError(err)
        return nil, err
    }

    // 注入追踪上下文
    trace.Inject(ctx, httpReq.Header)

    // 发送请求
    resp, err := c.client.Do(httpReq)
    if err != nil {
        span.RecordError(err)
        return nil, err
    }

    return resp, nil
}
```

### 3.5 异步任务追踪

```go
func (s *OrderService) ProcessOrderAsync(ctx context.Context, orderID string) {
    // 创建后台任务 Span
    ctx, span := trace.StartSpan(ctx, "OrderService.ProcessOrderAsync",
        trace.WithAttributes(
            attribute.String("order.id", orderID),
        ),
    )
    defer span.End()

    go func() {
        // 处理订单
        if err := s.process(orderID); err != nil {
            span.RecordError(err)
        }
    }()
}
```

## 4. 上下文传播

### 4.1 HTTP 请求间传播

```go
// 服务 A - 发起请求
func callServiceB(ctx context.Context) error {
    req, _ := http.NewRequest("GET", "http://service-b/api", nil)

    // 注入追踪上下文到 HTTP Header
    trace.Inject(ctx, req.Header)

    resp, err := http.DefaultClient.Do(req)
    // ...
}

// 服务 B - 接收请求
func handleRequest(w http.ResponseWriter, r *http.Request) {
    // 从 HTTP Header 提取追踪上下文
    ctx := trace.Extract(r.Context(), r.Header)

    // 使用包含追踪信息的 context
    processRequest(ctx)
}
```

### 4.2 gRPC 调用传播

```go
import (
    "google.golang.org/grpc/metadata"
)

// 客户端 - 发送
func callGRPC(ctx context.Context) error {
    // 创建 metadata
    md := metadata.Pairs()

    // 注入追踪上下文
    trace.InjectToGRPC(ctx, md)

    // 附加 metadata 到 context
    ctx = metadata.NewOutgoingContext(ctx, md)

    // 发起 gRPC 调用
    return client.Method(ctx, req)
}

// 服务端 - 接收
func handleGRPC(ctx context.Context) error {
    // 从 metadata 提取追踪上下文
    md, ok := metadata.FromIncomingContext(ctx)
    if ok {
        ctx = trace.ExtractFromGRPC(ctx, md)
    }

    // 处理请求
    return process(ctx)
}
```

## 5. 高级用法

### 5.1 自定义 Span 属性

```go
func ProcessData(ctx context.Context, data []byte) error {
    ctx, span := trace.StartSpan(ctx, "ProcessData",
        trace.WithAttributes(
            attribute.Int("data.size", len(data)),
            attribute.String("data.type", "json"),
            attribute.Bool("data.compressed", true),
        ),
    )
    defer span.End()

    // 处理数据
    return nil
}
```

### 5.2 错误记录

```go
func RiskyOperation(ctx context.Context) error {
    ctx, span := trace.StartSpan(ctx, "RiskyOperation")
    defer span.End()

    err := doRiskyThing()
    if err != nil {
        // 记录错误到 Span
        span.RecordError(err)
        span.SetStatus(codes.Error, err.Error())
        return err
    }

    span.SetStatus(codes.Ok, "Success")
    return nil
}
```

### 5.3 事件标记

```go
func LongRunningTask(ctx context.Context) error {
    ctx, span := trace.StartSpan(ctx, "LongRunningTask")
    defer span.End()

    // 标记任务开始
    span.AddEvent("Task started")

    // 阶段 1
    if err := phase1(ctx); err != nil {
        span.AddEvent("Phase 1 failed",
            trace.WithAttributes(attribute.String("error", err.Error())),
        )
        return err
    }
    span.AddEvent("Phase 1 completed")

    // 阶段 2
    if err := phase2(ctx); err != nil {
        span.AddEvent("Phase 2 failed")
        return err
    }
    span.AddEvent("Phase 2 completed")

    span.AddEvent("Task completed")
    return nil
}
```

### 5.4 采样策略

```go
// 基于采样率的配置
tp, err := trace.Init(&trace.Config{
    ServiceName: "my-service",
    SampleRate:  0.1, // 10% 采样率
})

// 自定义采样器
import (
    "go.opentelemetry.io/otel/sdk/trace"
)

// 始终采样
sampler := trace.AlwaysSample()

// 从不采样
sampler := trace.NeverSample()

// 按固定比例采样
sampler := trace.TraceIDRatioBased(0.1)

// 基于父 Span 采样
sampler := trace.ParentBased(trace.TraceIDRatioBased(0.1))
```

## 7. 最佳实践

### 7.1 Span 命名规范

```go
// 好的命名
trace.StartSpan(ctx, "UserService.GetUser")
trace.StartSpan(ctx, "DB.Query")
trace.StartSpan(ctx, "HTTP.GET")

// 避免的命名
trace.StartSpan(ctx, "process")      // 太模糊
trace.StartSpan(ctx, "step1")        // 无意义
```

### 7.2 属性命名规范

```go
// 使用标准属性名
attribute.String("db.system", "mysql")
attribute.String("http.method", "GET")
attribute.String("http.status_code", "200")

// 自定义属性使用命名空间
attribute.String("app.user.id", "123")
attribute.String("app.order.type", "vip")
```

### 7.3 性能优化

```go
// 1. 合理设置采样率
Config{
    SampleRate: 0.1, // 生产环境降低采样率
}

// 2. 使用批量导出
Config{
    MaxQueueSize:   10000,
    BatchTimeout:   5000,
    MaxExportBatch: 512,
}

// 3. 避免过多 Span
// 只在关键操作创建 Span，避免过度细化
```

### 7.4 错误处理

```go
func SafeOperation(ctx context.Context) (err error) {
    ctx, span := trace.StartSpan(ctx, "SafeOperation")
    defer func() {
        if err != nil {
            span.RecordError(err)
            span.SetStatus(codes.Error, err.Error())
        }
    }()

    return doSomething(ctx)
}
```

## 8. 常见问题

### Q1: 追踪数据没有上报？

**检查项：**

1. Jaeger 是否正常运行
2. Endpoint 地址是否正确
3. 网络是否可达
4. 采样率是否设置为 0

### Q2: 性能影响？

**优化建议：**

1. 生产环境降低采样率（如 0.1）
2. 使用异步批量导出
3. 避免创建过多 Span
4. 减少自定义属性数量

### Q3: 如何调试追踪问题？

**调试方法：**

1. 启用 Debug 日志
2. 检查 Span 是否正确创建
3. 验证上下文是否正确传播
4. 查看导出器错误信息
