### 2. 控制器代码修复

**修复前**（使用标准 Go HTTP）:
```go
// modules/user/module.go
type UserModule struct {
    controller *UserController
    service    *UserService
}

func (m *UserModule) RegisterRoutes(mux *http.ServeMux) {
    mux.HandleFunc("/api/users", m.controller.List)
    mux.HandleFunc("/api/users/", m.controller.Detail)
}
```

**修复后**（使用 Vigo MVC）:
```go
// app/controller/user/user_controller.go
type UserController struct {
    userService *service.UserService
}

// List 用户列表（使用 Vigo Context）
func (c *UserController) List(ctx *mvc.Context) {
    page, _ := strconv.Atoi(ctx.Query("page", "1"))
    pageSize, _ := strconv.Atoi(ctx.Query("page_size", "10"))

    users, total, err := c.userService.GetList(ctx.Request.Context(), page, pageSize)
    if err != nil {
        ctx.Json(http.StatusInternalServerError, map[string]interface{}{
            "code":    500,
            "message": err.Error(),
        })
        return
    }

    ctx.Json(http.StatusOK, map[string]interface{}{
        "code": 200,
        "data": map[string]interface{}{
            "list":  users,
            "total": total,
        },
    })
}
```

**改进点**:
- ✅ 使用 `mvc.Context` 替代标准 `http.ResponseWriter`
- ✅ 使用 `ctx.Query()`、`ctx.Param()` 等 Vigo 方法
- ✅ 使用 `ctx.Json()` 统一响应格式
- ✅ 添加完整的错误处理

### 3. 路由定义修复

**修复前**（使用标准 Go HTTP）:
```go
func (m *UserModule) RegisterRoutes(mux *http.ServeMux) {
    mux.HandleFunc("/api/users", m.controller.List)
    mux.HandleFunc("/api/users/", m.controller.Detail)
}
```

**修复后**（使用 Vigo Router）:
```go
// routes/routes.go
func RegisterRoutes(r *mvc.Router) {
    // 创建用户控制器
    userCtrl := user.NewUserController()

    // 用户路由
    r.GET("/api/users", userCtrl.List)
    r.GET("/api/users/:id", userCtrl.Detail)
    r.POST("/api/auth/login", userCtrl.Login)

    // 需要认证的路由组
    auth := r.Group("/api", middleware.Auth())
    {
        auth.GET("/users/:id", userCtrl.Detail)
        auth.POST("/users", userCtrl.Create)
        auth.PUT("/users/:id", userCtrl.Update)
        auth.DELETE("/users/:id", userCtrl.Delete)
    }
}
```

**改进点**:
- ✅ 使用 `mvc.Router` 替代 `http.ServeMux`
- ✅ 使用 `r.GET()`、`r.POST()` 等 Vigo 路由方法
- ✅ 使用路由组（`r.Group()`）管理认证路由
- ✅ 支持路由参数（`:id`）

### 4. 数据库配置修复

**修复前**（简单的 YAML 配置）:
```yaml
databases:
  user:
    dsn: "root:pass@tcp(user-db:3306)/user_db"
    max_open: 50
    max_idle: 5
```

**修复后**（使用 Vigo 多数据库配置）:
```yaml

databases:
  # 默认数据库
  default:
```

### 5. 数据库操作修复

**修复前**（使用原始 SQL）:
```go
// 直接查询用户表，造成耦合
user, err := db.NewQuery(db.GlobalDB).
    Table("users").
    Where("id", order.UserID).
    Find()
```

**修复后**（使用 Vigo ORM）:
```go
// app/service/order_service.go
type OrderService struct {
    orderRepo *repository.OrderRepository
    userDB    *db.DB  // 用户数据库连接
}

func NewOrderService() *OrderService {
    return &OrderService{
        orderRepo: repository.NewOrderRepository(),
        userDB: db.GetDB("user"),  // 获取用户数据库连接
    }
}

// GetOrderWithUser 获取订单及用户信息（跨数据库查询）
func (s *OrderService) GetOrderWithUser(ctx context.Context, orderID int64) (*model.Order, *model.User, error) {
    // 从订单数据库获取订单
    order, err := s.orderRepo.GetByID(ctx, orderID)
    if err != nil {
        return nil, nil, err
    }

    // 从用户数据库获取用户（使用独立数据库连接）
    var user model.User
    err = s.userDB.Table("users").
        Where("id", order.UserID).
        First(&user).Error
    if err != nil {
        return nil, nil, fmt.Errorf("获取用户信息失败：%w", err)
    }

    return order, &user, nil
}
```

**改进点**:
- ✅ 使用 `db.GetDB("user")` 获取指定数据库连接
- ✅ 使用 Vigo ORM 的 `First()` 方法
- ✅ 使用 `Error` 属性检查错误
- ✅ 添加错误包装（`fmt.Errorf`）
