# 微服务架构

## 简介
微服务架构是一种将应用拆分为多个小型、独立服务的架构风格。每个服务运行在自己的进程中，通过轻量级通信机制（如HTTP、gRPC）进行交互。

## 微服务拆分原则

### 服务拆分策略

#### 按业务领域拆分
```
单体应用
├── 用户模块
├── 订单模块
├── 商品模块
├── 支付模块
└── 库存模块

拆分为微服务
├── user-service      # 用户服务
├── order-service     # 订单服务
├── product-service   # 商品服务
├── payment-service   # 支付服务
└── inventory-service # 库存服务
```

#### 按数据边界拆分
- 每个服务拥有独立的数据库
- 避免跨服务的数据库访问
- 通过API进行数据交互

### 服务拆分示例

#### 用户服务
```go
// user-service/main.go
package main

import (
    "net/http"
    "user-service/controller"
    "user-service/model"
    "user-service/db"
)

func main() {
    // 初始化用户数据库
    db.Init("mysql", "user:pass@tcp(user-db:3306)/user_db", 100, 10, 3600)
    
    // 注册路由
    http.HandleFunc("/api/users", controller.UserList)
    http.HandleFunc("/api/users/", controller.UserDetail)
    http.HandleFunc("/api/auth/login", controller.Login)
    http.HandleFunc("/api/auth/register", controller.Register)
    
    // 启动服务
    http.ListenAndServe(":8001", nil)
}
```

## 服务间通信

### HTTP REST通信
```go
// 调用用户服务获取用户信息
func getUserFromUserService(userID int64) (*User, error) {
    resp, err := http.Get(fmt.Sprintf("http://user-service:8001/api/users/%d", userID))
    if err != nil {
        return nil, err
    }
    defer resp.Body.Close()
    
    var result struct {
        Code int         `json:"code"`
        Data interface{} `json:"data"`
    }
    
    if err := json.NewDecoder(resp.Body).Decode(&result); err != nil {
        return nil, err
    }
    
    return result.Data, nil
}

// 在订单服务中调用用户服务
func (oc *OrderController) GetOrderWithUser(w http.ResponseWriter, r *http.Request) {
    orderID := r.URL.Query().Get("id")
    
    // 获取订单信息
    order, err := getOrder(orderID)
    if err != nil {
        http.Error(w, err.Error(), http.StatusInternalServerError)
        return
    }
    
    // 调用用户服务获取用户信息
    user, err := getUserFromUserService(order.UserID)
    if err != nil {
        http.Error(w, err.Error(), http.StatusInternalServerError)
        return
    }
    
    json.NewEncoder(w).Encode(map[string]interface{}{
        "order": order,
        "user":  user,
    })
}
```

```go
syntax = "proto3";

package user;

```

```go
// user-service/grpc/server.go
package grpc

import (
    "context"
    pb "user-service/proto"
    "user-service/model"
)

type UserServer struct {
    pb.UnimplementedUserServiceServer
}

func (s *UserServer) GetUser(ctx context.Context, req *pb.GetUserRequest) (*pb.GetUserResponse, error) {
    user := &model.User{}
    userInfo, err := user.FindByID(req.Id)
    if err != nil {
        return nil, err
    }
    
    return &pb.GetUserResponse{
        Id:     userInfo.ID,
        Name:   userInfo.Name,
        Email:  userInfo.Email,
        Status: int32(userInfo.Status),
    }, nil
}

// 启动gRPC服务
func StartGRPCServer() {
    lis, err := net.Listen("tcp", ":50051")
    if err != nil {
        log.Fatalf("failed to listen: %v", err)
    }
    
    s := grpc.NewServer()
    pb.RegisterUserServiceServer(s, &UserServer{})
    
    if err := s.Serve(lis); err != nil {
        log.Fatalf("failed to serve: %v", err)
    }
}
```

## 服务注册与发现

### Nacos服务注册
```go
package discovery

import (
    "github.com/nacos-group/nacos-sdk-go/clients"
    "github.com/nacos-group/nacos-sdk-go/clients/naming_client"
    "github.com/nacos-group/nacos-sdk-go/vo"
)

// NacosDiscovery Nacos服务发现
type NacosDiscovery struct {
    client naming_client.INamingClient
}

func NewNacosDiscovery(serverAddr string) (*NacosDiscovery, error) {
    client, err := clients.NewNamingClient(
        vo.NacosClientParam{
            ServerConfigs: []constant.ServerConfig{
                {
                    IpAddr: serverAddr,
                    Port:   8848,
                },
            },
        },
    )
    if err != nil {
        return nil, err
    }
    
    return &NacosDiscovery{client: client}, nil
}

// RegisterService 注册服务
func (nd *NacosDiscovery) RegisterService(serviceName, ip string, port uint64) error {
    _, err := nd.client.RegisterInstance(vo.RegisterInstanceParam{
        ServiceName: serviceName,
        Ip:          ip,
        Port:        port,
        Weight:      10,
        Enable:      true,
        Healthy:     true,
        Ephemeral:   true,
    })
    return err
}

// DiscoverService 发现服务
func (nd *NacosDiscovery) DiscoverService(serviceName string) ([]string, error) {
    instances, err := nd.client.SelectInstances(vo.SelectInstancesParam{
        ServiceName: serviceName,
        HealthyOnly: true,
    })
    if err != nil {
        return nil, err
    }
    
    var addrs []string
    for _, instance := range instances {
        addrs = append(addrs, fmt.Sprintf("%s:%d", instance.Ip, instance.Port))
    }
    
    return addrs, nil
}
```

## API网关

### Gateway配置
```go
// gateway/main.go
package main

import (
    "net/http"
    "net/http/httputil"
    "net/url"
)

// Gateway API网关
type Gateway struct {
    routes map[string]string
}

func NewGateway() *Gateway {
    return &Gateway{
        routes: map[string]string{
            "/api/users":    "http://user-service:8001",
            "/api/orders":   "http://order-service:8002",
            "/api/products": "http://product-service:8003",
            "/api/payments": "http://payment-service:8004",
        },
    }
}

func (g *Gateway) ServeHTTP(w http.ResponseWriter, r *http.Request) {
    // 查找目标服务
    for prefix, target := range g.routes {
        if strings.HasPrefix(r.URL.Path, prefix) {
            g.proxy(w, r, target)
            return
        }
    }
    
    http.NotFound(w, r)
}

func (g *Gateway) proxy(w http.ResponseWriter, r *http.Request, target string) {
    targetURL, _ := url.Parse(target)
    
    proxy := httputil.NewSingleHostReverseProxy(targetURL)
    proxy.Director = func(req *http.Request) {
        req.URL.Scheme = targetURL.Scheme
        req.URL.Host = targetURL.Host
        req.URL.Path = strings.TrimPrefix(req.URL.Path, "/api")
        req.Header.Set("X-Forwarded-Host", r.Host)
    }
    
    proxy.ServeHTTP(w, r)
}

func main() {
    gateway := NewGateway()
    http.ListenAndServe(":8080", gateway)
}
```

## 分布式事务

### Saga模式
```go
package transaction

// SagaStep Saga步骤
type SagaStep struct {
    Name    string
    Execute func() error
    Compensate func() error
}

// Saga Saga事务
type Saga struct {
    steps []SagaStep
}

func NewSaga() *Saga {
    return &Saga{steps: make([]SagaStep, 0)}
}

func (s *Saga) AddStep(step SagaStep) {
    s.steps = append(s.steps, step)
}

func (s *Saga) Execute() error {
    executedSteps := make([]int, 0)
    
    for i, step := range s.steps {
        if err := step.Execute(); err != nil {
            // 执行失败，回滚已执行的步骤
            for j := len(executedSteps) - 1; j >= 0; j-- {
                s.steps[executedSteps[j]].Compensate()
            }
            return err
        }
        executedSteps = append(executedSteps, i)
    }
    
    return nil
}

// 创建订单Saga示例
func CreateOrderSaga(userID, productID int64, amount float64) error {
    saga := NewSaga()
    
    // 步骤1: 创建订单
    saga.AddStep(SagaStep{
        Name: "create_order",
        Execute: func() error {
            return createOrder(userID, productID, amount)
        },
        Compensate: func() error {
            return cancelOrder(userID, productID)
        },
    })
    
    // 步骤2: 扣减库存
    saga.AddStep(SagaStep{
        Name: "deduct_inventory",
        Execute: func() error {
            return deductInventory(productID, 1)
        },
        Compensate: func() error {
            return restoreInventory(productID, 1)
        },
    })
    
    // 步骤3: 扣款
    saga.AddStep(SagaStep{
        Name: "deduct_balance",
        Execute: func() error {
            return deductBalance(userID, amount)
        },
        Compensate: func() error {
            return restoreBalance(userID, amount)
        },
    })
    
    return saga.Execute()
}
```

## 注意事项
1. 服务拆分应遵循单一职责原则
2. 每个服务应有独立的数据库
3. 服务间通信应考虑超时和重试机制
4. 使用服务注册与发现管理服务实例
5. API网关统一入口，处理认证、限流等
6. 分布式事务需要考虑数据一致性
7. 监控和日志需要统一收集
8. 配置中心管理各服务配置
