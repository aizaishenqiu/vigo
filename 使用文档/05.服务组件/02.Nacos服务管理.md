# Nacos 服务管理

框架集成了 Nacos 客户端，支持配置管理、服务注册与发现、自动心跳，并提供 Web 管理界面。

---

## 1. Nacos 服务器准备

在使用 Nacos 服务前，需要先部署 Nacos 服务器。您可以选择以下任一方式：

### 1.1 手动下载安装

**Nacos 下载地址：** [https://github.com/alibaba/nacos/releases](https://github.com/alibaba/nacos/releases)

当前推荐版本：**3.1.1** (支持 Java 17)

安装和启动步骤：

1. 从上述链接下载 Nacos 压缩包
2. 解压后进入 `bin` 目录
3. Windows 系统执行：`startup.cmd -m standalone`
4. Linux/Mac 系统执行：`sh startup.sh -m standalone`
5. 等待控制台显示启动成功信息

### 1.2 Docker 部署（推荐）

您也可以使用 Docker 快速部署 Nacos：

```bash
# 使用 Docker Compose 部署（参考 Docker 部署指南）
docker run -d --name nacos-standalone \
  --env MODE=standalone \
  --env NACOS_APPLICATION_PORT=8848 \
  --env JVM_XMS=512m \
  --env JVM_XMX=512m \
  --env JVM_XMN=256m \
  -p 8848:8848 \
  -p 9848:9848 \
  nacos/nacos-server:v2.3.0
```

### 1.3 自动检测和启动

Vigo 框架提供了自动检测和启动 Nacos 功能。在配置文件中指定 Nacos 安装路径后，框架将自动：

- 检测 Nacos 是否正在运行
- 如果未运行，尝试自动启动 Nacos
- 如果未安装，提供详细的安装指导

配置示例：

```yaml
nacos:
  install_path: "/path/to/nacos" # 指定 Nacos 安装路径
  # ... 其他配置
```

> 注意：自动启动功能需要预先安装 Nacos 并正确配置 JAVA 环境。

## 2. 配置

在 `config.yaml` 中配置：

```yaml
nacos:
  host: 127.0.0.1
  port: 8848
  namespace: "" # 命名空间 ID（空为 public）
  data_id: vigo-config # 默认配置 Data ID
  group: DEFAULT_GROUP # 默认分组
  install_path: "" # Nacos安装路径，用于自动启动Nacos服务

  # 服务发现配置
  discovery:
    enabled: true # 启用服务发现
    service_name: vigo # 服务名称（默认使用 app.name）
    auto_register: true # 启动时自动注册
    heartbeat: 5 # 心跳间隔(秒)

  # 管理界面配置
  admin:
    enabled: true # 启用管理界面
    username: "admin" # 管理界面用户名（留空则不启用认证）
    password: "your_password" # 管理界面密码（留空则不启用认证）
    port: 8848
```

配置说明：

- `install_path`: 指定Nacos安装路径，如果配置此选项，框架将能够自动检测和启动Nacos服务
- `host/port`: Nacos服务器地址和端口
- `namespace`: Nacos命名空间ID
- `data_id/group`: 默认配置的Data ID和分组
- `discovery.enabled`: 是否启用服务发现功能
- `discovery.auto_register`: 是否自动注册服务
- `admin.enabled`: 是否启用Web管理界面

## 3. 自动注册

启用 `discovery.enabled` 和 `discovery.auto_register` 后，应用启动时会自动：

1. 检测 Nacos 服务是否可用
2. 自动获取本机 IP
3. 注册服务实例
4. 启动心跳保持服务在线

启动日志示例：

```
[INFO] Nacos 服务连接成功
[Nacos] 服务注册成功: vigo @ 192.168.1.100:8080
```

## 5. 代码中使用

### 通过门面访问

```go
import "vigo/framework/facade"

nc := facade.Nacos()
```

### 配置管理

```go
// 获取默认配置
content, err := nc.GetConfig()

// 获取指定配置
content, err := nc.GetConfigByID("app-config", "DEFAULT_GROUP")

// 发布配置
err := nc.PublishConfig("app-config", "DEFAULT_GROUP", `{"key": "value"}`)

// 删除配置
err := nc.DeleteConfig("app-config", "DEFAULT_GROUP")
```

### 服务注册与发现

```go
// 手动注册实例（通常使用自动注册）
err := nc.RegisterInstance("127.0.0.1", 8080, "my-service")

// 注销实例
err := nc.DeregisterInstance("127.0.0.1", 8080, "my-service")

// 获取服务列表
services, err := nc.ListServices(1, 20)

// 获取服务实例（原始数据）
instances, err := nc.GetServiceInstances("my-service")

// 服务发现 - 获取健康实例列表
instances, err := nc.DiscoverService("my-service")
for _, inst := range instances {
    if inst.Healthy {
        fmt.Printf("服务实例: %s:%d (权重: %.1f)\n", inst.IP, inst.Port, inst.Weight)
    }
}
```

### 心跳管理

```go
// 手动启动心跳（自动注册时会自动启动）
nc.StartHeartbeat(5 * time.Second)

// 停止心跳并注销服务
nc.Close()
```

### 命名空间

```go
namespaces, err := nc.ListNamespaces()
```

### 健康检查

```go
healthy := nc.CheckHealth()  // bool
connected := nc.IsConnected() // bool
```

## 7. 微服务架构

结合 gRPC 使用，可以实现完整的微服务架构：

```
┌────────────┐     ┌─────────┐     ┌────────────┐
│  Service A  │────>│  Nacos  │<────│  Service B  │
│  (gRPC)    │     │ (注册)   │     │  (gRPC)    │
└────────────┘     └─────────┘     └────────────┘
```

1. 各服务启动时向 Nacos 注册自身（自动注册）
2. 调用方从 Nacos 发现目标服务地址（服务发现）
3. 通过 gRPC 进行服务间通信

## 8. 服务发现示例

```go
// 在服务消费者中
nc := facade.Nacos()

// 发现目标服务
instances, err := nc.DiscoverService("user-service")
if err != nil || len(instances) == 0 {
    log.Fatal("服务不可用")
}

// 选择一个健康实例（可加入负载均衡逻辑）
var target *nacos.Instance
for _, inst := range instances {
    if inst.Healthy {
        target = &inst
        break
    }
}

// 连接 gRPC 服务
conn, err := grpc.Dial(fmt.Sprintf("%s:%d", target.IP, target.Port), ...)
```

## 9. 优雅关闭

应用关闭时会自动：

1. 停止心跳 goroutine
2. 从 Nacos 注销服务实例

```
[Nacos] 服务注销: vigo
```

## 10. 自动检测和启动机制

Vigo框架提供了智能的Nacos服务检测和启动机制：

### 10.1 连接检测

当应用启动时，框架会自动：

- 检查Nacos服务是否可达
- 验证服务健康状态
- 如果服务不可达，尝试启动Nacos（需要配置install_path）

### 10.2 自动启动

如果配置了`install_path`，框架会：

- 检测Nacos是否已安装
- 检测Nacos是否正在运行
- 尝试自动启动Nacos服务

### 10.3 用户友好提示

如果Nacos未安装或无法启动，框架会：

- 显示详细的安装说明
- 提供下载链接
- 给出启动指令

### 10.4 Docker环境支持

在Docker环境中，推荐使用Docker Compose统一管理服务依赖，参见 [《Docker部署指南》](Docker部署指南.md)。
