# gRPC 微服务

框架集成了 gRPC，支持在单体和微服务模式之间无缝切换。

---

## 1. 配置

在 `config.yaml` 中配置：

```yaml
grpc:
  enabled: true              # 设为 true 启用 gRPC 服务
  port: 9090                 # gRPC 监听端口
  service_name: vigo   # 服务名称
  enable_recovery: true      # 异常恢复拦截器
  enable_logger: true        # 日志拦截器
  max_recv_msg_size: 4       # 最大接收消息 (MB)
  max_send_msg_size: 4       # 最大发送消息 (MB)
```

## 2. 服务端

框架在 `App.Run()` 时自动启动 gRPC 服务：

```
[Vigo] gRPC 服务监听: :9090
```

### 内置功能

- **健康检查**: 自动注册 gRPC Health 服务
- **异常恢复**: panic 不会导致整个服务崩溃
- **日志拦截**: 记录每个 RPC 调用的耗时和状态
- **优雅停机**: `GracefulStop()` 等待进行中的请求完成

### 注册自定义服务

```go
import "vigo/framework/grpc"

// 获取 gRPC Server 实例
grpcServer := grpc.GetServer()

// 注册你的 protobuf 服务
pb.RegisterYourServiceServer(grpcServer.Server, &YourServiceImpl{})
```

## 3. 单体 vs 微服务切换

### 单体模式
```yaml
grpc:
  enabled: false  # 关闭 gRPC，只运行 HTTP
```

### 微服务模式
```yaml
grpc:
  enabled: true   # 同时运行 HTTP + gRPC
```

结合 Nacos 实现服务注册与发现：

```go
// 启动时注册到 Nacos
nc := facade.Nacos()
nc.RegisterInstance("127.0.0.1", 9090, "user-service")

// 停机时注销
nc.DeregisterInstance("127.0.0.1", 9090, "user-service")
```

## 4. 拦截器

### 内置拦截器

- **RecoveryInterceptor**: 捕获 panic，返回 Internal 错误
- **LoggerInterceptor**: 记录方法名、耗时、错误信息

### 自定义拦截器

```go
func AuthInterceptor() grpc.UnaryServerInterceptor {
    return func(ctx context.Context, req interface{}, info *grpc.UnaryServerInfo, handler grpc.UnaryHandler) (interface{}, error) {
        // 鉴权逻辑
        token := extractToken(ctx)
        if !validateToken(token) {
            return nil, status.Errorf(codes.Unauthenticated, "无效的 token")
        }
        return handler(ctx, req)
    }
}
```
