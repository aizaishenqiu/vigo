# Gateway API 网关使用详解

Vigo 框架提供轻量级的 API 网关功能，支持路由转发、请求聚合、协议转换、限流熔断等特性。

---

## 一、什么是 Gateway

Gateway（API 网关）是微服务架构中的核心组件，负责：

- **路由转发**: 将请求转发到对应的微服务
- **请求聚合**: 聚合多个微服务的响应
- **协议转换**: HTTP、gRPC、WebSocket 等协议互转
- **安全防护**: 认证、鉴权、限流、熔断
- **监控统计**: 请求统计、性能监控、日志记录

## 二、快速开始

### 2.1 启用 Gateway

在 `config.yaml` 中配置 Gateway：

```yaml
gateway:
  enabled: true
  port: 8888  # Gateway 监听端口
  
  # 超时配置
  timeout:
    connect: 10    # 连接超时（秒）
    read: 30       # 读取超时（秒）
    write: 30      # 写入超时（秒）
  
  # 限流配置
  rate_limit:
    enabled: true
    rate: 100      # 每秒请求数
    burst: 200     # 突发请求数
  
  # 熔断配置
  circuit_breaker:
    enabled: true
    threshold: 5   # 失败阈值
    timeout: 60    # 熔断超时（秒）
```

### 2.2 定义路由规则

```go
// app/gateway/routes.go
package gateway

import (
    "vigo/framework/gateway"
)

func InitRoutes() {
    // 添加路由规则
    gateway.AddRoute(gateway.Route{
        Name:        "user-service",     // 路由名称
        Path:        "/api/user/*path",  // 匹配路径
        Target:      "http://localhost:8081",  // 目标服务
        StripPrefix: true,              // 是否去除前缀
        Timeout:     30,                 // 超时时间（秒）
    })
    
    gateway.AddRoute(gateway.Route{
        Name:        "order-service",
        Path:        "/api/order/*path",
        Target:      "http://localhost:8082",
        StripPrefix: true,
        Timeout:     30,
    })
    
    gateway.AddRoute(gateway.Route{
        Name:        "product-service",
        Path:        "/api/product/*path",
        Target:      "http://localhost:8083",
        StripPrefix: true,
        Timeout:     30,
    })
}
```

## 三、路由转发

### 3.1 简单转发

```go
// 配置转发规则
gateway.AddRoute(gateway.Route{
    Name:   "user-service",
    Path:   "/user/*path",
    Target: "http://user-service:8080",
})

// 请求示例:
// GET http://gateway:8888/user/profile
// ↓ 转发到
// GET http://user-service:8080/profile
```

### 3.2 负载均衡

```go
// 配置多个目标地址，自动负载均衡
gateway.AddRoute(gateway.Route{
    Name:    "user-service",
    Path:    "/user/*path",
    Targets: []string{
        "http://user-service-1:8080",
        "http://user-service-2:8080",
        "http://user-service-3:8080",
    },
    LBStrategy: "round_robin",  // 负载均衡策略：round_robin, random, least_conn
})
```

### 3.3 路径重写

```go
// 配置路径重写
gateway.AddRoute(gateway.Route{
    Name:        "legacy-api",
    Path:        "/old/api/*path",
    Target:      "http://new-api:8080",
    StripPrefix: true,           // 去除 /old/api 前缀
    PathRewrite: "/v2/api/$1",   // 重写为 /v2/api/*path
})

// 请求示例:
// GET /old/api/user/123
// ↓ 转发到
// GET /v2/api/user/123
```

## 四、中间件

### 4.1 认证中间件

```go
// app/gateway/middleware/auth.go
package gateway

import (
    "net/http"
    "vigo/framework/gateway"
)

// Auth 认证中间件
func Auth() gateway.Middleware {
    return func(c *gateway.Context) {
        // 获取 Token
        token := c.GetHeader("Authorization")
        
        // 验证 Token
        if token == "" {
            c.AbortWithStatus(http.StatusUnauthorized)
            return
        }
        
        // 解析 Token
        claims, err := parseToken(token)
        if err != nil {
            c.AbortWithStatus(http.StatusUnauthorized)
            return
        }
        
        // 将用户信息传递给后端服务
        c.SetHeader("X-User-ID", claims.UserID)
        c.SetHeader("X-User-Role", claims.Role)
        
        c.Next()
    }
}

// 应用中间件
gateway.Use("/api/*", Auth())
```

### 4.2 限流中间件

```go
// 限流中间件
func RateLimit() gateway.Middleware {
    limiter := rate.NewLimiter(rate.Every(time.Second), 100)
    
    return func(c *gateway.Context) {
        if !limiter.Allow() {
            c.AbortWithStatusJSON(http.StatusTooManyRequests, map[string]interface{}{
                "code":    429,
                "message": "请求过于频繁",
            })
            return
        }
        c.Next()
    }
}

// 应用限流
gateway.Use("/*", RateLimit())
```

### 4.3 日志中间件

```go
// 日志中间件
func Logger() gateway.Middleware {
    return func(c *gateway.Context) {
        start := time.Now()
        
        c.Next()
        
        // 记录请求日志
        log.Infof("[%d] %s %s - %v",
            c.Response.Status,
            c.Request.Method,
            c.Request.URL.Path,
            time.Since(start),
        )
    }
}

// 全局应用
gateway.Use("/*", Logger())
```

## 五、请求聚合

### 5.1 并行请求

```go
// 定义聚合路由
gateway.AddAggregateRoute(gateway.AggregateRoute{
    Name:     "user-detail",
    Path:     "/api/user-detail/:id",
    Timeout:  10,
    Requests: []gateway.AggregateRequest{
        {
            Name:   "user",
            Method: "GET",
            URL:    "http://user-service/user/:id",
        },
        {
            Name:   "profile",
            Method: "GET",
            URL:    "http://profile-service/profile/:id",
        },
        {
            Name:   "orders",
            Method: "GET",
            URL:    "http://order-service/orders?user_id=:id",
        },
    },
})

// 响应结果:
// {
//     "user": { ... },
//     "profile": { ... },
//     "orders": [ ... ]
// }
```

### 5.2 串行请求

```go
// 依赖前一个请求的结果
gateway.AddAggregateRoute(gateway.AggregateRoute{
    Name:    "create-order",
    Path:    "/api/create-order",
    Timeout: 15,
    Requests: []gateway.AggregateRequest{
        {
            Name:   "validate",
            Method: "POST",
            URL:    "http://validate-service/validate",
        },
        {
            Name:      "create",
            Method:    "POST",
            URL:       "http://order-service/create",
            DependsOn: "validate",  // 依赖 validate 请求
            Body: map[string]interface{}{
                "data": "${validate.data}",  // 使用上一个请求的结果
            },
        },
    },
})
```

## 七、熔断降级

### 7.1 熔断配置

```go
// 配置熔断器
gateway.AddRoute(gateway.Route{
    Name:   "order-service",
    Path:   "/api/order/*path",
    Target: "http://order-service:8080",
    CircuitBreaker: gateway.CircuitBreakerConfig{
        Enabled:     true,
        Threshold:   5,    // 连续失败 5 次触发熔断
        Timeout:     60,   // 熔断 60 秒
        HalfOpen:    true, // 启用半开状态
    },
})
```

### 7.2 降级处理

```go
// 配置降级响应
gateway.AddRoute(gateway.Route{
    Name:   "order-service",
    Path:   "/api/order/*path",
    Target: "http://order-service:8080",
    Fallback: gateway.FallbackConfig{
        Mode: "static",  // static: 静态响应，proxy: 代理到其他服务
        Response: map[string]interface{}{
            "code":    503,
            "message": "服务暂时不可用",
        },
    },
})
```

```go
gateway.AddRoute(gateway.Route{
    Name:   "user-service",
    Path:   "/api/user/*path",
    Target: "http://user-service:8080",
    HealthCheck: gateway.HealthCheckConfig{
        Enabled:  true,
        Interval: 30,  // 每 30 秒检查一次

        Timeout:  5,
    },

health := gateway.GetServiceHealth("user-service")
if health.IsHealthy {
    fmt.Println("服务健康")
} else {
    fmt.Println("服务异常")
}
```

## 九、完整示例

### 9.1 Gateway 配置

```go
// app/gateway/init.go
package gateway

import (
    "vigo/framework/gateway"
)

func Init() {
    // 1. 注册中间件
    gateway.Use("/*", Logger())
    gateway.Use("/api/*", Auth())
    gateway.Use("/*", RateLimit())
    
    // 2. 注册路由
    AddRoutes()
    
    // 3. 注册聚合路由
    AddAggregateRoutes()
    
    // 4. 启动健康检查
    gateway.StartHealthCheck()
}

func AddRoutes() {
    // 用户服务
    gateway.AddRoute(gateway.Route{
        Name:        "user-service",
        Path:        "/api/user/*path",
        Target:      "http://localhost:8081",
        StripPrefix: true,
        Timeout:     30,
        LBStrategy:  "round_robin",
    })
    
    // 订单服务
    gateway.AddRoute(gateway.Route{
        Name:        "order-service",
        Path:        "/api/order/*path",
        Target:      "http://localhost:8082",
        StripPrefix: true,
        Timeout:     30,
        CircuitBreaker: gateway.CircuitBreakerConfig{
            Enabled:   true,
            Threshold: 5,
            Timeout:   60,
        },
    })
    
    // 商品服务
    gateway.AddRoute(gateway.Route{
        Name:        "product-service",
        Path:        "/api/product/*path",
        Target:      "http://localhost:8083",
        StripPrefix: true,
        Timeout:     30,
    })
}

func AddAggregateRoutes() {
    // 用户详情聚合
    gateway.AddAggregateRoute(gateway.AggregateRoute{
        Name:    "user-detail",
        Path:    "/api/user-detail/:id",
        Timeout: 10,
        Requests: []gateway.AggregateRequest{
            {
                Name:   "user",
                Method: "GET",
                URL:    "http://user-service/user/:id",
            },
            {
                Name:   "orders",
                Method: "GET",
                URL:    "http://order-service/orders?user_id=:id",
            },
        },
    })
}
```

### 9.2 启动 Gateway

```go
// main.go
func main() {
    // 初始化配置
    config.Init()
    
    // 初始化 Gateway
    gateway.Init()
    
    // 启动 Gateway 服务
    gatewayPort := 8888
    log.Printf("启动 API Gateway 端口：%d", gatewayPort)
    
    if err := gateway.Run(gatewayPort); err != nil {
        log.Fatalf("Gateway 启动失败：%v", err)
    }
}
```

## 十、最佳实践

### 10.1 路由设计

- 按业务模块划分路由
- 使用有意义的路由名称
- 合理设置超时时间
- 配置适当的限流策略

### 10.2 安全建议

- 实施统一的认证鉴权
- 配置 HTTPS 加密传输
- 限制请求大小和频率
- 记录详细的访问日志

### 10.3 性能优化

- 使用连接池复用连接
- 启用响应缓存
- 配置合理的超时时间
- 监控慢请求

### 10.4 监控告警

- 实时监控请求量
- 监控错误率
- 监控响应时间
- 配置熔断告警
