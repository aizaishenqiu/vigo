# 队列系统使用指南

Vigo 框架提供完整的队列系统，支持 Redis、Database、RabbitMQ 多种驱动，具备延迟队列、优先级队列、任务重试等高级功能。

## 一、快速开始

### 1.1 基本使用

```go
package main

import (
    "time"
    "vigo/framework/queue"
)

func main() {
    // 创建队列（使用 Redis 驱动）
    config := queue.RedisConfig{
        Host:  "localhost",
        Port:  6379,
        Queue: "default",
    }
    
    driver := queue.NewRedisDriver(config)
    q := queue.NewQueue(driver)
    
    // 推入任务
    q.Push(&SendEmailJob{
        To:      "user@example.com",
        Subject: "欢迎",
        Body:    "感谢注册！",
    })
    
    // 启动工作进程（5 个 worker）
    q.Listen("default", 5)
}
```

### 1.2 任务定义

```go
type SendEmailJob struct {
    To      string
    Subject string
    Body    string
}

// Handle 任务处理逻辑
func (j *SendEmailJob) Handle() error {
    // 发送邮件逻辑
    return nil
}

// Failed 任务失败处理
func (j *SendEmailJob) Failed(err error) {
    // 记录失败日志
    log.Printf("邮件发送失败：%v", err)
}
```

## 二、队列驱动

### 2.1 Redis 驱动（推荐）

**特点**: 高性能、支持延迟队列、生产环境首选

```go
config := queue.RedisConfig{
    Host:     "localhost",
    Port:     6379,
    Password: "",  // 可选
    DB:       0,   // 可选
    Queue:    "default",
}

driver := queue.NewRedisDriver(config)
q := queue.NewQueue(driver)
```

**配置说明**:
- `Host`: Redis 服务器地址
- `Port`: Redis 端口
- `Password`: 密码（可选）
- `DB`: 数据库编号（可选）
- `Queue`: 队列名称

---

### 2.2 Database 驱动

**特点**: 数据持久化、适合小规模应用

```go
config := queue.DatabaseConfig{
    Driver:   "mysql",  // mysql/postgres/sqlite
    Host:     "localhost",
    Port:     3306,
    User:     "root",
    Password: "password",
    Database: "vigo",
    Table:    "jobs",  // 队列表名
    Sleep:    1000,    // 轮询间隔（毫秒）
}

driver, err := queue.NewDatabaseDriver(config)
if err != nil {
    panic(err)
}

q := queue.NewQueue(driver)
```

**数据库表结构**:

```sql
CREATE TABLE jobs (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    payload TEXT NOT NULL,
    available_at DATETIME NOT NULL,
    reserved_at DATETIME DEFAULT NULL,
    retry_count INT DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 三、高级功能

### 3.1 延迟队列

```go
// 30 分钟后执行
q.PushWithDelay(&CancelOrderJob{
    OrderID: 1001,
}, 30*time.Minute)

// 明天上午 9 点执行
tomorrow9AM := time.Now().Add(24*time.Hour).Add(9*time.Hour)
q.PushAt(&ReminderJob{}, tomorrow9AM)
```

**应用场景**:
- 订单超时取消
- 定时提醒
- 延迟发送

---

### 3.2 任务重试

```go
// 自动重试 3 次
job := &SendEmailJob{}
jobWrapper := queue.NewJobWrapper(job)
jobWrapper.MaxRetries = 3

q.Push(jobWrapper)
```

**失败处理**:

```go
type Job struct {
    // ...
}

func (j *Job) Failed(err error) {
    // 任务最终失败时的处理
    log.Printf("任务失败：%v", err)
    
    // 可以发送告警通知
}
```

---

### 3.3 优先级队列

```go
// 高优先级任务
q.PushWithPriority(&VIPJob{}, queue.PriorityHigh)

// 普通优先级
q.PushWithPriority(&NormalJob{}, queue.PriorityNormal)

// 低优先级
q.PushWithPriority(&LowJob{}, queue.PriorityLow)
```

**优先级说明**:
- `PriorityHigh` (10): 高优先级，优先处理
- `PriorityNormal` (5): 普通优先级
- `PriorityLow` (1): 低优先级，最后处理

---

### 3.4 任务中间件

```go
// 定义任务中间件
func LoggingMiddleware(job queue.Job, next queue.NextFunc) error {
    log.Printf("开始处理任务：%T", job)
    start := time.Now()
    
    err := next(job)
    
    log.Printf("任务完成，耗时：%v", time.Since(start))
    return err
}

// 注册中间件
q.Use(LoggingMiddleware)
```

## 六、常见问题

### 6.1 任务重复执行

**原因**: 任务处理时间超过 `Reserved` 超时时间

**解决**:
```go
// 增加超时时间
jobWrapper.Reserved = 300 * time.Second  // 5 分钟
```

---

### 6.2 队列堆积

**原因**: 任务生产速度 > 消费速度

**解决**:
1. 增加 worker 数量
2. 优化任务处理逻辑
3. 使用优先级队列

---

### 6.3 Redis 连接失败

**解决**:
```go
// 添加重试逻辑
config := queue.RedisConfig{
    Host:          "localhost",
    Port:          6379,
    MaxRetries:    3,
    RetryInterval: 5 * time.Second,
}
```

## 九、完整示例

### 9.1 订单超时取消

```go
package jobs

type CancelOrderJob struct {
    OrderID int `json:"order_id"`
}

func (j *CancelOrderJob) Handle() error {
    // 检查订单状态
    order := db.Query("SELECT * FROM orders WHERE id = ?", j.OrderID)
    if order.Status != "pending" {
        return nil  // 已支付或已取消
    }
    
    // 取消订单
    db.Exec("UPDATE orders SET status = 'cancelled' WHERE id = ?", j.OrderID)
    
    // 发送通知
    notification.Send(order.UserID, "订单已超时取消")
    
    return nil
}

// 使用
job := &CancelOrderJob{OrderID: 1001}
q.PushWithDelay(job, 30*time.Minute)  // 30 分钟后检查
```

---

### 9.2 批量邮件发送

```go
package jobs

type BatchEmailJob struct {
    UserIDs []int  `json:"user_ids"`

    return nil
```

## 总结

Vigo 队列系统提供：

✅ **多驱动支持**: Redis/Database/RabbitMQ  
✅ **延迟队列**: 定时任务支持  
✅ **优先级队列**: 重要任务优先处理  
✅ **任务重试**: 自动重试机制  
✅ **多工作进程**: 并发处理  
✅ **优雅关闭**: 不丢失任务  

合理使用队列系统可以：
- 提升系统响应速度
- 削峰填谷
- 解耦业务逻辑
- 提高系统可靠性
