# WebSocket 实时通讯

框架内置了完整的 WebSocket 服务端，支持频道订阅、指令路由、心跳保活，适用于实时推送场景。

---

## 1. 架构设计

```
┌──────────┐    WebSocket     ┌──────────┐    BroadcastToChannel    ┌──────────┐
│  Browser  │ ────────────> │    Hub    │ <──────────────────── │  Service  │
│ (Client)  │ <──────────── │ (Server)  │ ──────────────────>  │ (业务层)  │
└──────────┘    push data    └──────────┘    OnCommand           └──────────┘
```

- **Hub**: 中央调度器，管理所有客户端连接、频道订阅、指令路由
- **Client**: 单个 WebSocket 连接的代理，负责读写分离和心跳
- **Channel**: 频道机制，客户端可订阅多个频道，服务端按频道广播

## 2. 服务端使用

### 注册指令处理器

```go
import "vigo/framework/websocket"

// 在 App.Initialize() 中创建 Hub
hub := websocket.NewHub()
go hub.Run()

// 注册指令
hub.OnCommand("chat", func(clientId string, data json.RawMessage) {
    var msg struct {
        Content string `json:"content"`
    }
    json.Unmarshal(data, &msg)
    
    // 广播到 chat 频道
    hub.BroadcastToChannel("chat", "message", map[string]interface{}{
        "from":    clientId,
        "content": msg.Content,
    })
})
```

### 向频道广播数据

```go
hub.BroadcastToChannel("benchmark", "stats", statsData)
```

### 检查在线状态

```go
count := hub.ClientCount()
hasSubs := hub.HasSubscribers("benchmark")
```

## 3. 客户端使用

### 连接

```javascript
const ws = new WebSocket('ws://localhost:8080/benchmark/ws?channel=benchmark');
```

### 订阅频道

连接时通过 URL 参数指定频道：`?channel=benchmark`

也可以在连接后发送订阅指令：
```javascript
ws.send(JSON.stringify({ type: "subscribe", channel: "chat" }));
```

### 发送指令

```javascript
ws.send(JSON.stringify({
    type: "command",
    channel: "benchmark",
    data: { action: "start", workers: 10 }
}));
```

### 接收消息

```javascript
ws.onmessage = function(event) {
    const msg = JSON.parse(event.data);
    // msg.type = "stats" / "message" / "response"
    // msg.data = { ... }
    console.log(msg);
};
```

## 4. 协议格式

### 客户端 -> 服务端

```json
{
    "type": "subscribe|unsubscribe|command",
    "channel": "频道名",
    "data": { "业务数据": "..." }
}
```

### 服务端 -> 客户端

```json
{
    "type": "消息类型",
    "data": { "业务数据": "..." }
}
```

## 5. 心跳机制

- 服务端每 30 秒发送一次 Ping
- 客户端需在 60 秒内响应 Pong
- 超时自动断开连接
- 客户端应实现自动重连（建议 3 秒延迟）

## 6. 最佳实践

- 使用频道机制隔离不同业务的消息流
- 高频推送场景（如监控数据）使用自适应频率
- 始终实现客户端重连逻辑
- 大量客户端时注意服务器内存和 goroutine 数量
