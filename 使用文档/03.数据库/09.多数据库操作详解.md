# 多数据库操作详解

Vigo 框架支持多数据库操作，允许不同的业务模块使用独立的数据库连接，实现数据隔离和性能优化。

---

## 一、什么是多数据库

多数据库是指在一个应用中配置和使用多个独立的数据库连接。每个数据库可以：

- **独立配置**: 不同的主机、端口、用户名、密码
- **独立业务**: 不同的业务模块使用不同的数据库
- **独立读写分离**: 每个数据库都可以配置自己的读写分离
- **独立连接池**: 每个数据库有独立的连接池配置

### 应用场景

- **业务隔离**: 用户中心、订单系统、商品系统使用不同数据库
- **数据分片**: 按地区、按用户 ID 等维度分库
- **历史数据**: 历史数据归档到独立数据库
- **多租户**: 不同租户使用独立数据库

## 二、配置多数据库

### 2.1 配置文件

在 `config.yaml` 中配置多个数据库：

```yaml
# ==============================================
# 主数据库配置（默认数据库）
# ==============================================
database:
  driver: mysql
  host: 192.168.1.10
  port: 3306
  name: vigo_main
  user: root
  pass: password
  charset: utf8mb4
  max_open_conns: 500
  max_idle_conns: 100
  conn_max_lifetime: 1800
  conn_max_idletime: 300

# ==============================================
# 多数据库配置（不同业务独立数据库）
# ==============================================
databases:
  # 用户中心数据库
  user_center:
    driver: mysql
    host: 192.168.2.10
    port: 3306
    name: user_center
    user: user_admin
    pass: user-pass
    charset: utf8mb4
    max_open_conns: 500
    max_idle_conns: 100
    conn_max_lifetime: 1800
    conn_max_idletime: 300
    
    # 可选：读写分离配置
    rw_split: true
    reads:
      - host: 192.168.2.20
        port: 3306
        user: user_reader
        pass: user-read-pass
        charset: utf8mb4

  # 订单数据库
  order_db:
    driver: mysql
    host: 192.168.3.10
    port: 3306
    name: order_db
    user: order_admin
    pass: order-pass
    charset: utf8mb4
    max_open_conns: 800
    max_idle_conns: 200
    conn_max_lifetime: 1800
    conn_max_idletime: 300

  # 商品数据库
  product_db:
    driver: mysql
    host: 192.168.4.10
    port: 3306
    name: product_db
    user: product_admin
    pass: product-pass
    charset: utf8mb4
    max_open_conns: 600
    max_idle_conns: 150
    conn_max_lifetime: 1800
    conn_max_idletime: 300
```

## 三、使用多数据库

### 3.1 基本使用

```go
import (
    "vigo/framework/db"
)

// 获取用户中心数据库连接
userDB, err := db.GetConnection("user_center")
if err != nil {
    log.Fatal("获取数据库连接失败:", err)
}

// 使用写库（默认）
users, err := db.NewQuery(userDB.GetWriteDB()).
    Table("users").
    Where("status", 1).
    Select()

// 使用读库（自动轮询）
user, err := db.NewQuery(userDB.GetReadDB()).
    Table("users").
    Where("id", 1).
    Find()
```

### 3.2 便捷方法

框架提供了便捷方法访问多数据库：

```go
// 方法 1: 使用 TableWithConn
users, err := db.TableWithConn("user_center", "users").
    Where("status", 1).
    Select()

// 方法 2: 先获取连接，再创建 Query
conn, err := db.GetConnection("user_center")
if err != nil {
    log.Fatal(err)
}

// 写入操作（使用写库）
id, err := db.NewQuery(conn.GetWriteDB()).
    Table("users").
    InsertGetId(map[string]interface{}{
        "name":  "张三",
        "email": "zhangsan@example.com",
    })

// 读取操作（自动使用读库）
users, err := db.NewQuery(conn.GetReadDB()).
    Table("users").
    Where("status", 1).
    Select()
```

### 3.3 事务操作

```go
// 在指定数据库中执行事务
err := db.TransactionWithConn("user_center", func(q *db.Query) error {
    // 插入用户
    _, err := q.Table("users").Insert(map[string]interface{}{
        "name":  "张三",
        "email": "zhangsan@example.com",
    })
    if err != nil {
        return err
    }
    
    // 插入用户资料
    _, err = q.Table("user_profile").Insert(map[string]interface{}{
        "user_id": 1,
        "age":     25,
        "city":    "北京",
    })
    if err != nil {
        return err
    }
    
    return nil
})

if err != nil {
    log.Fatal("事务执行失败:", err)
}
```

## 四、跨数据库操作

### 4.1 跨库查询

```go
// 从用户中心数据库查询
userDB, _ := db.GetConnection("user_center")
user, _ := db.NewQuery(userDB.GetReadDB()).
    Table("users").
    Where("id", 1).
    Find()

// 从订单数据库查询
orderDB, _ := db.GetConnection("order_db")
orders, _ := db.NewQuery(orderDB.GetReadDB()).
    Table("orders").
    Where("user_id", 1).
    Select()
```

### 4.2 跨库事务（不推荐）

由于不同数据库之间无法保证原子性，跨库事务需要谨慎使用：

```go
// 方案 1: 最终一致性（推荐）
// 1. 在用户中心创建用户
userDB, _ := db.GetConnection("user_center")
userID, _ := db.NewQuery(userDB.GetWriteDB()).
    Table("users").
    InsertGetId(userData)

// 2. 在订单数据库创建记录（通过消息队列保证最终一致性）
mq.Publish("user.created", map[string]interface{}{
    "user_id": userID,
})

// 方案 2: 补偿机制
err := func() error {
    // 1. 在用户中心创建用户
    userDB, _ := db.GetConnection("user_center")
    userID, err := db.NewQuery(userDB.GetWriteDB()).
        Table("users").
        InsertGetId(userData)
    if err != nil {
        return err
    }
    
    // 2. 在订单数据库创建记录
    orderDB, _ := db.GetConnection("order_db")
    _, err = db.NewQuery(orderDB.GetWriteDB()).
        Table("user_orders").
        Insert(map[string]interface{}{
            "user_id": userID,
        })
    if err != nil {
        // 补偿：删除用户中心的用户
        db.NewQuery(userDB.GetWriteDB()).
            Table("users").
            Where("id", userID).
            Delete()
        return err
    }
    
    return nil
}()
```

## 五、读写分离

每个数据库都可以独立配置读写分离：

```yaml
databases:
  user_center:
    driver: mysql
    host: 192.168.2.10  # 主库（写入）
    port: 3306
    name: user_center
    user: user_admin
    pass: user-pass
    
    # 启用读写分离
    rw_split: true
    
    # 从库配置（读取）
    reads:
      - host: 192.168.2.20
        port: 3306
        user: user_reader
        pass: user-read-pass
      - host: 192.168.2.21
        port: 3306
        user: user_reader
        pass: user-read-pass
```

使用方式：

```go
conn, _ := db.GetConnection("user_center")

// 自动使用读库（轮询从库）
users, _ := db.NewQuery(conn.GetReadDB()).
    Table("users").
    Where("status", 1).
    Select()

// 使用写库
id, _ := db.NewQuery(conn.GetWriteDB()).
    Table("users").
    InsertGetId(map[string]interface{}{
        "name":  "张三",
        "email": "zhangsan@example.com",
    })
```

## 六、最佳实践

### 6.1 连接池配置

根据不同数据库的负载情况，合理配置连接池：

```yaml
databases:
  # 高频读写的数据库（如订单）
  order_db:
    max_open_conns: 800    # 较大的连接池
    max_idle_conns: 200    # 保持较多空闲连接
    conn_max_lifetime: 1800
    conn_max_idletime: 300

  # 低频读写的数据库（如日志）
  log_db:
    max_open_conns: 100    # 较小的连接池
    max_idle_conns: 20     # 保持较少空闲连接
    conn_max_lifetime: 3600
    conn_max_idletime: 600
```

### 6.2 业务隔离

按业务模块划分数据库：

```
用户相关 -> user_center 数据库
  - users 表
  - user_profile 表
  - user_login_log 表

订单相关 -> order_db 数据库
  - orders 表
  - order_items 表
  - order_log 表

商品相关 -> product_db 数据库
  - products 表
  - categories 表
  - inventory 表
```

### 6.3 错误处理

```go
func GetUserByID(userID int64) (map[string]interface{}, error) {
    conn, err := db.GetConnection("user_center")
    if err != nil {
        log.Printf("获取数据库连接失败：%v", err)
        return nil, err
    }
    
    user, err := db.NewQuery(conn.GetReadDB()).
        Table("users").
        Where("id", userID).
        Find()
    
    if err != nil {
        log.Printf("查询用户失败：%v", err)
        return nil, err
    }
    
    if user == nil {
        return nil, fmt.Errorf("用户不存在")
    }
    
    return user, nil
}
```

### 6.4 性能优化

1. **避免跨库 JOIN**: 跨库 JOIN 性能极差，应通过应用层组装数据
2. **合理分库**: 按业务、按数据量、按访问频率分库
3. **连接复用**: 获取一次连接后复用，避免频繁调用 `GetConnection`
4. **监控连接池**: 定期检查连接池使用情况

```go
// ✅ 好的做法
conn, _ := db.GetConnection("user_center")
query := db.NewQuery(conn.GetReadDB())

// 多次使用同一个连接
users, _ := query.Table("users").Where("status", 1).Select()
count, _ := query.Table("users").Where("status", 1).Count()

// ❌ 不好的做法
users, _ := db.TableWithConn("user_center", "users").Where("status", 1).Select()
count, _ := db.TableWithConn("user_center", "users").Where("status", 1).Count()
```

## 七、常见问题

### Q1: 多数据库和读写分离可以同时使用吗？

**可以**。每个数据库都可以独立配置读写分离：

```yaml
databases:
  user_center:
    host: 192.168.2.10  # 主库
    rw_split: true
    reads:
      - host: 192.168.2.20  # 从库 1
      - host: 192.168.2.21  # 从库 2
```

### Q2: 如何判断应该使用哪个数据库？

根据业务模块划分：

- 用户相关操作 → `user_center`
- 订单相关操作 → `order_db`
- 商品相关操作 → `product_db`

建议在 Service 层封装数据库访问逻辑：

```go
type UserService struct {
    db *db.DBConnection
}

func NewUserService() *UserService {
    conn, _ := db.GetConnection("user_center")
    return &UserService{db: conn}
}

func (s *UserService) GetUserByID(id int64) (map[string]interface{}, error) {
    return db.NewQuery(s.db.GetReadDB()).
        Table("users").
        Where("id", id).
        Find()
}
```

### Q3: 多数据库会影响性能吗？

合理配置下，多数据库可以**提升性能**：

- **减少单库压力**: 分散到多个数据库
- **独立优化**: 每个库可以独立优化
- **故障隔离**: 一个库故障不影响其他库

但需要注意：
- 避免频繁的跨库操作
- 合理配置连接池大小
- 使用连接复用

### Q4: 如何迁移数据到多数据库？

1. **评估**: 分析现有数据结构和使用场景
2. **规划**: 设计数据库划分方案
3. **迁移**: 编写迁移脚本，逐步迁移数据
4. **验证**: 验证数据完整性和一致性
5. **切换**: 逐步切换代码，观察运行情况

### 8.2 模型层

```go
// app/model/user.go
package model

import (
    "vigo/framework/db"
)

type User struct {
    ID       int64  `json:"id"`
    Name     string `json:"name"`
    Email    string `json:"email"`
    Status   int    `json:"status"`
    CreateTime string `json:"create_time"`
}

func (u *User) TableName() string {
    return "users"
}

// GetUserByID 根据 ID 获取用户
func GetUserByID(id int64) (*User, error) {
    conn, err := db.GetConnection("user_center")
    if err != nil {
        return nil, err
    }
    
    data, err := db.NewQuery(conn.GetReadDB()).
        Table("users").
        Where("id", id).
        Find()
    
    if err != nil || data == nil {
        return nil, err
    }
    
    return &User{
        ID:         data["id"].(int64),
        Name:       data["name"].(string),
        Email:      data["email"].(string),
        Status:     int(data["status"].(int64)),
        CreateTime: data["create_time"].(string),
    }, nil
}

// CreateUser 创建用户
func CreateUser(data map[string]interface{}) (int64, error) {
    conn, err := db.GetConnection("user_center")
    if err != nil {
        return 0, err
    }
    
    return db.NewQuery(conn.GetWriteDB()).
        Table("users").
        InsertGetId(data)
}
```

### 8.3 服务层

```go
// app/service/order.go
package service

import (
    "vigo/framework/db"
    "vigo/app/model"
)

type OrderService struct {
    db *db.DBConnection
}

func NewOrderService() *OrderService {
    conn, _ := db.GetConnection("order_db")
    return &OrderService{db: conn}
}

// CreateOrder 创建订单
func (s *OrderService) CreateOrder(userID int64, items []map[string]interface{}) (int64, error) {
    var orderID int64
    
    // 事务操作
    err := db.TransactionWithConn("order_db", func(q *db.Query) error {
        // 1. 验证用户是否存在（跨库查询）
        user, err := model.GetUserByID(userID)
        if err != nil {
            return err
        }
        if user == nil {
            return fmt.Errorf("用户不存在")
        }
        
        // 2. 创建订单
        var err error
        orderID, err = q.Table("orders").InsertGetId(map[string]interface{}{
            "user_id": userID,
            "status":  1,
            "amount":  calculateAmount(items),
        })
        if err != nil {
            return err
        }
        
        // 3. 创建订单项
        for _, item := range items {
            item["order_id"] = orderID
            _, err := q.Table("order_items").Insert(item)
            if err != nil {
                return err
            }
        }
        
        return nil
    })
    
    return orderID, err
}

func calculateAmount(items []map[string]interface{}) float64 {
    var total float64
    for _, item := range items {
        price := item["price"].(float64)
        quantity := item["quantity"].(int64)
        total += price * float64(quantity)
    }
    return total
}
```

### 8.4 控制器层

```go
// app/controller/user.go
package controller

import (
    "net/http"
    "vigo/framework/mvc"
    "vigo/app/model"
)

// GetUser 获取用户信息
func GetUser(c *mvc.Context) {
    userID := c.ParamInt64("id")
    
    user, err := model.GetUserByID(userID)
    if err != nil {
        c.Json(http.StatusInternalServerError, map[string]interface{}{
            "code":    500,
            "message": "查询失败",
        })
        return
    }
    
    if user == nil {
        c.Json(http.StatusNotFound, map[string]interface{}{
            "code":    404,
            "message": "用户不存在",
        })
        return
    }
    
    c.Json(http.StatusOK, map[string]interface{}{
        "code": 0,
        "data": user,
    })
}

// CreateUser 创建用户
func CreateUser(c *mvc.Context) {
    var data map[string]interface{}
    if err := c.BindJSON(&data); err != nil {
        c.Json(http.StatusBadRequest, map[string]interface{}{
            "code":    400,
            "message": "参数错误",
        })
        return
    }
    
    id, err := model.CreateUser(data)
    if err != nil {
        c.Json(http.StatusInternalServerError, map[string]interface{}{
            "code":    500,
            "message": "创建失败",
        })
        return
    }
    
    c.Json(http.StatusCreated, map[string]interface{}{
        "code": 0,
        "data": map[string]interface{}{
            "id": id,
        },
    })
}
```

## 九、总结

多数据库是 Vigo 框架的重要特性，可以实现：

- ✅ **业务隔离**: 不同业务使用不同数据库
- ✅ **性能提升**: 分散压力，独立优化
- ✅ **扩展性强**: 支持水平扩展
- ✅ **故障隔离**: 单库故障不影响全局

使用时注意：
- 合理划分数据库边界
- 避免频繁跨库操作
- 配置合适的连接池
- 做好监控和日志

通过多数据库和读写分离的配合使用，可以构建高性能、高可用的企业级应用。
