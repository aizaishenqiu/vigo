## 概述

Vigo 框架提供完整的分库分表解决方案，支持：

- ✅ **水平分表** - 单库多表
- ✅ **水平分库** - 多库多表
- ✅ **多种分片策略** - 取模、哈希、范围、自定义
- ✅ **自动路由** - 根据分片键自动路由到对应分片
- ✅ **批量操作** - 跨分片批量查询和插入

### 适用场景

| 场景 | 说明 | 推荐方案 |
|------|------|----------|
| **单表数据量大** | 单表超过 500 万行 | 水平分表 |
| **并发写入高** | QPS > 5,000 | 水平分库 |
| **业务隔离** | 不同业务线数据隔离 | 水平分库 |
| **历史数据归档** | 按时间范围查询 | 范围分片 |

## 分片策略

### 1. 取模分片（ShardTypeMod）

**原理**: `shard_index = shard_key % shard_count`

**适合**: 数值型分片键（用户 ID、订单 ID 等）

**优点**: 
- 数据分布均匀
- 计算简单，性能高

**缺点**: 
- 扩容困难（需要数据迁移）

```go
// 示例：按用户 ID 分 4 张表
config := &db.ShardingConfig{
    Table:      "orders",
    ShardKey:   "user_id",
    ShardType:  db.ShardTypeMod,
    ShardCount: 4,
}
```

---

### 2. 哈希分片（ShardTypeHash）

**原理**: `shard_index = hash(shard_key) % shard_count`

**适合**: 字符串型分片键（订单号、用户名等）

**优点**: 
- 数据分布均匀
- 支持字符串类型

**缺点**: 
- 扩容困难

```go
// 示例：按订单号分 8 张表
config := &db.ShardingConfig{
    Table:      "orders",
    ShardKey:   "order_no",
    ShardType:  db.ShardTypeHash,
    ShardCount: 8,
}
```

---

### 3. 范围分片（ShardTypeRange）

**原理**: 按分片键的范围分布数据

**适合**: 时间范围、数值范围（创建时间、金额等）

**优点**: 
- 支持范围查询
- 扩容容易（添加新范围）

**缺点**: 
- 数据可能分布不均

```go
// 示例：按创建时间分表（每月一张）
config := &db.ShardingConfig{
    Table:      "orders",
    ShardKey:   "created_at",
    ShardType:  db.ShardTypeRange,
    ShardCount: 12, // 12 个月
}
```

---

### 4. 自定义分片（ShardTypeCustom）

**原理**: 使用自定义分片函数

**适合**: 特殊业务需求

**优点**: 
- 灵活性高
- 可实现复杂逻辑

**缺点**: 
- 需要自己实现分片逻辑

```go
// 示例：自定义分片函数
config := &db.ShardingConfig{
    Table:      "orders",
    ShardKey:   "user_id",
    ShardType:  db.ShardTypeCustom,
    ShardCount: 4,
    ShardFunc: func(key interface{}) int {
        userID := key.(int64)
        // VIP 用户单独分片
        if userID > 1000000 {
            return 0 // VIP 分片
        }
        return int(userID) % 4
    },
}
```

## 快速开始

### 步骤 1: 配置分片表

```go
import "vigo/framework/db"

// 初始化分片配置
err := db.AddShardingTable(&db.ShardingConfig{
    Table:      "orders",      // 表名
    ShardKey:   "user_id",     // 分片键
    ShardType:  db.ShardTypeMod, // 取模分片
    ShardCount: 4,             // 4 张表
})

if err != nil {
    log.Fatalf("配置分片表失败：%v", err)
}
```

### 步骤 2: 创建分片表

```sql
-- MySQL 示例
CREATE TABLE orders_00 LIKE orders;
CREATE TABLE orders_01 LIKE orders;
CREATE TABLE orders_02 LIKE orders;
CREATE TABLE orders_03 LIKE orders;
```

### 步骤 3: 使用分片查询

```go
// 自动路由到对应的分片
query, err := db.GetShardingQuery("orders", userID)
if err != nil {
    return err
}

// 正常查询
orderList, err := query.Where("user_id", userID).Select()
```

## 使用示例

### 示例 1: 按用户 ID 分表

#### 配置

```go
// 按用户 ID 取模分 4 张表
err := db.AddShardingTable(&db.ShardingConfig{
    Table:      "orders",
    ShardKey:   "user_id",
    ShardType:  db.ShardTypeMod,
    ShardCount: 4,
})
```

#### 插入数据

```go
// 方式 1：使用 Query
query, err := db.GetShardingQuery("orders", userID)
if err != nil {
    return err
}

_, err = query.Insert(map[string]interface{}{
    "user_id":    userID,
    "amount":     99.99,
    "status":     "pending",
    "created_at": time.Now(),
})

// 方式 2：使用 Model
model, err := db.GetShardingModel("orders", userID)
if err != nil {
    return err
}

model.SetAttr("user_id", userID)
model.SetAttr("amount", 99.99)
_, err = model.Insert()
```

#### 查询数据

```go
// 单条查询
query, err := db.GetShardingQuery("orders", userID)
order, err := query.Where("id", orderID).First()

// 列表查询
query, err := db.GetShardingQuery("orders", userID)
list, err := query.
    Where("user_id", userID).
    Where("status", "pending").
    OrderBy("created_at", "DESC").
    Limit(10).
    Select()
```

#### 更新数据

```go
query, err := db.GetShardingQuery("orders", userID)
_, err = query.
    Where("id", orderID).
    Update(map[string]interface{}{
        "status": "completed",
    })
```

#### 删除数据

```go
query, err := db.GetShardingQuery("orders", userID)
_, err = query.
    Where("id", orderID).
    Delete()
```

### 示例 3: 分库分表（多库多表）

#### 配置

```go
// 2 个数据库，每个库 4 张表，共 8 张表
dbMap := map[int]*sql.DB{
    0: db1, // 数据库 1
    1: db2, // 数据库 2
}

err := db.AddShardingTable(&db.ShardingConfig{
    Table:       "orders",
    ShardKey:    "order_id",
    ShardType:   db.ShardTypeHash,
    ShardCount:  8, // 8 个分片
    DatabaseMap: dbMap,
})
```

#### 原理

```
order_id = 1001 → 分片 3 → 数据库 1 → orders_03
order_id = 2002 → 分片 6 → 数据库 2 → orders_06
```

### 示例 4: 按时间范围分表

#### 配置

```go
// 自定义分片函数：按年份分表
err := db.AddShardingTable(&db.ShardingConfig{
    Table:      "orders",
    ShardKey:   "created_at",
    ShardType:  db.ShardTypeCustom,
    ShardCount: 5, // 5 年
    ShardFunc: func(key interface{}) int {
        t := key.(time.Time)
        year := t.Year()
        // 2020-2024 年，分别对应 0-4 分片
        return (year - 2020) % 5
    },
})
```

#### 使用

```go
// 插入 2024 年的订单
query, _ := db.GetShardingQuery("orders", time.Now())
query.Insert(map[string]interface{}{
    "user_id":    1001,
    "created_at": time.Now(),
})
```

## 批量操作

### 批量插入（跨分片）

```go
// 准备数据
dataList := []map[string]interface{}{
    {"user_id": 1001, "amount": 99.99},
    {"user_id": 1002, "amount": 199.99},
    {"user_id": 1003, "amount": 299.99},
    // ... 更多数据
}

// 获取路由器
router, err := GlobalShardingManager.GetRouter("orders")
if err != nil {
    return err
}

// 批量插入（自动按分片分组，并行插入）
err = router.BatchInsert(dataList)
if err != nil {
    return err
}
```

### 批量查询（跨所有分片）

```go
var allOrders []map[string]interface{}
var mu sync.Mutex

router, _ := GlobalShardingManager.GetRouter("orders")

// 跨所有分片查询
err := router.BatchQuery(func(q *db.Query) error {
    orders, err := q.
        Where("status", "pending").
        Limit(100).
        Select()
    
    if err != nil {
        return err
    }
    
    // 合并结果
    mu.Lock()
    allOrders = append(allOrders, orders...)
    mu.Unlock()
    
    return nil
})

if err != nil {
    log.Printf("批量查询失败：%v", err)
}
```

## 最佳实践

### 1. 分片键选择

✅ **推荐**:
- 用户 ID（适合按用户维度查询）
- 订单 ID（适合按订单维度查询）
- 商户 ID（适合多商户系统）

❌ **不推荐**:
- 自增 ID（数据分布不均）
- 时间戳（热点问题）
- 状态字段（数据倾斜）

---

### 2. 分片数量

| 数据量 | 推荐分片数 | 说明 |
|--------|-----------|------|
| < 100 万 | 2-4 | 初期无需太多分片 |
| 100-500 万 | 4-8 | 中等规模 |
| 500-1000 万 | 8-16 | 大规模 |
| > 1000 万 | 16-32 | 超大规模 |

**原则**: 
- 分片数 = 单表数据量 / 目标单表数据量（建议 100-200 万）
- 预留 20-30% 的扩容空间

---

### 3. 表命名规范

```
{table_name}_{shard_index}

示例:
orders_00
orders_01
orders_02
orders_03
```

**规范**:
- 使用下划线分隔
- 分片索引用 2 位数字（00-99）
- 统一小写

---

### 4. 扩容方案

#### 方案 1: 预留分片

```go
// 初期配置 4 个分片，预留 12 个
config := &db.ShardingConfig{
    Table:      "orders",
    ShardCount: 16, // 预留足够空间
}

// 初期只创建 4 张表
CREATE TABLE orders_00 ...
CREATE TABLE orders_01 ...
CREATE TABLE orders_02 ...
CREATE TABLE orders_03 ...

// 数据增长后，再创建新表
CREATE TABLE orders_04 ...
CREATE TABLE orders_05 ...
```

#### 方案 2: 双写迁移

```
1. 双写阶段：同时写入旧分片和新分片
2. 数据迁移：后台同步历史数据
3. 切换读：逐步切换读取到新分片
4. 切换写：完全切换到新分片
5. 清理旧数据：删除旧分片数据
```

---

### 5. 性能优化

```go
// ✅ 推荐：使用分片键查询
query, _ := db.GetShardingQuery("orders", userID)
query.Where("user_id", userID).Select()

// ❌ 不推荐：全分片扫描（性能差）
router, _ := GlobalShardingManager.GetRouter("orders")
router.BatchQuery(func(q *db.Query) error {
    // 需要扫描所有分片
    return q.Where("status", "pending").Select()
})
```

### 6. 监控告警

```go
// 监控各分片数据量
router, _ := GlobalShardingManager.GetRouter("orders")
shards := router.GetAllShards()

for _, shard := range shards {
    count, _ := db.NewQuery(shard.Database).
        Table(shard.TableName).
        Count()
    
    log.Printf("分片 %s: %d 条数据", shard.TableName, count)
    
    // 告警：单分片数据量过大
    if count > 2000000 {
        alert("分片数据量过大：%s - %d", shard.TableName, count)
    }
}
```

## 常见问题

### Q1: 分库分表后，如何全局唯一 ID？

**A**: 使用雪花算法（Snowflake）或 Redis 自增 ID

```go
// 雪花算法示例
type Snowflake struct {
    nodeID     int64
    sequence   int64
    lastTime   int64
}

func (sf *Snowflake) NextID() int64 {
    // 生成全局唯一 ID
    // 格式：时间戳 (41 位) + 节点 ID (10 位) + 序列号 (12 位)
}

// 使用
idGenerator := &Snowflake{nodeID: 1}
orderID := idGenerator.NextID()
```

---

### Q2: 分库分表后，如何做分页查询？

**A**: 三种方案

**方案 1**: 禁止跨分片分页（推荐）
```go
// 只查询单个分片
query, _ := db.GetShardingQuery("orders", userID)
query.Where("user_id", userID).Limit(10).Offset(0).Select()
```

**方案 2**: 全局分页（性能差）
```go
// 查询所有分片，内存排序分页
var allData []map[string]interface{}
router.BatchQuery(func(q *db.Query) error {
    data, _ := q.Limit(100).Select()
    allData = append(allData, data...)
    return nil
})
// 内存排序分页
sort.Slice(allData, ...)
pageData := allData[offset:offset+limit]
```

**方案 3**: 二次查询（折中）
```go
// 1. 各分片查询前 N 条
// 2. 排序后取前 limit 条
// 3. 用这些 ID 再次查询完整数据
```

---

### Q3: 分库分表后，如何做聚合查询（COUNT、SUM）？

**A**: 跨分片聚合

```go
var totalCount int64
var mu sync.Mutex

router, _ := GlobalShardingManager.GetRouter("orders")

// 并行查询各分片
router.BatchQuery(func(q *db.Query) error {
    count, err := q.Where("status", "completed").Count()
    if err != nil {
        return err
    }
    
    mu.Lock()
    totalCount += count
    mu.Unlock()
    
    return nil
})

log.Printf("总订单数：%d", totalCount)
```

### Q4: 如何扩容分片？

**A**: 双写迁移方案

```
步骤 1: 修改代码，支持双写
  - 写入旧分片
  - 同时写入新分片

步骤 2: 数据迁移
  - 后台任务同步历史数据
  - 校验数据一致性

步骤 3: 切换读
  - 10% 流量读新分片
  - 50% 流量读新分片
  - 100% 流量读新分片

步骤 4: 切换写
  - 停止双写旧分片
  - 只写新分片

步骤 5: 清理
  - 删除旧分片数据
  - 下线旧分片表
```

### Q5: 分片键如何选择？

**A**: 根据业务查询模式

| 查询模式 | 推荐分片键 |
|----------|-----------|
| 按用户查询订单 | user_id |
| 按订单号查询 | order_no |
| 按商户查询 | merchant_id |
| 按时间范围查询 | created_at（范围分片） |

**原则**:
1. 查询条件中必须包含分片键
2. 避免跨分片查询
3. 数据分布均匀

## 总结

### 核心概念

| 概念 | 说明 |
|------|------|
| **分片键** | 用于路由的字段（user_id、order_id 等） |
| **分片数** | 分片表/库的数量 |
| **分片策略** | 取模、哈希、范围、自定义 |
| **路由器** | 自动路由到对应分片 |

### 使用流程

```
1. 选择分片键 → 2. 配置分片策略 → 3. 创建分片表 → 4. 使用分片查询
```

### 性能对比

| 操作 | 单表 | 分片表 | 提升 |
|------|------|--------|------|
| 插入 | 1000 QPS | 4000 QPS (4 分片) | +300% |
| 查询 | 5000 QPS | 20000 QPS (4 分片) | +300% |
| 单表数据量 | 1000 万 | 250 万 (4 分片) | -75% |

### 注意事项

⚠️ **重要**:
1. 分片键一旦确定，很难修改
2. 避免跨分片查询
3. 提前规划扩容方案
4. 监控各分片数据量
