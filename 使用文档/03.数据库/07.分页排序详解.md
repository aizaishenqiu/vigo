# 分页排序详解

## 简介
分页和排序是数据库查询中最常用的功能，框架提供简洁的分页和排序方法。

## 排序操作

### Order - 单字段排序
```go
// 降序排列
users, err := db.Table("users").
    Order("create_time DESC").
    Select()

// 升序排列
users, err := db.Table("users").
    Order("id ASC").
    Select()
```

### 多字段排序
```go
// 多字段排序
users, err := db.Table("users").
    Order("status ASC").
    Order("create_time DESC").
    Select()
// 生成: ORDER BY status ASC, create_time DESC

// 复杂排序
orders, err := db.Table("orders").
    Order("priority DESC").
    Order("create_time ASC").
    Select()
```

### 排序方向
```go
// ASC - 升序
Order("id ASC")

// DESC - 降序
Order("id DESC")

// 默认升序
Order("id")  // 等同于 Order("id ASC")
```

## 分页操作

### Limit - 限制数量
```go
// 限制返回10条
users, err := db.Table("users").
    Order("id DESC").
    Limit(10).
    Select()
```

### Offset - 偏移量
```go
// 跳过前10条，取后10条
users, err := db.Table("users").
    Order("id DESC").
    Limit(10).
    Offset(10).
    Select()
// 生成: LIMIT 10 OFFSET 10
```

### Page - 分页方法
```go
// 第1页，每页10条
users, err := db.Table("users").
    Order("id DESC").
    Page(1, 10).
    Select()

// 第2页，每页10条
users, err := db.Table("users").
    Order("id DESC").
    Page(2, 10).
    Select()
// 生成: LIMIT 10 OFFSET 10

// 第3页，每页20条
users, err := db.Table("users").
    Order("id DESC").
    Page(3, 20).
    Select()
// 生成: LIMIT 20 OFFSET 40
```

## 分页计算

### 分页公式
```go
// Page(page, pageSize) 内部计算
// LIMIT pageSize OFFSET (page - 1) * pageSize

// 例如：Page(3, 10)
// LIMIT 10 OFFSET 20
```

### 分页信息计算
```go
func GetPagination(page, pageSize, total int64) map[string]interface{} {
    totalPages := total / pageSize
    if total%pageSize > 0 {
        totalPages++
    }
    
    return map[string]interface{}{
        "page":        page,
        "page_size":   pageSize,
        "total":       total,
        "total_pages": totalPages,
        "has_prev":    page > 1,
        "has_next":    page < totalPages,
    }
}
```

## 完整分页示例

### 用户列表分页
```go
func GetUserList(page, pageSize int64) (map[string]interface{}, error) {
    // 查询条件
    query := db.Table("users").Where("status", 1)
    
    // 统计总数
    total, err := query.Count()
    if err != nil {
        return nil, err
    }
    
    // 分页查询
    users, err := query.
        Order("create_time DESC").
        Page(page, pageSize).
        Select()
    if err != nil {
        return nil, err
    }
    
    // 计算分页信息
    totalPages := total / pageSize
    if total%pageSize > 0 {
        totalPages++
    }
    
    return map[string]interface{}{
        "list":        users,
        "page":        page,
        "page_size":   pageSize,
        "total":       total,
        "total_pages": totalPages,
        "has_prev":    page > 1,
        "has_next":    page < totalPages,
    }, nil
}
```

### 订单列表分页
```go
func GetOrderList(userID, page, pageSize int64) (map[string]interface{}, error) {
    query := db.Table("orders o").
        Join("users u", "o.user_id = u.id", "LEFT").
        Field("o.*, u.name as user_name").
        Where("o.user_id", userID).
        Where("o.status", "completed")
    
    // 统计总数
    total, err := query.Count()
    if err != nil {
        return nil, err
    }
    
    // 分页查询
    orders, err := query.
        Order("o.create_time DESC").
        Page(page, pageSize).
        Select()
    if err != nil {
        return nil, err
    }
    
    return map[string]interface{}{
        "list":  orders,
        "total": total,
        "page":  page,
        "page_size": pageSize,
    }, nil
}
```

## 排序最佳实践

### 索引优化
```go
// 确保排序字段有索引
// CREATE INDEX idx_create_time ON users(create_time)

users, err := db.Table("users").
    Order("create_time DESC").  // 使用索引
    Limit(10).
    Select()
```

### 避免全表排序
```go
// 不推荐：无索引排序大数据量
users, err := db.Table("users").
    Order("RAND()").  // 性能差
    Select()

// 推荐：使用索引排序
users, err := db.Table("users").
    Order("id DESC").
    Select()
```

## 注意事项
1. 分页查询建议配合 `Order` 使用，确保结果稳定
2. 大数据量分页时，`OFFSET` 性能会下降，可考虑游标分页
3. 排序字段建议添加索引
4. `Page(1, 10)` 等同于 `Limit(10)`
5. 分页参数应做边界检查，防止非法输入
