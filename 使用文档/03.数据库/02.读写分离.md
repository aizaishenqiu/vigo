# 读写分离

## 简介
框架支持数据库读写分离，查询操作自动路由到从库，写入操作路由到主库，有效提升数据库性能。同时支持多主库写入负载均衡，实现高可用架构。

---

## 一、读写分离配置

### 1.1 配置文件方式

```yaml
database:
  driver: mysql
  host: 192.168.1.10    # 主库（写库）
  port: 3306
  name: vigo
  user: root
  pass: password
  charset: utf8mb4
  
  # 开启读写分离
  rw_split: true
  
  # 从库配置（读取负载均衡）
  reads:
    - host: 192.168.1.20
      port: 3306
      user: reader1
      pass: read-pass-1
      charset: utf8mb4
    - host: 192.168.1.21
      port: 3306
      user: reader2
      pass: read-pass-2
      charset: utf8mb4
```

### 1.2 配置参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| rw_split | bool | 是否开启读写分离 |
| reads | []DBNode | 从库节点列表 |
| reads[].host | string | 从库主机地址 |
| reads[].port | int | 从库端口 |
| reads[].user | string | 从库用户名 |
| reads[].pass | string | 从库密码 |
| reads[].charset | string | 字符集（可选） |

## 二、多主库配置

### 2.1 多主库写入负载均衡

```yaml
database:
  driver: mysql
  host: 192.168.1.10    # 默认主库
  port: 3306
  name: vigo
  user: root
  pass: password
  rw_split: true
  
  # 多主库配置（写入负载均衡）
  writes:
    - host: 192.168.1.10
      port: 3306
      user: writer1
      pass: write-pass-1
      charset: utf8mb4
    - host: 192.168.1.11
      port: 3306
      user: writer2
      pass: write-pass-2
      charset: utf8mb4
  
  # 从库配置
  reads:
    - host: 192.168.1.20
      port: 3306
      user: reader1
      pass: read-pass-1
    - host: 192.168.1.21
      port: 3306
      user: reader2
      pass: read-pass-2
```

## 三、自动读写分离

### 3.1 查询自动路由到从库

```go
// Select 查询自动路由到从库
users, err := db.Table("users").
    Where("status", 1).
    Select()

// Find 查询自动路由到从库
user, err := db.Table("users").
    Where("id", 1).
    Find()

// 聚合查询自动路由到从库
count, err := db.Table("users").Count()
```

### 3.2 写入自动路由到主库

```go
// Insert 自动路由到主库（多主库时轮询）
id, err := db.Table("users").InsertGetId(map[string]interface{}{
    "name":  "张三",
    "email": "zhangsan@example.com",
})

// Update 自动路由到主库
affected, err := db.Table("users").
    Where("id", 1).
    Update(map[string]interface{}{
        "name": "李四",
    })

// Delete 自动路由到主库
affected, err := db.Table("users").
    Where("id", 1).
    Delete()
```

## 四、代码初始化方式

### 4.1 初始化主库

```go
package main

import (
    "log"
    "vigo/framework/db"
)

func main() {
    err := db.Init(
        "mysql",
        "root:password@tcp(master-db:3306)/myapp",
        100,
        10,
        3600,
        300,
    )
    if err != nil {
        log.Fatal("主库连接失败:", err)
    }
}
```

### 4.2 初始化从库

```go
readDSNs := []string{
    "readonly:password@tcp(slave1:3306)/myapp",
    "readonly:password@tcp(slave2:3306)/myapp",
    "readonly:password@tcp(slave3:3306)/myapp",
}

err := db.InitReadDBs("mysql", readDSNs, 50, 5, 1800, 300)
if err != nil {
    log.Printf("部分从库连接失败: %v", err)
}
```

### 4.3 初始化多主库

```go
writeDSNs := []string{
    "writer:password@tcp(master1:3306)/myapp",
    "writer:password@tcp(master2:3306)/myapp",
}

err := db.InitWriteDBs("mysql", writeDSNs, 100, 10, 3600, 300)
if err != nil {
    log.Printf("部分主库连接失败: %v", err)
}
```

## 五、手动指定数据库

### 5.1 获取从库连接

```go
// 获取从库连接（轮询方式）
readDB := db.GetReadDB()

// 使用指定连接查询
users, err := db.NewQuery(readDB).
    Table("users").
    Where("status", 1).
    Select()
```

### 5.2 获取主库连接

```go
// 获取写库连接（多主库时轮询）
writeDB := db.GetWriteDB()

// 强制使用主库查询
users, err := db.NewQuery(writeDB).
    Table("users").
    Where("status", 1).
    Select()
```

### 5.3 使用默认主库

```go
// 强制使用主库查询
users, err := db.NewQuery(db.GlobalDB).
    Table("users").
    Where("status", 1).
    Select()
```

## 六、事务中的读写分离

### 6.1 事务使用主库

```go
err := db.Transaction(func(q *db.Query) error {
    // 插入数据（主库）
    id, err := q.Table("orders").InsertGetId(map[string]interface{}{
        "user_id": 1,
        "amount":  100,
    })
    if err != nil {
        return err
    }
    
    // 查询数据（同一事务中使用主库）
    order, err := q.Table("orders").Where("id", id).Find()
    if err != nil {
        return err
    }
    
    return nil
})
```

## 七、从库负载均衡

### 7.1 轮询策略

框架使用原子计数器实现从库轮询：

```go
// GetReadDB 获取只读连接 (原子轮询)
func GetReadDB() *sql.DB {
    n := len(ReadDBs)
    if n == 0 {
        return GlobalDB  // 没有从库时返回主库
    }
    idx := atomic.AddUint64(&readCounter, 1) % uint64(n)
    return ReadDBs[idx]
}
```

### 7.2 多主库轮询

```go
// GetWriteDB 获取写库连接 (多主库轮询)
func GetWriteDB() *sql.DB {
    if len(WriteDBs) == 0 {
        return GlobalDB
    }
    idx := atomic.AddUint64(&writeCounter, 1) % uint64(len(WriteDBs))
    return WriteDBs[idx]
}
```

## 八、配置建议

### 8.1 主库配置

```yaml
# 主库配置
database:
  max_open_conns: 100      # 较大值，支持写入并发
  max_idle_conns: 20       # 保持一定空闲连接
  conn_max_lifetime: 3600  # 1小时
  conn_max_idletime: 300   # 5分钟
```

### 8.2 从库配置

```yaml
# 从库配置（每个从库独立配置）
  max_open_conns: 50       # 每个从库连接数
  max_idle_conns: 10       # 空闲连接数
  conn_max_lifetime: 1800  # 30分钟
  conn_max_idletime: 300   # 5分钟
```

### 8.3 高可用配置

```yaml
# 高可用配置（推荐）
database:
  rw_split: true
  
  # 双主库
  writes:
    - host: 192.168.1.10
      port: 3306
      user: writer
      pass: write-pass
    - host: 192.168.1.11
      port: 3306
      user: writer
      pass: write-pass
  
  # 三从库
  reads:
    - host: 192.168.1.20
      port: 3306
      user: reader
      pass: read-pass
    - host: 192.168.1.21
      port: 3306
      user: reader
      pass: read-pass
    - host: 192.168.1.22
      port: 3306
      user: reader
      pass: read-pass
```

## 九、注意事项

1. 查询操作自动路由到从库，写入操作自动路由到主库
2. 事务中的所有操作都使用主库连接
3. 没有配置从库时，所有操作都使用主库
4. 从库故障时会自动跳过，不影响主库操作
5. 注意主从延迟问题，关键数据应从主库读取
6. 建议配置多个从库，实现负载均衡和高可用
7. 多主库写入时注意数据同步问题
8. 生产环境建议至少配置 2 主 2 从
