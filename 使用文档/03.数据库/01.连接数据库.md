# 连接数据库

## 简介
框架支持 MySQL、PostgreSQL、SQLite、SQLServer 等多种数据库驱动，通过简单的配置即可建立数据库连接。支持读写分离、多主库写入、多数据库连接等高级特性。

---

## 一、数据库配置

### 1.1 配置文件

在项目根目录的 `config.yaml` 中配置数据库连接：

```yaml
database:
  driver: mysql
  host: ${DB_HOST}
  port: ${DB_PORT:3306}
  name: ${DB_NAME}
  user: ${DB_USER}
  pass: ${DB_PASSWORD}
  charset: utf8mb4
  max_open_conns: 200
  max_idle_conns: 100
  conn_max_lifetime: 3600
  conn_max_idletime: 300
```

### 1.2 配置参数说明

| 参数 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| driver | string | 数据库驱动：mysql / postgres / sqlite / mssql | mysql |
| host | string | 数据库主机地址 | 127.0.0.1 |
| port | int | 数据库端口 | 3306 |
| name | string | 数据库名称 | - |
| user | string | 数据库用户名 | - |
| pass | string | 数据库密码 | - |
| charset | string | 字符集 | utf8mb4 |
| max_open_conns | int | 最大打开连接数 | 100 |
| max_idle_conns | int | 最大空闲连接数 | 10 |
| conn_max_lifetime | int | 连接最大生命周期（秒） | 3600 |
| conn_max_idletime | int | 空闲连接最大存活时间（秒） | 300 |

### 1.3 环境变量配置

使用环境变量管理敏感信息：

```bash
# 必需环境变量
export DB_HOST=127.0.0.1
export DB_PORT=3306
export DB_USER=vigo
export DB_PASSWORD=your_password
export DB_NAME=vigo

# JWT 密钥
export JWT_SECRET=your_jwt_secret_at_least_32_characters
```

## 二、连接池配置

### 2.1 连接池参数详解

| 参数 | 说明 | 单机推荐 | 分布式推荐 |
|------|------|----------|------------|
| max_open_conns | 最大打开连接数 | 500-1000 | 300-500 |
| max_idle_conns | 最大空闲连接数 | 100-200 | 50-100 |
| conn_max_lifetime | 连接最大生命周期（秒） | 1800-3600 | 1800 |
| conn_max_idletime | 空闲连接回收时间（秒） | 300 | 300 |

### 2.2 不同场景配置建议

#### 开发环境
```yaml
database:
  max_open_conns: 50
  max_idle_conns: 10
  conn_max_lifetime: 1800
  conn_max_idletime: 300
```

#### 生产环境（单机）
```yaml
database:
  max_open_conns: 500
  max_idle_conns: 100
  conn_max_lifetime: 3600
  conn_max_idletime: 300
```

#### 生产环境（高并发）
```yaml
database:
  max_open_conns: 1000
  max_idle_conns: 200
  conn_max_lifetime: 1800
  conn_max_idletime: 300
```

#### 生产环境（分布式）
```yaml
database:
  max_open_conns: 300
  max_idle_conns: 50
  conn_max_lifetime: 1800
  conn_max_idletime: 300
```

## 三、不同数据库驱动

### 3.1 MySQL

```yaml
database:
  driver: mysql
  host: 127.0.0.1
  port: 3306
  name: myapp
  user: root
  pass: "123456"
  charset: utf8mb4
```

DSN 格式：
```go
// username:password@tcp(host:port)/database?charset=utf8mb4&parseTime=True&loc=Local
dsn := "root:123456@tcp(127.0.0.1:3306)/myapp?charset=utf8mb4&parseTime=True&loc=Local"
```

### 3.2 PostgreSQL

```yaml
database:
  driver: postgres
  host: 127.0.0.1
  port: 5432
  name: myapp
  user: postgres
  pass: "123456"
```

DSN 格式：
```go
// postgres://username:password@host:port/database?sslmode=disable
dsn := "postgres://postgres:123456@127.0.0.1:5432/myapp?sslmode=disable"
```

### 3.3 SQLite

```yaml
database:
  driver: sqlite
  name: ./data/myapp.db
```

DSN 格式：
```go
// 文件路径
dsn := "./data/myapp.db"
```

### 3.4 SQL Server

```yaml
database:
  driver: mssql
  host: 127.0.0.1
  port: 1433
  name: myapp
  user: sa
  pass: "123456"
```

DSN 格式：
```go
// sqlserver://username:password@host:port/database
dsn := "sqlserver://sa:123456@127.0.0.1:1433/myapp"
```

## 四、代码中使用

### 4.1 初始化连接

```go
package main

import (
    "log"
    "vigo/framework/db"
)

func main() {
    err := db.Init(
        "mysql",
        "root:123456@tcp(127.0.0.1:3306)/myapp?charset=utf8mb4",
        100,
        10,
        3600,
        300,
    )
    if err != nil {
        log.Fatal("数据库连接失败:", err)
    }
    
    log.Println("数据库连接成功")
}
```

### 4.2 测试连接

```go
err := db.GlobalDB.Ping()
if err != nil {
    log.Fatal("数据库连接测试失败:", err)
}
log.Println("数据库连接正常")
```

### 4.3 快捷方法

```go
// 获取查询构造器
query := db.Table("users")

// 别名
query := db.Name("users")

// 执行原生查询
results, err := db.QuerySQL("SELECT * FROM users WHERE id = ?", 1)

// 执行原生SQL
affected, err := db.ExecuteSQL("UPDATE users SET status = ? WHERE id = ?", 1, 1)

// 事务
err := db.Transaction(func(q *db.Query) error {
    // 事务操作
    return nil
})
```

## 五、全局变量

框架提供以下全局变量：

```go
// 全局数据库连接（主库/写库）
db.GlobalDB *sql.DB

// 当前数据库驱动
db.CurrentDriver string  // "mysql" / "postgres" / "sqlite" / "mssql"

// 只读数据库连接池（从库）
db.ReadDBs []*sql.DB

// 多写库连接池
db.WriteDBs []*sql.DB
```

## 六、注意事项

1. 数据库连接应在应用启动时初始化
2. 连接池参数应根据实际负载调整
3. 生产环境建议使用环境变量管理敏感信息
4. 定期监控连接池状态，避免连接泄漏
5. 使用预处理语句防止SQL注入
6. 高并发场景适当增大连接池
7. 分布式部署时每个实例独立配置连接池
