# 事务操作详解

## 简介

事务用于保证多个数据库操作的原子性，要么全部成功，要么全部回滚，确保数据一致性。

## 基本事务

### Transaction - 自动事务

```go
// 自动事务，成功自动提交，失败自动回滚
err := db.Transaction(func(q *db.Query) error {
    // 扣减用户余额
    _, err := q.Table("users").
        Where("id", 1).
        Dec("balance", 100)
    if err != nil {
        return err  // 返回错误会自动回滚
    }

    // 增加订单
    _, err = q.Table("orders").InsertGetId(map[string]interface{}{
        "user_id": 1,
        "amount":  100,
        "status":  "pending",
    })
    if err != nil {
        return err
    }

    return nil  // 返回 nil 会自动提交
})

if err != nil {
    log.Printf("事务失败: %v", err)
}
```

### 手动事务

```go
// 开启事务（使用 GlobalDB.Begin()）
tx, err := db.GlobalDB.Begin()
if err != nil {
    log.Fatal(err)
}

// 使用事务对象执行原生 SQL
_, err = tx.Exec("UPDATE users SET balance = balance - ? WHERE id = ?", 100, 1)
if err != nil {
    tx.Rollback()  // 回滚
    log.Fatal(err)
}

_, err = tx.Exec("INSERT INTO orders (user_id, amount) VALUES (?, ?)", 1, 100)
if err != nil {
    tx.Rollback()  // 回滚
    log.Fatal(err)
}

// 提交事务
err = tx.Commit()
if err != nil {
    log.Fatal(err)
}
```

> **注意**: 推荐使用 `db.Transaction()` 方法，它会自动处理事务的开启、提交和回滚，代码更简洁可靠。

## 事务方法

### Transaction - 自动事务（推荐）

```go
// 自动管理事务，成功提交，失败回滚
err := db.Transaction(func(q *db.Query) error {
    // 在这里执行数据库操作
    return nil  // 返回 nil 提交，返回 error 回滚
})
```

### GlobalDB.Begin - 手动开启事务

```go
tx, err := db.GlobalDB.Begin()
if err != nil {
    log.Fatal("开启事务失败:", err)
}
```

### Commit - 提交事务

```go
err := tx.Commit()
if err != nil {
    log.Fatal("提交事务失败:", err)
}
```

### Rollback - 回滚事务

```go
err := tx.Rollback()
if err != nil {
    log.Fatal("回滚事务失败:", err)
}
```

## 完整示例

### 转账事务

```go
func Transfer(fromID, toID int64, amount float64) error {
    return db.Transaction(func(q *db.Query) error {
        // 检查余额
        user, err := q.Table("users").Where("id", fromID).Find()
        if err != nil {
            return err
        }
        if user == nil {
            return fmt.Errorf("转出用户不存在")
        }

        balance, _ := user["balance"].(float64)
        if balance < amount {
            return fmt.Errorf("余额不足")
        }

        // 扣减转出方余额
        affected, err := q.Table("users").
            Where("id", fromID).
            Where("balance", ">=", amount).
            Dec("balance", amount)
        if err != nil {
            return err
        }
        if affected == 0 {
            return fmt.Errorf("扣减余额失败")
        }

        // 增加接收方余额
        affected, err = q.Table("users").
            Where("id", toID).
            Inc("balance", amount)
        if err != nil {
            return err
        }
        if affected == 0 {
            return fmt.Errorf("接收用户不存在")
        }

        // 记录转账日志
        _, err = q.Table("transfer_logs").Insert(map[string]interface{}{
            "from_id": fromID,
            "to_id":   toID,
            "amount":  amount,
            "create_time": time.Now().Format("2006-01-02 15:04:05"),
        })

        return err
    })
}
```

### 创建订单事务

```go
func CreateOrder(userID, productID int64, quantity int) (int64, error) {
    var orderID int64

    err := db.Transaction(func(q *db.Query) error {
        // 查询商品库存
        product, err := q.Table("products").
            Where("id", productID).
            Lock().
            Find()
        if err != nil {
            return err
        }
        if product == nil {
            return fmt.Errorf("商品不存在")
        }

        stock, _ := product["stock"].(int64)
        if stock < int64(quantity) {
            return fmt.Errorf("库存不足")
        }

        // 创建订单
        id, err := q.Table("orders").InsertGetId(map[string]interface{}{
            "user_id":    userID,
            "product_id": productID,
            "quantity":   quantity,
            "amount":     product["price"].(float64) * float64(quantity),
            "status":     "pending",
            "create_time": time.Now().Format("2006-01-02 15:04:05"),
        })
        if err != nil {
            return err
        }
        orderID = id

        // 扣减库存
        affected, err := q.Table("products").
            Where("id", productID).
            Where("stock", ">=", quantity).
            Dec("stock", quantity)
        if err != nil {
            return err
        }
        if affected == 0 {
            return fmt.Errorf("扣减库存失败")
        }

        return nil
    })

    return orderID, err
}
```

## 事务隔离级别

### 设置隔离级别

```go
// MySQL 默认隔离级别为 REPEATABLE READ
// 可通过原生SQL设置

tx, _ := db.GlobalDB.Begin()
tx.Exec("SET TRANSACTION ISOLATION LEVEL READ COMMITTED")
// 执行事务操作
tx.Commit()
```

### 常见隔离级别

- `READ UNCOMMITTED` - 读未提交
- `READ COMMITTED` - 读已提交
- `REPEATABLE READ` - 可重复读（MySQL默认）
- `SERIALIZABLE` - 串行化

## 注意事项

1. 事务操作自动使用主库连接
2. 事务中不要执行耗时操作，避免长事务
3. 事务中查询建议使用 `Lock()` 防止并发问题
4. 返回错误会自动回滚，返回 nil 会自动提交
5. 手动事务必须显式调用 Commit 或 Rollback
6. 建议使用 `Transaction` 方法，避免忘记提交或回滚
