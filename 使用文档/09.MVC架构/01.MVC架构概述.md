# MVC架构概述

## 简介
框架采用经典的MVC（Model-View-Controller）架构模式，将应用程序分为三个核心组件：

- **Model（模型）**：负责数据访问和业务逻辑
- **View（视图）**：负责页面展示和用户界面
- **Controller（控制器）**：负责接收请求、协调模型和视图

## 目录结构

```
app/
├── controller/          # 控制器目录
│   ├── user.go         # 用户控制器
│   └── order.go        # 订单控制器
├── model/              # 模型目录
│   ├── user.go         # 用户模型
│   └── order.go        # 订单模型
└── view/               # 视图目录
    ├── layouts/        # 布局模板
    ├── partials/       # 公共组件
    └── user/           # 用户页面
```

## MVC工作流程

```
用户请求 → 路由(Router) → 控制器(Controller) → 模型(Model) → 数据库
                                                    ↓
用户响应 ← 视图(View) ← 控制器(Controller) ←─────────┘
```

## 核心组件

### Router（路由器）
路由器负责将HTTP请求分发到对应的控制器方法。

```go
// 创建路由器
router := mvc.NewRouter()

// 注册路由
router.GET("/users", userController.List)
router.POST("/users", userController.Create)
router.GET("/users/:id", userController.Detail)
router.PUT("/users/:id", userController.Update)
router.DELETE("/users/:id", userController.Delete)

// 路由分组
api := router.Group("/api")
api.GET("/users", userController.List)
api.POST("/users", userController.Create)

// 启动服务
router.Run(":8080")
```

### Controller（控制器）
控制器接收请求，调用模型处理数据，返回响应。

```go
// 用户控制器
type UserController struct {
    mvc.Controller
}

// 用户列表
func (c *UserController) List(ctx *mvc.Context) {
    page := ctx.InputInt("page", 1)
    pageSize := ctx.InputInt("page_size", 10)
    
    // 调用模型获取数据
    users, total, _ := userModel.GetList(page, pageSize)
    
    // 返回JSON响应
    ctx.Success(map[string]interface{}{
        "list": users,
        "total": total,
    })
}
```

### Model（模型）
模型负责数据访问和业务逻辑。

```go
// 用户模型
type UserModel struct {
    db *db.Query
}

func (m *UserModel) GetList(page, pageSize int) ([]map[string]interface{}, int64, error) {
    total, _ := m.db.Table("users").Count()
    list, _ := m.db.Table("users").
        Page(page, pageSize).
        Select()
    return list, total, nil
}
```

### View（视图）
视图负责渲染页面模板。

```go
// 渲染视图
func (c *UserController) Detail(ctx *mvc.Context) {
    id := ctx.Param("id")
    user, _ := userModel.FindByID(id)
    
    // 渲染模板
    ctx.HTML(200, "user/detail.html", map[string]interface{}{
        "Title": "用户详情",
        "User":  user,
    })
}
```

## Context上下文

Context是请求上下文，提供丰富的请求处理方法：

### 请求数据获取
```go
// 获取参数
ctx.Input("name")                    // 获取请求参数
ctx.InputInt("page", 1)              // 获取整型参数，带默认值
ctx.Query("keyword")                 // 获取URL查询参数
ctx.Param("id")                      // 获取路由参数
ctx.Cookie("token")                  // 获取Cookie
ctx.GetHeader("Authorization")       // 获取请求头
ctx.GetClientIP()                    // 获取客户端IP
ctx.BindJSON(&user)                  // 绑定JSON到结构体
```

### 响应输出
```go
ctx.Success(data)                    // 成功响应
ctx.Error(400, "参数错误")            // 错误响应
ctx.Json(200, data)                  // JSON响应
ctx.HTML(200, "template.html", data) // HTML响应
ctx.String(200, "Hello %s", name)    // 文本响应
ctx.Redirect(302, "/login")          // 重定向
```

### 中间件控制
```go
ctx.Next()                           // 执行下一个中间件
ctx.Abort()                          // 中止中间件链
ctx.IsAborted()                      // 检查是否已中止
```

### 上下文存储
```go
ctx.Set("user_id", 123)              // 存储值
ctx.Get("user_id")                   // 获取值
```

## 中间件

中间件用于处理横切关注点，如认证、日志、跨域等。

### 定义中间件
```go
// 认证中间件
func AuthMiddleware(ctx *mvc.Context) {
    token := ctx.GetHeader("Authorization")
    if token == "" {
        ctx.Error(401, "未授权访问")
        ctx.Abort()
        return
    }
    
    // 验证token
    userID, err := validateToken(token)
    if err != nil {
        ctx.Error(401, "无效的token")
        ctx.Abort()
        return
    }
    
    ctx.Set("user_id", userID)
    ctx.Next()
}
```

### 注册中间件
```go
// 全局中间件
router.Use(AuthMiddleware, LogMiddleware)

// 路由分组中间件
api := router.Group("/api", AuthMiddleware)
api.GET("/profile", userController.Profile)
```

## 完整示例

### main.go
```go
package main

import (
    "vigo/framework/db"
    "vigo/framework/mvc"
    "vigo/framework/view"
    "myapp/controller"
)

func main() {
    // 初始化数据库
    db.Init("mysql", "root:pass@tcp(localhost:3306)/myapp", 100, 10, 3600)
    
    // 创建路由器
    router := mvc.NewRouter()
    
    // 设置视图引擎
    engine := view.NewTemplateEngine("app/view")
    router.SetViewEngine(engine)
    
    // 全局中间件
    router.Use(middleware.CORS, middleware.Logger)
    
    // 注册路由
    registerRoutes(router)
    
    // 启动服务
    router.Run(":8080")
}

func registerRoutes(router *mvc.Router) {
    // 静态资源
    router.Handle("/static/", http.StripPrefix("/static/", 
        http.FileServer(http.Dir("public"))))
    
    // API路由
    api := router.Group("/api")
    {
        api.GET("/users", controller.User.List)
        api.POST("/users", controller.User.Create)
        api.GET("/users/:id", controller.User.Detail)
        api.PUT("/users/:id", controller.User.Update)
        api.DELETE("/users/:id", controller.User.Delete)
    }
    
    // 需要认证的路由
    auth := router.Group("/api", middleware.Auth)
    {
        auth.GET("/profile", controller.User.Profile)
        auth.PUT("/profile", controller.User.UpdateProfile)
    }
    
    // 页面路由
    router.GET("/", controller.Page.Home)
    router.GET("/users", controller.Page.UserList)
    router.GET("/users/:id", controller.Page.UserDetail)
}
```

## 注意事项
1. 控制器保持轻量，复杂业务逻辑放在模型或服务层
2. 使用中间件处理横切关注点
3. 路由参数使用 `:id` 格式定义
4. Context提供完整的请求处理方法
5. 视图模板放在统一目录便于管理
