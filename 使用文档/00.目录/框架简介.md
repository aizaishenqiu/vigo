# Vigo 框架简介 v2.0

## 什么是 Vigo？

**Vigo** 是一个基于 Go 1.21+ 构建的现代化企业级 SaaS 开发框架，提供完整的 MVC 架构、ThinkPHP/Laravel 风格 ORM、微服务支持、实时通讯等能力。

> "大道至简，开发由我"

**版本**: 2.0.0 | **最后更新**: 2026-03-05

---

## 核心理念

Vigo 的设计哲学可以概括为三个词：**简单**、**高效**、**安全**。

- **简单**: 提供 ThinkPHP/Laravel 风格的 ORM，降低学习成本，让开发者快速上手
- **高效**: 极致性能优化，单机支持 10 万 + 并发，QPS 15 万+
- **安全**: 内置全方位安全防护，让应用更安全可靠

---

## 核心特性

### 🚀 高性能

- **10 万 + 并发**: 经过压力测试验证，单机支持 10 万 + 并发连接
- **QPS 15 万+**: 优化的路由匹配和上下文复用机制
- **连接池优化**: 数据库、Redis、gRPC 连接池深度优化，支持连接复用
- **零拷贝技术**: 减少内存分配，提升性能
- **异步处理**: 支持异步任务和事件驱动
- **内存池化**: `sync.Pool` 复用上下文，减少 GC 压力
- **HTTP/2 支持**: 多路复用，性能提升 2x

### 🏗️ MVC 架构

- **清晰的分层**: Controller、Model、View 分层明确，职责清晰
- **路由系统**: 灵活的路由定义，支持 RESTful 风格、路由组
- **中间件**: 请求/响应中间件链，支持全局和路由级中间件
- **依赖注入**: IoC 容器支持，代码更灵活、可测试
- **视图引擎**: 支持 Go 原生模板，可选 Blade 引擎

### 💾 数据库 ORM

- **ThinkPHP/Laravel 风格**: 熟悉的链式操作，降低学习成本
- **读写分离**: 自动路由查询到从库，写入到主库
- **多主库**: 支持多主库写入负载均衡
- **多数据库**: 支持多业务独立数据库，跨库操作
- **多驱动**: MySQL、PostgreSQL、SQLite、SQL Server
- **事务支持**: 自动/手动事务，嵌套事务
- **模型事件**: before/after 插入、更新、删除事件回调
- **软删除**: 支持软删除，数据更安全
- **自动时间戳**: 自动维护创建/更新时间
- **模型关联**: HasOne/HasMany/BelongsToMany

### 🔒 安全防护

- **XSS 防护**: 自动转义危险字符，防止跨站脚本攻击
- **SQL 注入防护**: 预处理语句，拦截 99% 注入攻击
- **CSRF 防护**: Token 验证机制，防止跨站请求伪造
- **速率限制**: 防 DDoS 攻击，支持单 IP 限流、分布式限流
- **JWT 认证**: 无状态身份验证，支持 Token 刷新
- **安全头**: 自动添加安全相关的 HTTP 头

### 🌐 微服务支持

- **gRPC**: 高性能 RPC 框架，支持流式传输
- **Nacos**: 服务注册与发现、配置中心、健康检查
- **RabbitMQ**: 消息队列，支持发布/订阅、延迟队列
- **API 网关**: 统一入口管理，支持限流、鉴权
- **服务拆分**: 支持按业务拆分，独立部署
- **负载均衡**: 随机/轮询/最少连接/加权
- **gRPC 连接池**: 连接复用，性能提升 5x

### 📊 监控与管理

- **Web 管理界面**: 可视化管理工具，无需额外部署
- **压力测试**: 内置压测工具，支持 HTTP/Redis/MySQL/MQ 压测
- **性能监控**: CPU、内存、磁盘、网络实时监控大屏
- **日志系统**: 分级日志、按日期分割、支持日志轮转
- **健康检查**: 微服务探活接口，支持基础/详细检查
- **Prometheus 指标**: 系统指标收集，Grafana 集成

## 快速开始

### 1. 安装

```bash
git clone https://github.com/yourusername/vigo.git
cd vigo
go mod tidy
```

### 2. 配置

```bash
cp config.yaml config.local.yaml
# 编辑 config.local.yaml，配置数据库、Redis 等
```

### 3. 运行

```bash
go run main.go
```

### 4. 访问

- 应用主页：http://localhost:8080
- 系统监控：http://localhost:8080/monitor
- 迁移管理：http://localhost:8080/migration ⭐新增
- API 文档：http://localhost:8080/docs
