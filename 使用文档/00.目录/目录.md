# Vigo 开发手册

**Vigo** 是一个基于 Go 语言构建的现代化企业级 SaaS 开发框架，提供完整的 MVC 架构、ThinkPHP/Laravel 风格 ORM、微服务支持、实时通讯等能力。

> "大道至简，开发由我"

**版本**: 2.0.0 | **最后更新**: 2026-03-05

### 数据库 ORM 重点

框架提供 ThinkPHP/Laravel 风格的完整 ORM 能力：

- **查询构造器**: Where / WhereIn / WhereBetween / WhereNull / WhereLike / WhereRaw / WhereMap
- **链式操作**: Field / Order / Limit / Page / Group / Having / Join / Distinct / Lock
- **聚合查询**: Count / Sum / Avg / Max / Min / Exists
- **高级功能**: Inc(自增) / Dec(自减) / InsertAll(批量) / GetLastSql
- **事务支持**: Transaction / Begin / Commit / Rollback
- **读写分离**: 自动路由查询到从库、写入到主库
- **多数据库**: 支持多业务独立数据库、跨库操作

## 模块总览

```
framework/
├── app/          应用生命周期管理 (初始化、启动、优雅停机)
├── balancer/     负载均衡器 (随机/轮询/最少连接/加权) ⭐新增
├── cache/        缓存组件 (内存/文件/BigCache)
├── circuit/      熔断器 (自适应熔断)
├── cli/          命令行工具 (数据迁移、框架 CLI) ⭐新增
├── config/       配置中心 (Nacos Config) ⭐新增
├── container/    IoC 容器 (依赖注入)
├── db/           数据库 (查询构造器、读写分离、多驱动、多数据库、数据迁移)
├── discovery/    服务发现 (Nacos Discovery) ⭐新增
├── event/        事件系统 (监听与异步触发)
├── facade/       门面模式 (Redis/RabbitMQ/Nacos 快捷访问)
├── grpc/         gRPC 微服务 (健康检查、拦截器)
├── grpcpool/     gRPC 连接池 (连接复用、自动扩缩容) ⭐新增
├── http2/        HTTP/2 支持 (h2c/h2) ⭐新增
├── json/         JSON 序列化 (json-iterator 优化)
├── limiter/      限流器 (令牌桶/漏桶/滑动窗口/固定窗口)
├── log/          日志系统 (分级、按日期分割)
├── metrics/      指标监控 (Prometheus) ⭐新增
├── middleware/   中间件 (安全防护、请求日志)
├── model/        ORM 模型 (Active Record、关联、事件)
├── mvc/          MVC 核心 (Router、Context、Controller、View)
├── nacos/        Nacos 客户端 (配置管理、服务发现)
├── pool/         对象池 (Context/Buffer 池)
├── port/         端口管理 (检测、释放)
├── radixtree/    Radix Tree 路由 (高性能路由) ⭐新增
├── rabbitmq/     RabbitMQ 客户端 (发布、消费、管理)
├── redis/        Redis 客户端
├── trace/        链路追踪 (OpenTelemetry) ⭐新增
├── validate/     验证器 (链式验证规则)
├── view/         模板引擎 (支持 go:embed 嵌入)
└── websocket/    WebSocket 服务 (Hub、频道、指令)
```
