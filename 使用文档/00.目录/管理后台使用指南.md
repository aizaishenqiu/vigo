# Vigo 管理后台使用指南

## 🎯 功能概述

Vigo 管理后台是一个现代化的、响应式的后端管理系统，采用侧边栏导航设计，提供直观的配置管理界面。

## 🚀 访问方式

- **管理后台地址**: `http://localhost:8080/admin`
- **原系统设置页面**: `http://localhost:8080/settings`

## 📋 功能模块

### 1. 控制台（Dashboard）
- 系统状态概览
- 快速操作入口
- 实时统计数据

### 2. 系统设置
- **应用配置**: 修改应用名称、端口、运行模式
- **数据库配置**: 配置主数据库连接信息
- **Redis 配置**: 配置 Redis 缓存服务
- **安全配置**: 启用/禁用安全中间件、CSRF 保护、速率限制

### 3. 服务配置
- **RabbitMQ**: 消息队列配置
- **Nacos**: 配置中心和服务发现
- **gRPC**: gRPC 服务配置

### 4. 支付配置
- **支付宝**: 支付宝支付参数配置
- **微信支付**: 微信支付参数配置

### 5. 压测配置
- 内存限制百分比
- CPU 限制百分比

## 🎨 界面特点

### 现代化设计
- 侧边栏导航，支持多级菜单
- 卡片式布局，清晰的信息层级
- 响应式设计，适配不同屏幕尺寸

### 配色方案
- 主色调：#409EFF（科技蓝）
- 成功色：#67C23A（绿色）
- 警告色：#E6A23C（橙色）
- 危险色：#F56C6C（红色）

### 交互体验
- 菜单展开/收起动画
- 页面切换流畅过渡
- 表单输入实时反馈

## 🔧 技术栈

- **前端**: 原生 HTML5 + CSS3 + JavaScript
- **图标**: Font Awesome 6.4.0
- **后端**: Go + Vigo Framework
- **架构**: MVC 模式

## 📝 使用示例

### 保存应用配置
1. 点击侧边栏 "系统设置" → "应用配置"
2. 修改应用名称、端口等参数
3. 点击 "保存配置" 按钮
4. 系统提示保存结果

### 配置数据库
1. 点击侧边栏 "系统设置" → "数据库配置"
2. 填写数据库连接信息
3. 点击 "保存配置"
4. 系统会自动更新 config.yaml 文件

## 💡 最佳实践

1. **修改配置前备份**: 重要配置修改前建议先备份 config.yaml
2. **测试环境验证**: 生产环境修改前先在测试环境验证
3. **逐步生效**: 部分配置需要重启服务才能生效
4. **权限控制**: 建议添加管理员登录验证

## 🔐 安全建议

1. 启用 CSRF 保护
2. 启用速率限制防止暴力破解
3. 生产环境启用 CORS 域名验证
4. 配置 IP 白名单/黑名单
5. 定期更新管理员密码
