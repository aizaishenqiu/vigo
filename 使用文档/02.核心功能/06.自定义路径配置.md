# 自定义路径配置详解

Vigo 框架提供灵活的路径配置系统，支持自定义应用目录结构，满足不同的项目组织需求。

---

## 一、路径常量

框架提供 ThinkPHP 风格的路径常量，可以在全局任意位置使用：

### 1.1 内置路径常量

```go
// 框架自动初始化的路径常量
ROOT_PATH   string  // 项目根目录
APP_PATH    string  // 应用目录
VIEW_PATH   string  // 视图目录
CONFIG_PATH string  // 配置目录
LIB_PATH    string  // 类库目录
RUN_PATH    string  // 缓存目录
```

### 1.2 使用示例

```go
package controller

import (
    "vigo/framework/config"
)

func Index(c *mvc.Context) {
    // 获取项目根目录
    root := config.ROOT_PATH
    // /home/www/vigo

    // 获取应用目录
    app := config.APP_PATH
    // /home/www/vigo/app

    // 获取视图目录
    view := config.VIEW_PATH
    // /home/www/vigo/app/view

    // 获取配置目录
    conf := config.CONFIG_PATH
    // /home/www/vigo/config

    // 读取文件
    content, _ := os.ReadFile(config.APP_PATH + "/data/config.json")

    // 保存缓存
    os.WriteFile(config.RUN_PATH + "/cache/data.txt", []byte("data"), 0644)
}
```

## 二、自定义路径配置

### 2.1 配置应用目录

在 `config.yaml` 中自定义应用目录：

```yaml
app:
  # 应用名称
  name: vigo

  # 应用目录（相对于项目根目录）
  # 默认：app
  # 可以自定义为：application, src, backend 等
  app_path: app

  # 视图目录（相对于应用目录）
  # 默认：view
  # 可以自定义为：views, template, templates 等
  view_path: view

  # 运行时目录（相对于项目根目录）
  # 默认：runtime
  # 可以自定义为：storage, cache, tmp 等
  runtime_path: runtime
```

### 2.2 完整示例

```yaml
# ==============================================
# 应用基础配置
# ==============================================
app:
  # 应用信息
  name: my-application
  version: 1.0.0
  env: development
  debug: true

  # 路径配置
  app_path: application # 应用目录名
  view_path: templates # 视图目录名
  runtime_path: storage # 运行时目录名

  # 服务配置
  host: 0.0.0.0
  port: 8000
  read_timeout: 30
  write_timeout: 30
```

### 2.3 目录结构示例

#### 默认结构

```

├── app/              # 默认应用目录
│   ├── common/       # 公共方法
```

## 三、环境变量覆盖

可以使用环境变量覆盖配置文件中的路径设置：

```bash
# .env 文件
VIGO_APP_PATH=application
VIGO_VIEW_PATH=templates
VIGO_RUNTIME_PATH=storage
```

或在启动时设置：

```bash
# Linux
export VIGO_APP_PATH=application
export VIGO_VIEW_PATH=templates
./vigo

# Windows
set VIGO_APP_PATH=application
set VIGO_VIEW_PATH=templates
vigo.exe
```

## 四、路径工具函数

### 4.1 路径拼接

```go
// 使用 filepath 包安全拼接路径
configPath := filepath.Join(config.ROOT_PATH, "config", "database.yaml")
logFile := filepath.Join(config.RUN_PATH, "logs", "app.log")
cacheFile := filepath.Join(config.RUN_PATH, "cache", "data.cache")
```

### 4.2 路径检查

```go
// 检查目录是否存在
if _, err := os.Stat(config.RUN_PATH); os.IsNotExist(err) {
    // 创建目录
    os.MkdirAll(config.RUN_PATH, 0755)
}

// 检查文件是否存在
if _, err := os.Stat(filepath.Join(config.VIEW_PATH, "index.html")); err == nil {
    // 文件存在
}
```

### 4.3 获取绝对路径

```go
// 将相对路径转换为绝对路径
absPath, _ := filepath.Abs(config.APP_PATH)
```

## 五、最佳实践

### 5.1 目录命名规范

- **小写字母**: 使用小写字母命名目录
- **下划线分隔**: 使用下划线分隔单词（如：`app_path`）
- **语义化**: 目录名应清晰表达用途

### 5.2 路径安全

- 使用 `filepath.Join()` 拼接路径，避免手动拼接
- 检查路径是否存在
- 确保目录有正确的权限

### 5.3 跨平台兼容

```go
// 错误：使用硬编码的路径分隔符
path := config.ROOT_PATH + "/config/database.yaml"  // Windows 不兼容

// 正确：使用 filepath.Join
path := filepath.Join(config.ROOT_PATH, "config", "database.yaml")
```

## 六、常见问题

### Q1: 如何修改默认的应用目录？

在 `config.yaml` 中设置：

```yaml
app:
  app_path: application # 改为 application 目录
```

### Q2: 视图文件放在哪里？

视图文件放在 `VIEW_PATH` 目录下：

```
{VIEW_PATH}/
├── index.html
├── user/
│   ├── list.html
│   └── detail.html
└── layout/
    └── main.html
```

### Q3: 如何获取当前运行环境的完整路径？

```go
// 获取完整路径
pwd, _ := os.Getwd()
fmt.Println("工作目录:", pwd)

fmt.Println("根目录:", config.ROOT_PATH)
fmt.Println("应用目录:", config.APP_PATH)
```

### Q4: 路径配置不生效怎么办？

1. 检查 `config.yaml` 中的配置是否正确
2. 检查环境变量是否覆盖配置
3. 重启应用使配置生效
4. 检查目录权限
