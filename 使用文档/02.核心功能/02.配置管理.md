# ⚙️ 配置管理

框架使用 `config.yaml` 文件进行全局配置管理。YAML 提供了清晰的层级结构和原生类型支持，极大地提升了 Go 语言开发中的配置体验。

---

## 一、配置文件

### 1.1 完整配置示例

```yaml
# ==============================================
# 应用基础配置
# ==============================================
app:
  name: vigo # 应用名称
  version: 1.0.1 # 版本号
  port: 8080 # HTTP 服务端口
  debug: false # 调试模式
  mode: prod # 运行模式: dev | prod
  console: true # 控制台输出
  auto_kill_port: true # 自动释放端口

# ==============================================
# 数据库配置（主库 - 默认数据库）
# ==============================================
database:
  driver: mysql # 驱动: mysql | postgres | sqlite | mssql
  host: ${DB_HOST} # 主机地址
  port: ${DB_PORT:3306} # 端口（默认 3306）
  name: ${DB_NAME} # 数据库名
  user: ${DB_USER} # 用户名
  pass: ${DB_PASSWORD} # 密码
  charset: utf8mb4 # 字符集
  max_open_conns: 200 # 最大连接数
  max_idle_conns: 100 # 最大空闲连接
  conn_max_lifetime: 3600 # 连接生命周期（秒）
  conn_max_idletime: 300 # 空闲连接回收时间（秒）

  # 读写分离
  rw_split: false
  writes: [] # 多主库配置
  reads: [] # 从库配置

# ==============================================
# 多数据库配置（不同业务独立数据库）
# ==============================================
databases: {}
  # user_center:
  #   driver: mysql
  #   host: 192.168.2.10
  #   port: 3306
  #   name: user_center
  #   user: user_admin
  #   pass: user-pass
  #   rw_split: true
  #   reads:
  #     - host: 192.168.2.20
  #       port: 3306

# ==============================================
# Nacos 配置中心
# ==============================================
nacos:
  host: ${NACOS_HOST:127.0.0.1}
  port: ${NACOS_PORT:8848}
  namespace: ${NACOS_NAMESPACE:}
  data_id: vigo-config
  group: DEFAULT_GROUP

  # 服务发现
  discovery:
    enabled: false
    service_name: vigo
    auto_register: true
    heartbeat: 5

  # 管理界面
  admin:
    enabled: false
    username: "" # 管理界面用户名（留空则不启用认证）
    password: "" # 管理界面密码（留空则不启用认证）

# ==============================================
# RabbitMQ 消息队列
# ==============================================
rabbitmq:
  enabled: false
  host: ${RABBITMQ_HOST:127.0.0.1}
  port: 5672
  user: ${RABBITMQ_USER:guest}
  pass: ${RABBITMQ_PASSWORD:guest}
  vhost: /

  # 管理界面
  admin:
    enabled: false
    username: ""
    password: ""

# ==============================================
# Redis 缓存
# ==============================================
redis:
  host: ${REDIS_HOST:127.0.0.1}
  port: ${REDIS_PORT:6379}
  password: ${REDIS_PASSWORD:}
  db: 0
  pool_size: 100 # 连接池大小
  min_idle_conns: 10 # 最小空闲连接

# ==============================================
# gRPC 微服务
# ==============================================
grpc:
  enabled: false
  port: 9090
  service_name: vigo

# ==============================================
# 安全配置
# ==============================================
security:
  jwt:
    secret: ${JWT_SECRET} # JWT 密钥（至少 32 位）
    expire: 7200 # 过期时间（秒）
    issuer: vigo
  cors:
    allow_origins: ["*"]
    allow_methods: ["GET", "POST", "PUT", "DELETE", "OPTIONS"]
    allow_headers: ["*"]
  dos:
    enabled: true
    limit: 1000 # 单 IP 限流（次/分钟）

# ==============================================
# 压力测试配置
# ==============================================
benchmark:
  enabled: true
  max_concurrent: 10000 # 最大并发数
```

## 二、环境变量

### 2.1 必需环境变量

```bash
# 数据库配置
export DB_HOST=127.0.0.1
export DB_PORT=3306
export DB_USER=vigo
export DB_PASSWORD=your_password
export DB_NAME=vigo

# Redis 配置
export REDIS_HOST=127.0.0.1
export REDIS_PORT=6379
export REDIS_PASSWORD=

# JWT 密钥（至少 32 位）
export JWT_SECRET=your_jwt_secret_at_least_32_characters
```

### 2.2 可选环境变量

```bash
# 应用配置
export APP_ENV=prod
export APP_DEBUG=false

# Nacos 配置
export NACOS_HOST=127.0.0.1
export NACOS_PORT=8848
export NACOS_NAMESPACE=
export NACOS_ADMIN_USER=admin      # Nacos 管理界面用户名
export NACOS_ADMIN_PASSWORD=your_password  # Nacos 管理界面密码

# RabbitMQ 配置
export RABBITMQ_HOST=127.0.0.1
export RABBITMQ_USER=guest
export RABBITMQ_PASSWORD=guest
export RABBITMQ_ADMIN_USER=admin      # RabbitMQ 管理界面用户名
export RABBITMQ_ADMIN_PASSWORD=your_password  # RabbitMQ 管理界面密码
```

### 2.3 环境变量占位符

配置支持 `${ENV_VAR}` 和 `${ENV_VAR:default}` 格式：

```yaml
host: ${DB_HOST} # 必须设置
port: ${DB_PORT:3306} # 可选，默认 3306
```

## 三、多环境配置

### 3.1 配置加载顺序

框架按以下顺序加载配置文件（后加载的覆盖先加载的）：

1. `config.yaml` - 基础配置
2. `config.{env}.yaml` - 环境配置（如 config.prod.yaml）
3. `config.local.yaml` - 本地覆盖配置（不提交到 git）

环境由 `APP_ENV` 环境变量决定，默认为 `dev`。

### 3.2 开发环境配置

创建 `config.dev.yaml`：

```yaml
app:
  debug: true
  mode: dev

database:
  max_open_conns: 50
  max_idle_conns: 10

redis:
  pool_size: 20
```

### 3.3 生产环境配置

创建 `config.prod.yaml`：

```yaml
app:
  debug: false
  mode: prod

database:
  max_open_conns: 500
  max_idle_conns: 100
  rw_split: true
  reads:
    - host: 192.168.1.20
      port: 3306

redis:
  pool_size: 200
  min_idle_conns: 50
```

## 四、在代码中读取配置

### 4.1 使用全局变量

```go
import "vigo/config"

func main() {
    // 应用配置
    port := config.App.App.Port
    mode := config.App.App.Mode
    debug := config.App.App.Debug

    // 数据库配置
    dbHost := config.App.Database.Host
    dbPort := config.App.Database.Port
    dbName := config.App.Database.Name

    // Redis 配置
    redisHost := config.App.Redis.Host
    redisPort := config.App.Redis.Port

    // Nacos 配置
    nacosHost := config.App.Nacos.IpAddr
    discoveryEnabled := config.App.Nacos.Discovery.Enabled

    // 安全配置
    jwtSecret := config.App.Security.JWT.Secret
    jwtExpire := config.App.Security.JWT.Expire
}
```

### 4.2 门面模式访问

```go
import "vigo/framework/facade"

conf := facade.Config() // 获取整个 AppConfig 实例
```

## 五、配置验证

### 5.1 自动验证

框架启动时会自动验证必填配置：

| 配置项              | 说明       |
| ------------------- | ---------- |
| database.host       | 数据库地址 |
| database.user       | 数据库用户 |
| database.pass       | 数据库密码 |
| database.name       | 数据库名称 |
| redis.host          | Redis 地址 |
| security.jwt.secret | JWT 密钥   |

缺失时会输出错误日志并退出。

### 5.2 手动验证

```go
import "vigo/config"

func validateConfig() error {
    if config.App.Database.Host == "" {
        return fmt.Errorf("数据库地址不能为空")
    }
    if config.App.Security.JWT.Secret == "" {
        return fmt.Errorf("JWT 密钥不能为空")
    }
    if len(config.App.Security.JWT.Secret) < 32 {
        return fmt.Errorf("JWT 密钥长度至少 32 位")
    }
    return nil
}
```

## 六、端口配置

### 6.1 服务端口

| 服务     | 默认端口 | 说明         |
| -------- | -------- | ------------ |
| HTTP     | 8080     | Web 服务端口 |
| gRPC     | 9090     | 微服务端口   |
| MySQL    | 3306     | 数据库端口   |
| Redis    | 6379     | 缓存端口     |
| RabbitMQ | 5672     | 消息队列端口 |
| Nacos    | 8848     | 配置中心端口 |

### 6.2 修改端口

```yaml
app:
  port: 8080 # HTTP 端口

grpc:
  enabled: true
  port: 9090 # gRPC 端口
```

## 七、配置最佳实践

### 7.1 敏感信息管理

1. 使用环境变量存储敏感信息
2. 不要将密码提交到版本控制
3. 生产环境使用配置中心（Nacos）

### 7.2 配置文件管理

1. 提供 `config.*.yaml.example` 示例文件
2. 使用 `config.local.yaml` 存储本地配置
3. 不同环境使用不同的配置文件

### 7.3 连接池配置建议

| 场景               | max_open_conns | max_idle_conns |
| ------------------ | -------------- | -------------- |
| 开发环境           | 50             | 10             |
| 生产环境（单机）   | 500            | 100            |
| 生产环境（高并发） | 1000           | 200            |
| 生产环境（分布式） | 300            | 50             |
