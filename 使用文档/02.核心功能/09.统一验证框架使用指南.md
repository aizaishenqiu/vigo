## 一、概述

Vigo 框架提供统一的输入验证框架，用于验证用户输入数据的合法性、安全性和完整性。

### 核心特性

- ✅ 丰富的验证规则
- ✅ 灵活的自定义规则
- ✅ 链式调用
- ✅ 错误信息自定义
- ✅ 数据过滤和清理
- ✅ 嵌套验证支持
- ✅ 条件验证支持

## 二、快速开始

### 2.1 基本使用

```go
import "vigo/framework/validate"

// 创建验证器
validator := validate.NewEnhanced()

// 添加验证规则
validator.AddRule("username", validate.Required())
validator.AddRule("username", validate.MinLength(3))
validator.AddRule("email", validate.Required())
validator.AddRule("email", validate.Email())

// 设置验证数据
validator.SetData(map[string]interface{}{
    "username": "zhangsan",
    "email": "zhangsan@example.com",
})

// 执行验证
if !validator.Validate() {
    // 获取错误信息
    errors := validator.GetErrors()
    // 处理错误
}
```

### 2.2 简化使用

```go
// 链式调用
validator := validate.NewEnhanced().
    Rule("username", validate.Required(), validate.MinLength(3)).
    Rule("email", validate.Required(), validate.Email()).
    Rule("age", validate.Required(), validate.Min(18)).
    Data(map[string]interface{}{
        "username": "zhangsan",
        "email": "zhangsan@example.com",
        "age": 25,
    })

if !validator.Validate() {
    // 处理错误
}
```

## 三、验证规则

### 3.1 常用验证规则

#### 必填验证

```go
// 字段必须存在且不为空
validate.Required()

// 示例
validator.AddRule("username", validate.Required())
```

#### 类型验证

```go
// 字符串类型
validate.IsString()

// 整数类型
validate.IsInt()

// 浮点数类型
validate.IsFloat()

// 布尔类型
validate.IsBool()

// 数组类型
validate.IsArray()

// 对象类型
validate.IsObject()
```

#### 长度验证

```go
// 最小长度
validate.MinLength(3)

// 最大长度
validate.MaxLength(20)

// 长度范围
validate.LengthBetween(3, 20)

// 精确长度
validate.ExactLength(11)
```

#### 数值验证

```go
// 最小值
validate.Min(18)

// 最大值
validate.Max(100)

// 数值范围
validate.Between(18, 100)
```

#### 格式验证

```go
// 邮箱格式
validate.Email()

// 手机号格式（中国）
validate.Mobile()

// 电话号码格式
validate.Phone()

// 身份证格式（中国）
validate.IDCard()

// URL 格式
validate.URL()

// IP 地址格式
validate.IP()

// IPv4 格式
validate.IPv4()

// IPv6 格式
validate.IPv6()

// 日期格式
validate.Date("2006-01-02")

// 时间格式
validate.Time("15:04:05")

// 日期时间格式
validate.DateTime("2006-01-02 15:04:05")
```

#### 字符验证

```go
// 只允许字母
validate.Alpha()

// 只允许数字
validate.Numeric()

// 只允许字母和数字
validate.AlphaNum()

// 只允许中文
validate.Chinese()

// 只允许字母、数字和下划线
validate.AlphaNumDash()
```

### 3.2 高级验证规则

#### 正则表达式验证

```go
// 自定义正则表达式
validate.Regex(`^[A-Za-z0-9_]+$`)

// 示例：验证用户名只允许字母、数字和下划线
validator.AddRule("username", validate.Regex(`^[A-Za-z0-9_]+$`))
```

#### 自定义验证函数

```go
// 自定义验证规则
customRule := validate.Custom(func(value interface{}) bool {
    str, ok := value.(string)
    if !ok {
        return false
    }
    // 自定义验证逻辑
    return len(str) >= 3 && len(str) <= 20
})

validator.AddRule("username", customRule)
```

#### 条件验证

```go
// 当 condition 为 true 时才验证
validate.When(condition, validate.Required())

// 示例：只有当 type 为 "email" 时才验证 email 字段
validator.AddRule("email", validate.When(
    validator.GetData("type") == "email",
    validate.Required(),
))
```

#### 排除验证

```go
// 当 condition 为 false 时才验证
validate.Unless(condition, validate.Required())
```

### 3.3 业务规则验证

#### 唯一性验证

```go
// 验证数据库中是否已存在
validate.Unique("users", "username")

// 示例
validator.AddRule("username", validate.Unique("users", "username"))
```

#### 存在性验证

```go
// 验证数据库中是否存在
validate.Exists("users", "id")

// 示例：验证 parent_id 是否存在
validator.AddRule("parent_id", validate.Exists("departments", "id"))
```

#### 比较验证

```go
// 等于某个字段
validate.EqualTo("password_confirmation")

// 不等于某个字段
validate.NotEqualTo("old_password")

// 大于某个字段
validate.GreaterThan("min_price")

// 小于某个字段
validate.LessThan("max_price")
```

## 四、数据过滤和清理

### 4.1 内置过滤器

```go
// 去除前后空格
validate.Trim()

// 去除 HTML 标签
validate.StripTags()

// 转换为整数
validate.ToInt()

// 转换为浮点数
validate.ToFloat()

validate.ToString()

validate.ToBool()

// 转换为小写
validate.ToLower()

// 转换为大写
validate.ToUpper()

// 首字母大写
validate.UcFirst()

// 每个单词首字母大写
validate.UcWords()
```

### 4.2 使用过滤器

```go
// 在验证前过滤数据

    Filter("username", validate.Trim()).
    Filter("email", validate.Trim(), validate.ToLower()).
    Filter("age", validate.ToInt()).
    Rule("username", validate.Required(), validate.MinLength(3)).
    Rule("email", validate.Required(), validate.Email()).
    Rule("age", validate.Required(), validate.Min(18)).
    Data(data)
```

### 4.3 自定义过滤器

```go

    if !ok {
        return value
    }
    // 自定义过滤逻辑
    return strings.TrimSpace(str)
})

validator.Filter("username", customFilter)
```

## 五、错误处理

### 5.1 获取错误信息

```go
if !validator.Validate() {
    // 获取所有错误
    errors := validator.GetErrors()

    // 获取指定字段的错误
    fieldErrors := validator.GetFieldErrors("username")

    // 获取第一个错误
    firstError := validator.GetFirstError()

    // 获取指定字段的第一个错误
    firstFieldError := validator.GetFirstFieldError("username")
}
```

### 5.2 自定义错误信息

```go
// 为规则指定错误信息
validator.AddRule("username",
    validate.Required().WithError("用户名不能为空"))
validator.AddRule("username",
    validate.MinLength(3).WithError("用户名至少 3 个字符"))

// 批量设置错误信息
validator.SetErrorMessages(map[string]map[string]string{
    "username": {
        "required": "用户名不能为空",
        "min_length": "用户名至少 3 个字符",
    },
    "email": {
        "required": "邮箱不能为空",
        "email": "邮箱格式不正确",
    },
})
```

### 5.3 错误格式化

```go
// 获取格式化的错误信息
formattedErrors := validator.FormatErrors()

// 输出示例：
// {
//     "username": ["用户名不能为空", "用户名至少 3 个字符"],
//     "email": ["邮箱格式不正确"]
// }
```

## 六、高级用法

### 6.1 嵌套验证

```go
// 验证嵌套对象
validator := validate.NewEnhanced().
    Rule("name", validate.Required()).
    Rule("address", validate.IsObject()).
    Rule("address.province", validate.Required()).
    Rule("address.city", validate.Required()).
    Rule("address.district", validate.Required())
```

### 6.2 数组验证

```go
// 验证数组
validator := validate.NewEnhanced().
    Rule("tags", validate.IsArray()).
    Rule("tags.*", validate.Required(), validate.MinLength(1))

// 验证对象数组
validator.Rule("users", validate.IsArray()).
        Rule("users.*.username", validate.Required()).
        Rule("users.*.email", validate.Required(), validate.Email())
```

### 6.3 分组验证

```go
// 创建验证组
validator := validate.NewEnhanced()

// 基础验证组
validator.Group("base").
    Rule("username", validate.Required()).
    Rule("email", validate.Required(), validate.Email())

// 创建验证组
validator.Group("create").
    Rule("password", validate.Required(), validate.MinLength(6))

// 执行指定组的验证
if !validator.ValidateGroup("base") {
    // 处理基础验证错误
}

if !validator.ValidateGroup("create") {
    // 处理创建验证错误
}
```

### 6.4 场景验证

```go
// 定义验证场景
validator := validate.NewEnhanced()

// 创建场景
validator.Scene("create", []string{"username", "email", "password"})
validator.Scene("update", []string{"username", "email"})

// 执行场景验证
if !validator.ValidateScene("create") {
    // 处理创建场景验证错误
}

if !validator.ValidateScene("update") {
    // 处理更新场景验证错误
}
```

## 八、最佳实践

### 8.1 验证器复用

```go
// 创建全局验证器工厂
func NewUserValidator() *validate.EnhancedValidator {
    return validate.NewEnhanced().
        Rule("username", validate.Required(), validate.MinLength(3), validate.MaxLength(20)).
        Rule("email", validate.Required(), validate.Email()).
        Rule("password", validate.Required(), validate.MinLength(6)).
        Rule("mobile", validate.Optional(), validate.Mobile())
}

// 在多个地方复用
validator := NewUserValidator().Data(data)
```

### 8.2 验证与过滤分离

```go
// 先过滤，后验证
validator := validate.NewEnhanced().
    Filter("username", validate.Trim()).
    Filter("email", validate.Trim(), validate.ToLower()).
    Rule("username", validate.Required(), validate.MinLength(3)).
    Rule("email", validate.Required(), validate.Email()).
    Data(data)
```

### 8.3 性能优化

```go
// 避免重复创建验证器
var userValidator = validate.NewEnhanced().
    Rule("username", validate.Required(), validate.MinLength(3)).
    Rule("email", validate.Required(), validate.Email())

// 复用验证器
validator := userValidator.Clone().Data(data)
```

## 九、常见问题

### Q1: 如何处理可选字段？

**A**: 使用 `Optional()` 规则：

```go
validator.AddRule("nickname", validate.Optional(), validate.MinLength(2))
```

### Q2: 如何验证文件上传？

**A**: 使用文件验证规则：

```go
validator.AddRule("avatar",
    validate.Required(),
    validate.FileType("jpg", "png", "gif"),
    validate.FileSize(2 * 1024 * 1024)) // 2MB
```

### Q3: 如何验证多个字段？

**A**: 使用数组语法：

```go
validator.AddRules([]string{"username", "email"}, validate.Required())
```

### Q4: 如何自定义错误信息？

**A**: 使用 `WithError()` 方法：

```go
validator.AddRule("username",
    validate.Required().WithError("用户名不能为空"))
```

### Q5: 如何验证嵌套数据？

**A**: 使用点号语法：

```go
validator.AddRule("user.username", validate.Required())
validator.AddRule("user.email", validate.Required(), validate.Email())
```

| ------------------ | -------------- | ------------------------------ |
| `Required()`       | 必填           | `validate.Required()`          |
