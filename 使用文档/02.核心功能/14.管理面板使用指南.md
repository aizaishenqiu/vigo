## 🎯 功能概述

Vigo 管理面板是一个功能强大的后端管理系统，提供可视化的配置管理、系统监控和服务管理功能。

### 核心特性

- ✅ **可视化配置**: 无需手动编辑 YAML 文件
- ✅ **实时监控**: 查看系统运行状态
- ✅ **服务管理**: RabbitMQ、Nacos、Redis 等配置
- ✅ **安全防护**: 安全中间件配置
- ✅ **响应式设计**: 适配各种屏幕尺寸

## 🚀 快速开始

### 1. 访问管理面板

启动服务后，访问：

```
http://localhost:8080/admin
```

### 2. 默认登录信息

- **用户名**: `admin`
- **密码**: `admin123`

> ⚠️ **安全提示**: 首次使用后请立即修改默认密码！

### 3. 配置文件设置

在 `config.yaml` 中确认以下配置：

```yaml
admin:
  enabled: true          # 启用管理面板
  base_path: /admin      # 访问路径
  username: admin        # 用户名
  password: admin123     # 密码
  allow_ips:             # 允许的 IP（* 表示所有）
    - "*"
  auto_register: true    # 自动注册路由
```

## 🛠️ 常见问题

### 1. 404 错误

**问题**: 访问 `/admin/` 显示 404

**解决方案**:
1. 检查 `config.yaml` 中 `admin.enabled` 是否为 `true`
2. 确认服务已启动并监听正确端口
3. 检查启动日志是否有 "管理面板已启用" 信息

### 2. 静态资源加载失败

**问题**: CSS/JS 文件 404 或 MIME 类型错误

**解决方案**:
1. 清除浏览器缓存（Ctrl+F5 强制刷新）
2. 检查静态文件是否存在于 `framework/admin/static/` 目录
3. 确认路由已正确注册

### 3. 配置保存失败

**问题**: 保存配置时提示错误

**解决方案**:
1. 检查文件写入权限
2. 确认 `config.yaml` 文件未被其他进程占用
3. 查看日志中的详细错误信息

### 4. 端口配置不生效

**问题**: 修改端口后服务仍在原端口运行

**解决方案**:
1. 完全停止服务后重新启动
2. 检查是否有其他配置文件（如 `config.local.yaml`）覆盖了端口
3. 确认修改的是正确的配置文件

## 📝 使用示例

### 示例 1: 修改服务端口

1. 访问 `/admin/settings`
2. 点击 "应用配置" 标签
3. 修改 "端口" 字段（如改为 8888）
4. 点击 "保存配置"
5. 重启服务

### 示例 2: 配置数据库连接

1. 访问 `/admin/settings`
2. 点击 "数据库配置" 标签
3. 填写数据库信息：
   - 主机：`127.0.0.1`
   - 端口：`3306`
   - 数据库名：`your_database`
   - 用户名：`root`
   - 密码：`your_password`
4. 点击 "保存配置"
5. 系统会自动测试连接

### 示例 3: 启用安全中间件

1. 访问 `/admin/settings`
2. 点击 "安全配置" 标签
3. 勾选需要启用的安全选项：
   - ✅ CSRF 保护
   - ✅ XSS 防护
   - ✅ 速率限制
4. 点击 "保存配置"

1. **修改默认密码**: 首次使用后立即修改 admin 密码

3. **启用 HTTPS**: 生产环境使用 HTTPS 加密传输
4. **定期备份配置**: 重要配置修改前先备份
5. **审计日志**: 记录所有配置修改操作
