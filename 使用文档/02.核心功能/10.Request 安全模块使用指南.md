# Request 安全模块使用指南

## 概述

Request 模块是一个强大的安全请求处理模块，提供自动化的输入验证和攻击防护。所有通过该模块获取的数据都会经过严格的安全检测，包括：

- ✅ **SQL 注入检测**：防止数据库注入攻击
- ✅ **XSS 攻击防护**：过滤跨站脚本攻击
- ✅ **命令注入检测**：防止系统命令注入
- ✅ **路径穿越防护**：阻止非法文件访问
- ✅ **数据自动净化**：自动清理恶意输入

## 核心特性

### 1. 自动验证（推荐）

通过中间件自动验证所有请求，无需手动调用：

```go
// 在路由配置中添加安全中间件
router.Use(middleware.SecurityMiddleware())
```

### 2. 助手函数（简洁优雅）

框架提供了简洁的助手函数，让安全数据获取变得异常简单：

```go
// 获取字符串
name := request.Input(c, "name", "默认值")

// 获取整数
age := request.InputInt(c, "age", 0)

// 获取浮点数
price := request.InputFloat(c, "price", 0.0)

// 获取布尔值
active := request.InputBool(c, "active", false)

// 获取切片
ids := request.InputSlice(c, "ids", []string{})

// 检查字段是否存在
if request.Has(c, "email") {
    // 处理 email
}

// 获取所有数据
data := request.GetAll(c)
```

### 3. 链式调用（面向对象）

也可以使用 Request 对象进行链式调用：

```go
req := request.R(c)

name := req.Get("name", "")
age := req.GetInt("age", 0)
active := req.GetBool("active", false)
```

### 示例 2：文章搜索控制器

```go
package controller

import (
    "strings"
    "vigo/framework/mvc"
    "vigo/framework/request"
)

type ArticleController struct {
    mvc.Controller
}

func (a *ArticleController) Search() {
    c := a.Context

    // 获取搜索关键词（已自动过滤 XSS 和 SQL 注入）
    keyword := request.Input(c, "keyword", "")
    page := request.InputInt(c, "page", 1)
    pageSize := request.InputInt(c, "page_size", 10)

    // 获取排序字段
    sortBy := request.Input(c, "sort_by", "created_at")
    order := request.Input(c, "order", "DESC")

    // 获取标签（逗号分隔）
    tags := request.InputSlice(c, "tags", []string{})

    // 验证排序字段（白名单）
    allowedSortFields := []string{"created_at", "updated_at", "views", "likes"}
    if !contains(allowedSortFields, sortBy) {
        sortBy = "created_at"
    }

    // 验证排序方向
    if order != "ASC" && order != "DESC" {
        order = "DESC"
    }

    // 构建查询（数据已净化，安全）
    query := db.Model(&Article{})

    if keyword != "" {
        query = query.Where("title LIKE ? OR content LIKE ?",
            "%"+keyword+"%", "%"+keyword+"%")
    }

    if len(tags) > 0 {
        query = query.Where("tags IN ?", tags)
    }

    // 执行查询
    var articles []Article
    query.Order(sortBy + " " + order).
        Offset((page - 1) * pageSize).
        Limit(pageSize).
        Find(&articles)

    c.Json(http.StatusOK, map[string]interface{}{
        "code":    200,
        "message": "success",
        "success": true,
        "data":    articles,
    })
}

func contains(slice []string, item string) bool {
    for _, s := range slice {
        if strings.EqualFold(s, item) {
            return true
        }
    }
    return false
}
```

### 示例 3：文件上传控制器

```go
package controller

import (
    "net/http"
    "path/filepath"
    "strings"
    "vigo/framework/mvc"
    "vigo/framework/request"
)

type FileController struct {
    mvc.Controller
}

func (f *FileController) Upload() {
    c := f.Context

    // 获取上传文件
    file, err := c.Request.FormFile("file")
    if err != nil {
        c.Json(http.StatusBadRequest, map[string]interface{}{
            "code":    400,
            "message": "文件上传失败",
            "success": false,
        })
        return
    }

    // 验证文件类型
    allowedTypes := []string{".jpg", ".jpeg", ".png", ".gif", ".pdf"}
    ext := strings.ToLower(filepath.Ext(file.Filename))

    if !containsString(allowedTypes, ext) {
        c.Json(http.StatusBadRequest, map[string]interface{}{
            "code":    400,
            "message": "不支持的文件类型",
            "success": false,
        })
        return
    }

    // 获取文件描述（已自动净化）
    description := request.Input(c, "description", "")
    category := request.Input(c, "category", "default")

    // 保存文件...

    c.Json(http.StatusOK, map[string]interface{}{
        "code":    200,
        "message": "上传成功",
        "success": true,
        "data": map[string]interface{}{
            "filename":    file.Filename,
            "description": description,
        },
    })
}

func containsString(slice []string, item string) bool {
    for _, s := range slice {
        if s == item {
            return true
        }
    }
    return false
}
```

## 中间件配置

### 全局启用（推荐）

在应用启动时全局注册安全中间件：

```go
// 在 main.go 或路由配置中
func SetupRoutes(router *mvc.Router) {
    // 全局启用安全中间件
    router.Use(middleware.SecurityMiddleware())

    // 其他中间件
    router.Use(middleware.CORS())
    router.Use(middleware.Logger())

    // 路由定义
    router.Get("/", HomeController.Index)
    router.Post("/user/create", UserController.Create)
    // ...
}
```

### 局部启用

只为特定路由组启用：

```go
// 只为 API 路由启用安全中间件
api := router.Group("/api")
api.Use(middleware.SecurityMiddleware())
{
    api.Post("/user/create", UserController.Create)
    api.Post("/order/create", OrderController.Create)
    api.Get("/search", SearchController.Index)
}
```

## 安全检测规则

### 1. SQL 注入检测

检测以下攻击模式：

- `SELECT ... FROM` 完整 SQL 语句
- `UNION SELECT` 联合查询
- `DROP TABLE`, `TRUNCATE` 危险操作
- `; --` SQL 注释注入
- `xp_cmdshell` 命令执行
- `../` 路径穿越

### 2. XSS 攻击检测

检测以下攻击模式：

- `<script>`, `<iframe>` 危险标签
- `onclick=`, `onerror=` 事件处理器
- `javascript:`, `vbscript:` 危险协议
- `expression()` CSS 注入
- HTML 实体绕过

### 3. 命令注入检测

检测以下攻击模式：

- `|`, `;`, `&` 命令分隔符
- `$(...)`, `` `...` `` 命令替换
- `cat`, `ls`, `wget` 危险命令

### 4. 路径穿越检测

检测以下攻击模式：

- `../`, `..\` 相对路径遍历
- URL 编码的变体
- 混合编码绕过

## 最佳实践

### ✅ 推荐做法

1. **始终使用 request 模块获取数据**

   ```go
   // ✅ 正确
   name := request.Input(c, "name", "")

   // ❌ 错误（绕过安全验证）
   name := c.Request.FormValue("name")
   ```

2. **提供合理的默认值**

   ```go
   // ✅ 提供默认值
   page := request.InputInt(c, "page", 1)
   pageSize := request.InputInt(c, "page_size", 10)
   ```

3. **二次验证敏感数据**

   ```go
   email := request.Input(c, "email", "")
   if !validateEmail(email) {
       // 处理无效邮箱
   }
   ```

4. **使用白名单验证**
   ```go
   sortBy := request.Input(c, "sort_by", "created_at")
   allowedFields := []string{"created_at", "updated_at", "views"}
   if !contains(allowedFields, sortBy) {
       sortBy = "created_at"
   }
   ```

### ❌ 避免做法

1. **不要直接使用原始数据**

   ```go
   // ❌ 危险：绕过安全验证
   rawValue := c.Request.FormValue("input")
   ```

2. **不要信任客户端数据**

   ```go
   // ❌ 危险：直接使用未验证数据
   userID := request.InputInt(c, "user_id", 0)
   // 还需要验证权限
   ```

3. **不要忽略验证错误**

   ```go
   // ❌ 危险：忽略错误
   _ = request.Validate(c)

   // ✅ 正确：处理错误
   if err := request.Validate(c); err != nil {
       c.Error(http.StatusBadRequest, err.Error())
       return
   }
   ```

## 常见问题

### Q1: 如何关闭自动验证？

A: 对于内部路径，可以添加到白名单：

```go
// 在 request.go 的 isInternalPath 函数中添加路径
internalPaths := []string{
    "/health",
    "/metrics",
    "/your-internal-path", // 添加这里
}
```

### Q2: 如何处理文件上传？

A: 文件上传会自动验证文件名和表单数据：

```go
file, _ := c.Request.FormFormFile("file")
description := request.Input(c, "description", "")
// description 已自动净化
```

### Q3: 如何自定义验证规则？

A: 可以扩展 Request 结构体或添加自定义中间件：

```go
// 添加自定义验证
func CustomValidation(c *mvc.Context) {
    req := request.R(c)

    // 自定义验证逻辑
    if req.Get("type", "") == "special" {
        // 特殊验证
    }
}
```

### Q4: 验证失败会返回什么？

A: 验证失败会返回 HTTP 400 错误：

```json
{
  "code": 400,
  "message": "安全验证失败：检测到 SQL 注入攻击 (参数：id)",
  "success": false
}
```

## 总结

Request 模块提供了全面的安全防护，让开发者无需担心常见的 Web 攻击。只需使用简单的助手函数，就能确保所有输入数据都是安全的。

**核心原则**：

- ✅ 所有输入都是危险的
- ✅ 始终通过 request 模块获取数据
- ✅ 自动验证 + 手动验证相结合
- ✅ 最小权限原则

**记住**：安全不是一次性的工作，而是持续的过程。Request 模块为您提供第一道防线，但还需要结合其他安全措施（如权限验证、速率限制等）来构建完整的安全体系。
