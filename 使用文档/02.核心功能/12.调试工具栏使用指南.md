# 🛠️ 调试工具栏使用指南

## 简介

Vigo 框架内置了强大的调试工具栏（Debug Toolbar），为开发者提供实时的请求/响应信息、性能指标、内存使用等调试数据。

### 主要特性

- ✅ **实时性能监控**: 显示请求耗时、内存使用、Goroutine 数量
- ✅ **请求/响应详情**: 完整的请求头、响应头、状态码
- ✅ **美观的 UI**: 现代化渐变设计，支持折叠/展开
- ✅ **智能着色**: 根据状态码自动着色（成功绿色、警告黄色、错误红色）
- ✅ **开发环境专用**: 仅在开发环境启用，不影响生产环境
- ✅ **HTML 响应注入**: 自动注入到 HTML 响应中，无需手动集成

---

## 快速开始

### 1. 配置启用

在 `config.yaml` 中配置调试工具栏：

```yaml
app:
  # 开发环境配置
  mode: dev          # 运行模式：dev | test | prod
  debug: true        # 开启调试模式
  
  # 调试工具栏配置
  debug_toolbar: true  # 启用调试工具栏（仅开发环境有效）
```

### 2. 访问页面

启用后，访问任何 HTML 页面，调试工具栏会自动出现在页面底部。

**示例访问**:
- http://localhost:8080
- http://localhost:8080/dashboard
- http://localhost:8080/docs

## 工作原理

### 自动注入机制

```
请求 → 调试中间件 → 记录开始时间/内存 → 处理请求 → 记录结束时间
                                          ↓
                              收集调试数据 → 生成工具栏 HTML
                                          ↓
                              注入到</body>前 → 返回响应
```

### 响应包装器

调试中间件使用响应包装器拦截响应：

```go
type responseWriter struct {
    http.ResponseWriter
    statusCode int
    body       *bytes.Buffer
}
```

### 条件注入

仅在以下条件同时满足时注入：

1. ✅ 开发环境 (`mode=dev` 或 `debug=true`)
2. ✅ 启用调试工具栏 (`debug_toolbar=true`)
3. ✅ 响应类型为 HTML (`Content-Type: text/html`)

## 配置选项

### config.yaml

```yaml
app:
  # 运行模式
  # dev:   开发环境，启用所有调试功能
  # test:  测试环境
  # prod:  生产环境，禁用所有调试功能
  mode: dev
  
  # 调试模式
  # true:  启用调试模式
  # false: 关闭调试模式
  debug: true
  
  # 调试工具栏
  # true:  启用调试工具栏
  # false: 禁用调试工具栏
  debug_toolbar: true
```

### 环境变量

支持使用环境变量配置：

```yaml
app:
  mode: ${APP_MODE:dev}
  debug: ${APP_DEBUG:true}
  debug_toolbar: ${DEBUG_TOOLBAR:true}
```

## 使用示例

### 开发环境配置

**config.dev.yaml**:
```yaml
app:
  name: my-app
  version: 1.0.0
  port: 8080
  mode: dev
  debug: true
  debug_toolbar: true  # ✅ 启用调试工具栏
  console: true
```

**config.prod.yaml**:
```yaml

  name: my-app
  version: 1.0.0
  port: 8080
  mode: prod
  debug: false
  debug_toolbar: false  # ❌ 禁用调试工具栏
  console: true
```

## 代码示例

### 手动控制调试工具栏

```go
import "vigo/framework/debug"

// 获取调试工具栏实例
toolbar := debug.NewDebugToolbar()

// 启用调试
toolbar.Enable()

// 禁用调试
toolbar.Disable()
```

### 在中间件中使用

```go
func DebugMiddleware() mvc.HandlerFunc {
    toolbar := debug.NewDebugToolbar()
    return toolbar.Middleware()
}
```

### 性能分析

```go
import "vigo/framework/debug"

// 创建性能分析器
profiler := debug.NewProfiler()

// 记录性能事件
profiler.Record("database_query", 50*time.Millisecond)
profiler.Record("cache_hit", 5*time.Millisecond)

// 获取性能摘要
summary := profiler.Summary()
fmt.Printf("总事件数：%d\n", summary["total_events"])
fmt.Printf("平均耗时：%d ms\n", summary["avg_time"])
```

### 数据库查询日志

```go
import "vigo/framework/debug"

// 创建查询日志记录器
queryLogger := debug.NewQueryLogger()

// 记录查询
queryLogger.Record(
    "SELECT * FROM users WHERE id = ?",
    []interface{}{1},
    10*time.Millisecond,
    nil,
)

// 获取所有查询
queries := queryLogger.GetQueries()

// 获取总查询时间
totalTime := queryLogger.TotalTime()

// 清空记录
queryLogger.Clear()
```

## 调试技巧

### 1. 查看慢请求

在详情面板中查看耗时超过 1 秒的请求，优化相关代码。

### 2. 内存泄漏检测

持续观察内存使用量，如果发现持续增长，可能存在内存泄漏。

### 3. Goroutine 泄漏检测

观察 Goroutine 数量，如果持续增长，可能存在 Goroutine 泄漏。

### 4. 状态码监控

注意非 200 状态码的响应，及时修复错误。

### 5. 性能对比

在不同环境下运行相同请求，对比性能指标，找出性能瓶颈。

## 最佳实践

### ✅ 推荐做法

1. **开发环境启用**: 在开发环境始终启用调试工具栏
2. **生产环境禁用**: 在生产环境禁用，避免性能开销
3. **定期检查**: 定期查看性能指标，优化慢请求
4. **配合日志**: 结合日志系统，更全面地了解应用状态

### ❌ 不推荐做法

1. **生产环境启用**: 会影响性能并暴露敏感信息
2. **忽略警告**: 看到慢请求或高内存使用不处理
3. **过度依赖**: 调试工具栏只是辅助，不能替代日志和监控

---

## 性能影响

### 开发环境

- **额外耗时**: ~1-5ms
- **内存开销**: ~10-50KB
- **影响**: 可接受，建议启用

### 生产环境

- **额外耗时**: ~5-10ms
- **内存开销**: ~50-100KB
- **影响**: 较大，建议禁用

## 故障排查

### 调试工具栏不显示

**检查清单**:

1. ✅ 确认 `mode=dev` 或 `debug=true`
2. ✅ 确认 `debug_toolbar=true`
3. ✅ 确认响应类型为 HTML
4. ✅ 清除浏览器缓存
5. ✅ 检查浏览器控制台是否有错误

### 样式显示异常

**解决方案**:

1. 清除浏览器缓存
2. 检查是否有 CSS 冲突
3. 使用浏览器开发者工具检查元素

### 数据不更新

**解决方案**:

1. 刷新页面
2. 检查中间件是否正确注册
3. 查看应用日志是否有错误
