## 🔧 配置方式

### 开发环境配置

```yaml
app:
  mode: dev              # 开发环境
  debug: true            # 启用调试模式
  debug_toolbar: true    # 启用调试工具栏
```

### 生产环境配置

```yaml
app:
  mode: prod             # 生产环境
  debug: false           # 关闭调试模式
  debug_toolbar: false   # 禁用调试工具栏
```

### 环境变量配置

```yaml
app:
  mode: ${APP_MODE:dev}
  debug: ${APP_DEBUG:true}
  debug_toolbar: ${DEBUG_TOOLBAR:true}
```

## 🎯 核心功能

### 1. 实时性能监控

- **请求耗时**: 精确到微秒（μs）
- **内存使用**: 实时监控内存变化（KB）
- **Goroutine 数量**: 跟踪并发协程数

### 2. 请求/响应分析

- **请求信息**: 方法、URL、Headers、Content-Type
- **响应信息**: 状态码、响应大小、Headers
- **完整数据**: JSON 格式的调试数据

### 3. 智能状态码着色

| 状态码 | 颜色 | 说明 |
|--------|------|------|
| 200-299 | 🟢 绿色 | 成功 |
| 300-399 | 🟡 黄色 | 重定向 |
| 400-499 | 🟠 粉色 | 客户端错误 |
| 500-599 | 🔴 红色 | 服务器错误 |

### 4. 自动注入

- ✅ 自动检测 HTML 响应
- ✅ 在 `</body>` 前注入工具栏
- ✅ 不影响非 HTML 响应（API、文件下载等）

## 💡 调试技巧

### 1. 性能优化

- 查看耗时超过 1 秒的请求
- 优化数据库查询
- 减少不必要的计算

### 2. 内存泄漏检测

- 持续观察内存使用量
- 如果持续增长，可能存在内存泄漏
- 使用 Profiler 进行详细分析

### 3. Goroutine 泄漏检测

- 观察 Goroutine 数量变化
- 如果持续增长，可能存在泄漏
- 检查是否有未正确关闭的 channel

### 4. 错误排查

- 注意非 200 状态码
- 查看完整调试数据
- 结合日志系统定位问题

## 🔍 技术实现

### 中间件架构

```go
func (dt *DebugToolbar) Middleware() mvc.HandlerFunc {
    return func(c *mvc.Context) {
        // 1. 记录开始时间和内存
        startTime := time.Now()
        startMem := getMemoryUsage()
        
        // 2. 包装响应写入器
        rw := &responseWriter{
            ResponseWriter: c.Writer,
            statusCode:     200,
            body:           &bytes.Buffer{},
        }
        c.Writer = rw
        
        // 3. 处理请求
        c.Next()
        
        // 4. 计算耗时和内存使用
        duration := time.Since(startTime)
        endMem := getMemoryUsage()
        
        // 5. 收集调试数据
        debugData := map[string]interface{}{...}
        
        // 6. 如果是 HTML 响应，注入工具栏
        if strings.Contains(c.GetHeader("Content-Type"), "text/html") {
            dt.injectToolbar(c, rw.body.String(), debugData)
        }
    }
}
```

### 响应包装器

```go
type responseWriter struct {
    http.ResponseWriter
    statusCode int
    body       *bytes.Buffer  // 缓存响应体
}
```

### 条件注入

```go
isDev := config.App.App.Mode == "dev" || config.App.Debug
if isDev && config.App.App.DebugToolbar {
    debugToolbar := debug.NewDebugToolbar()
    middleware.Use(debugToolbar.Middleware())
}
```
