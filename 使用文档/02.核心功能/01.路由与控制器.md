# 🎮 路由与控制器

## 路由定义

路由文件位于 `route/route.go`，包含两个初始化函数：

```go
package route

import (
    "vigo/app/controller"
    "vigo/framework/middleware"
    "vigo/framework/mvc"
    "vigo/framework/websocket"
)

// Init 注册路由
func Init(r *mvc.Router) {
    // 全局安全中间件（可选）
    if config.App.Security.EnableSecurityMiddleware {
        r.Use(middleware.SecurityMiddleware())
    }

    // 实例化控制器
    index := &controller.IndexController{}

    // 注册 GET 请求
    r.GET("/", index.Website)        // 官网首页
    r.GET("/dashboard", index.Index) // 后台控制面板
    r.GET("/hello", index.Hello)

    // 注册 POST 请求（带限流保护）
    auth := &controller.AuthController{}
    loginGroup := r.Group("/api", middleware.IPBasedRateLimitMiddleware(10))
    loginGroup.POST("/login", auth.Login)

    // 注册标准 http.Handler (如 Swagger)
    // r.Handle("/path", handler)
}

// InitWithHub 带 WebSocket Hub 的路由初始化
func InitWithHub(r *mvc.Router, hub *websocket.Hub) {
    // 初始化 WebSocket Hub
    // 调用 Init 注册路由
    Init(r)
}
```

## 控制器编写

控制器位于 `app/controller/` 目录下，可以选择继承 `BaseController` 或直接定义结构体。

```go
package controller

import "vigo/framework/mvc"

// 方式 1：直接定义结构体（无需认证）
type PublicController struct{}

// 示例方法
func (c *PublicController) Info(ctx *mvc.Context) {
    // 直接使用 ctx，不需要 Init 方法

    // 获取参数
    uid := ctx.Input("uid")

    // 返回 JSON
    ctx.Success(map[string]string{"uid": uid})
}

// 方式 2：继承 BaseController（需要认证和权限检查）
type UserController struct {
    BaseController
}

func (c *UserController) Profile(ctx *mvc.Context) {
    // 必须调用 Init 来初始化 BaseController
    c.Init(ctx)

    // Before() 已自动执行认证和权限检查

    // 获取当前用户 ID（由 BaseController 存储）
    userID, _ := c.GetUserID()

    // 返回 JSON
    c.Success(map[string]interface{}{"userID": userID})
}
```

## 请求与响应 (Context)

`mvc.Context` 封装了常用的 HTTP 操作。

### 获取参数

- `ctx.Input("key")`: 获取 URL Query 或 Form Post 参数
- `ctx.BindJSON(&struct)`: 绑定 JSON 请求体到结构体

### 返回响应

- `ctx.Json(code, data)`: 返回原始 JSON
- `ctx.Success(data)`: 返回标准成功结构 `{"code":0, "msg":"success", "data": ...}`
- `ctx.Error(code, msg)`: 返回标准错误结构 `{"code":401, "msg":"...", "data": nil}`
- `ctx.HTML(status, template, data)`: 渲染 HTML 模板
- `ctx.Redirect(url)`: 重定向

### BaseController 的便捷方法

继承 `BaseController` 后可以使用以下方法：

- `c.Init(ctx)`: 初始化 BaseController（必须在控制器方法中首先调用）
- `c.GetUserID()`: 获取当前登录用户 ID
- `c.IsLoggedIn()`: 检查用户是否已登录
- `c.GetLoginTime()`: 获取用户登录时间

## 健康检查接口

框架提供了两个健康检查接口，用于微服务探活和监控：

| 接口           | 描述           | 特点                                     |
| -------------- | -------------- | ---------------------------------------- |
| `/health`      | 轻量级健康检查 | 无 IO 操作，高性能，适合高并发场景       |
| `/health/full` | 完整健康检查   | 包含数据库和 Redis 连接状态，带 3 秒缓存 |

### 示例响应

**/health 响应：**

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "status": "healthy",
    "service": "vigo"
  }
}
```

**/health/full 响应：**

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "status": "healthy",
    "service": "vigo",
    "database": "up",
    "redis": "up"
  }
}
```
