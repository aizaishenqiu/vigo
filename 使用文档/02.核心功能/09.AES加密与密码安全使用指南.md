## 一、AES-256-GCM 加密解密

### 1.1 简介

Vigo 框架提供 **AES-256-GCM** 认证加密，这是目前最安全的对称加密方案之一。

**核心特性**:
- ✅ **256 位密钥** - 军事级安全
- ✅ **GCM 模式** - 提供完整性保护
- ✅ **随机 nonce** - 每次加密都不同
- ✅ **认证标签** - 防止篡改和伪造
- ✅ **前后端统一** - Go 和 JavaScript 使用相同算法

### 1.2 后端使用（Go）

#### 基本用法

```go
package main

import (
    "fmt"
    "log"
    "vigo/framework/crypto"
)

func main() {
    // 1. 创建加密器
    // 密钥可以是任意长度，会自动转换为 32 字节
    cryptoInstance, err := crypto.NewAESGCM("my-secret-key-0123456789abcdef")
    if err != nil {
        log.Fatal(err)
    }
    
    // 2. 加密数据
    plaintext := "敏感数据：密码 123456"
    encrypted, err := cryptoInstance.Encrypt(plaintext)
    if err != nil {
        log.Fatal(err)
    }
    fmt.Println("加密后:", encrypted)
    
    // 3. 解密数据
    decrypted, err := cryptoInstance.Decrypt(encrypted)
    if err != nil {
        log.Fatal(err)
    }
    fmt.Println("解密后:", decrypted)
    // 输出：敏感数据：密码 123456
}
```

#### 全局加密器

```go

    log.Fatal(err)

#### 生成安全密钥

```

#### 加密字节数据

```go

    log.Fatal(err)

```

### 1.3 前端使用（JavaScript）

```javascript
// AES-256-GCM 加密类
class AESGCM {
    constructor(key) {
        this.key = key;
    }

    async encrypt(plaintext) {
        const encoder = new TextEncoder();
        const data = encoder.encode(plaintext);
        
        // 导入密钥
        const cryptoKey = await crypto.subtle.importKey(
            'raw',
            new TextEncoder().encode(this.key),
            'AES-GCM',
            false,
            ['encrypt']
        );

        // 生成随机 IV
        const iv = crypto.getRandomValues(new Uint8Array(12));
        
        // 加密
        const encrypted = await crypto.subtle.encrypt(
            { name: 'AES-GCM', iv: iv },
            cryptoKey,
            data
        );

        // 组合 IV + 密文
        const result = new Uint8Array(iv.length + encrypted.byteLength);
        result.set(iv, 0);
        result.set(new Uint8Array(encrypted), iv.length);

        // Base64 编码
        return btoa(String.fromCharCode(...result));
    }

    async decrypt(ciphertext) {
        // Base64 解码
        const data = Uint8Array.from(atob(ciphertext), c => c.charCodeAt(0));
        
        // 导入密钥
        const cryptoKey = await crypto.subtle.importKey(
            'raw',
            new TextEncoder().encode(this.key),
            'AES-GCM',
            false,
            ['decrypt']
        );

        // 分离 IV 和密文
        const iv = data.slice(0, 12);
        const ciphertextBytes = data.slice(12);

        // 解密
        const decrypted = await crypto.subtle.decrypt(
            { name: 'AES-GCM', iv: iv },
            cryptoKey,
            ciphertextBytes
        );

        return new TextDecoder().decode(decrypted);
    }
}

// 使用示例
const crypto = new AESGCM('my-secret-key-0123456789abcdef');

// 加密
const encrypted = await crypto.encrypt('敏感数据');
console.log('加密后:', encrypted);

// 解密
const decrypted = await crypto.decrypt(encrypted);
console.log('解密后:', decrypted);
```

## 二、密码安全处理

### 2.1 简介

Vigo 框架使用 **bcrypt** 算法处理密码，这是目前最安全的密码哈希方案之一。

**核心特性**:
- ✅ **自动加盐** - 每个密码独立盐值
- ✅ **抗暴力破解** - 故意设计得慢
- ✅ **可调节成本** - 随硬件升级
- ✅ **行业标准** - OWASP 推荐

### 2.2 基本用法

#### 密码加密

```go
package main

import (
    "fmt"
    "log"
    "vigo/framework/security"
)

func main() {
    // 1. 加密密码
    password := "MyPassword123"
    hashed, err := security.HashPassword(password)
    if err != nil {
        log.Fatal(err)
    }
    
    fmt.Println("加密后:", hashed)
    // 输出：$2a$12$KIXxQ...（bcrypt 哈希）
    
    // 2. 保存到数据库
    user := &User{
        Username: "zhangsan",
        Password: hashed, // 存储加密后的密码
    }
    db.Save(user)
}
```

#### 密码验证

```go
// 用户登录
func login(username, password string) error {
    // 1. 查找用户
    user, err := db.FindUserByUsername(username)
    if err != nil {
        return errors.New("用户名或密码错误")
    }
    
    // 2. 验证密码
    if !security.VerifyPassword(password, user.Password) {
        return errors.New("用户名或密码错误")
    }
    
    // 3. 登录成功
    return nil
}
```

### 2.4 完整注册流程

```go
type UserService struct{}

// 用户注册
func (s *UserService) Register(username, password string) error {
    // 1. 检查密码强度
    policy := security.DefaultPasswordPolicy
    if err := policy.CheckPasswordStrength(password); err != nil {
        return err
    }
    
    // 2. 检查用户名是否存在
    exists, _ := db.UserExists(username)
    if exists {
        return errors.New("用户名已存在")
    }
    
    // 3. 加密密码
    hashed, err := security.HashPassword(password)
    if err != nil {
        return err
    }
    
    // 4. 保存用户
    user := &User{
        Username: username,
        Password: hashed,
        Status:   1,
    }
    
    return db.Save(user)
}

// 用户登录
func (s *UserService) Login(username, password string) (*User, error) {
    // 1. 查找用户
    user, err := db.FindUserByUsername(username)
    if err != nil {
        return nil, errors.New("用户名或密码错误")
    }
    
    // 2. 检查用户状态
    if user.Status != 1 {
        return nil, errors.New("账号已被禁用")
    }
    
    // 3. 验证密码
    if !security.VerifyPassword(password, user.Password) {
        return nil, errors.New("用户名或密码错误")
    }
    
    // 4. 登录成功
    return user, nil
}
```

### 2.5 密码升级

```go
// 检查是否需要重新加密（成本参数升级）
func (s *UserService) UpdatePasswordIfNeeded(userID int, password, oldHash string) error {
    // 验证旧密码
    if !security.VerifyPassword(password, oldHash) {
        return errors.New("旧密码错误")
    }
    
    // 检查是否需要重新加密
    if security.NeedsRehash(oldHash, 12) {
        // 重新加密（使用新的成本参数）
        newHash, _ := security.HashPassword(password)
        
        // 更新数据库
        db.Exec("UPDATE users SET password = ? WHERE id = ?", newHash, userID)
    }
    
    return nil
}
```

## 三、常见问题

### Q1: AES 密钥应该多长？

**A**: 推荐 32 字节（256 位）。框架会自动将任意长度密钥转换为 32 字节。

```go
// 都可以
crypto.NewAESGCM("short")              // 自动转换
crypto.NewAESGCM("32 字符的密钥 012345678901") // 正好 32 字符
crypto.NewAESGCM("very-long-key-string...")   // 自动转换
```

### Q2: bcrypt 成本参数选多少？

**A**: 
- **开发环境**: 10（快速）
- **生产环境**: 12（平衡）
- **高安全场景**: 14（非常安全）

### Q3: 如何更换加密密钥？

**A**: 
1. 生成新密钥
2. 用旧密钥解密所有数据
3. 用新密钥重新加密
4. 更新配置

### Q4: 密码忘记了怎么办？

**A**: 
1. 生成重置令牌
2. 发送邮件/短信
3. 验证令牌
4. 重置密码

### Q5: 前后端如何统一加密？

**A**: 
1. 使用相同密钥
2. 使用相同算法（AES-256-GCM）
3. 确保 IV/nonce 传输正确
