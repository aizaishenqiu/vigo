## 1. Skill 系统概述

### 1.1 什么是 Skill

**Skill** 是 Vigo 框架中的**可插拔功能模块**，类似于：

- VS Code 的 **Extensions**
- IntelliJ 的 **Plugins**
- WordPress 的 **Plugins**
- Laravel 的 **Packages**

每个 Skill 都是一个独立的功能包，可以：

- ✅ 提供新的路由和控制器
- ✅ 注册中间件和服务
- ✅ 添加命令行工具
- ✅ 提供 Web 管理界面
- ✅ 定义数据模型和迁移
- ✅ 注册事件监听器

### 1.2 为什么需要 Skill

#### 当前问题

1. **功能耦合严重**

   ```
   框架核心代码包含：
   - 调试工具栏
   - 数据迁移
   - 文件上传
   - 权限管理
   - ...

   问题：
   - 核心代码臃肿
   - 不需要的功能无法移除
   - 新功能添加需要修改核心代码
   ```

2. **扩展性差**

   ```
   添加新功能流程：
   1. 修改框架核心代码
   2. 添加路由
   3. 注册中间件
   4. 添加配置项

   问题：
   - 需要修改多处代码
   - 容易引入 bug
   - 升级困难
   ```

3. **复用性低**
   ```
   优秀功能无法独立复用：
   - 数据迁移功能很好，但其他项目想用却用不了
   - 权限管理很完善，但需要整个框架
   - 每个项目都要重复实现相同功能
   ```

#### Skill 解决方案

```
Skill 系统优势：

1. 模块化
   - 每个功能独立成 Skill
   - 按需加载，不需要的不加载
   - 核心框架保持精简

2. 可插拔
   - 安装：go get + 配置
   - 卸载：删除配置
   - 不影响核心框架

3. 可复用
   - 一次开发，多处使用
   - 官方 Skill + 社区 Skill
   - 版本独立管理

4. 易扩展
   - 第三方开发者可以开发 Skill
   - 不修改框架核心代码
   - 通过 Hook 机制扩展功能
```

## 2. 核心设计理念

### 2.1 设计原则

#### 1. 约定优于配置 (Convention over Configuration)

```go
// Skill 标准结构
skill-name/
├── skill.yaml              # Skill 描述文件（必须）
├── main.go                 # Skill 入口（必须）
├── routes/                 # 路由定义（可选）
│   └── routes.go
├── handlers/               # 处理器（可选）
├── middleware/             # 中间件（可选）
├── models/                 # 模型（可选）
├── migrations/             # 数据迁移（可选）
├── config/                 # 配置文件（可选）
│   └── config.yaml
├── views/                  # 视图模板（可选）
├── static/                 # 静态资源（可选）
├── commands/               # 命令行工具（可选）
└── tests/                  # 测试文件（可选）
```

**好处**：

- ✅ 目录结构清晰
- ✅ 自动发现和加载
- ✅ 减少配置工作

#### 2. 依赖注入 (Dependency Injection)

```go
// Skill 可以声明依赖
type Skill struct {
    Name        string
    Version     string
    Dependencies []string  // 依赖的其他 Skill

    // 生命周期方法
    Register    func()     // 注册服务
    Boot        func()     // 启动服务
    Shutdown    func()     // 关闭服务
}
```

**好处**：

- ✅ 自动解决依赖关系
- ✅ 按需加载
- ✅ 避免循环依赖

#### 3. 钩子机制 (Hook System)

```go
// 框架提供 Hook 点
hooks := map[string][]func(){
    "app.start":      { /* Skill A, Skill B */ },
    "app.stop":       { /* Skill C */ },
    "request.before": { /* Skill D - 认证 */ },
    "request.after":  { /* Skill E - 日志 */ },
    "db.query":       { /* Skill F - SQL 审计 */ },
}
```

**好处**：

- ✅ 不修改核心代码
- ✅ 多个 Skill 可以扩展同一点
- ✅ 灵活性强

#### 4. 配置隔离 (Configuration Isolation)

```yaml
# 每个 Skill 有独立配置
skill-name:
  enabled: true
  config_key1: value1
  config_key2: value2

# 框架配置
app:
  name: myapp
  skills:
    - debug-toolbar
    - migration
    - permission
```

**好处**：

- ✅ 配置不冲突
- ✅ 易于管理
- ✅ 支持覆盖

### 3.3 生命周期

```
App Start
    │
    ├─ 1. 加载配置文件
    │     └─ 读取 skills 列表
    │
    ├─ 2. 创建 Skill Manager
    │
    ├─ 3. 扫描 Skill 目录
    │     └─ 发现所有可用 Skill
    │
    ├─ 4. 解析依赖关系
    │     └─ 构建依赖图，检测循环依赖
    │
    ├─ 5. 按依赖顺序加载 Skill
    │     ├─ Load(skill)
    │     └─ Register(container)
    │
    ├─ 6. 启动 Skill
    │     └─ Boot(app)
    │
    ├─ 7. 触发 app.start Hook
    │
    └─ App Running


App Stop
    │
    ├─ 1. 触发 app.stop Hook
    │
    ├─ 2. 倒序关闭 Skill
    │     └─ Shutdown()
    │
    └─ App Stopped
```

```yaml

    - migration
    - auth
    - file-upload

    enabled: true
    position: bottom # 工具栏位置
    show_queries: true

  migration:
```

### 6.2 自动加载流程

```go
// framework/skill/loader.go

type Loader struct {
    config    *Config
    manager   *Manager
    registry  *Registry
}

// Load 加载所有启用的 Skill
func (l *Loader) Load() error {
    // 1. 扫描 Skill 目录
    skills, err := l.registry.Scan("./skills")
    if err != nil {
        return err
    }

    // 2. 构建依赖图
    graph := l.buildDependencyGraph(skills)

    // 3. 检测循环依赖
    if hasCycle := l.detectCycle(graph); hasCycle {
        return errors.New("检测到循环依赖")
    }

    // 4. 拓扑排序（确定加载顺序）
    order := l.topologicalSort(graph)

    // 5. 按顺序加载 Skill
    for _, skillName := range order {
        if !l.config.IsEnabled(skillName) {
            continue
        }

        skill := l.registry.Get(skillName)
        if err := l.manager.Load(skill); err != nil {
            return fmt.Errorf("加载 Skill %s 失败：%w", skillName, err)
        }
    }

    // 6. 启动所有 Skill
    for _, skillName := range order {
        if !l.config.IsEnabled(skillName) {
            continue
        }

        skill := l.manager.Get(skillName)
        if err := skill.Boot(l.manager.App()); err != nil {
            return fmt.Errorf("启动 Skill %s 失败：%w", skillName, err)
        }
    }

    return nil
}
```

### 6.3 依赖解析

```go
// 构建依赖图
func (l *Loader) buildDependencyGraph(skills []string) map[string][]string {
    graph := make(map[string][]string)

    for _, skillName := range skills {
        skill := l.registry.Get(skillName)
        graph[skillName] = skill.Dependencies()
    }

    return graph
}

// 检测循环依赖（DFS）
func (l *Loader) detectCycle(graph map[string][]string) bool {
    visited := make(map[string]bool)
    recStack := make(map[string]bool)

    var dfs func(node string) bool
    dfs = func(node string) bool {
        visited[node] = true
        recStack[node] = true

        for _, dep := range graph[node] {
            if !visited[dep] {
                if dfs(dep) {
                    return true
                }
            } else if recStack[dep] {
                return true  // 检测到循环
            }
        }

        recStack[node] = false
        return false
    }

    for node := range graph {
        if !visited[node] {
            if dfs(node) {
                return true
            }
        }
    }

    return false
}

// 拓扑排序（Kahn 算法）
func (l *Loader) topologicalSort(graph map[string][]string) []string {
    inDegree := make(map[string]int)

    // 计算入度
    for node := range graph {
        inDegree[node] = 0
    }
    for _, deps := range graph {
        for _, dep := range deps {
            inDegree[dep]++
        }
    }

    // 入度为 0 的节点
    queue := []string{}
    for node, degree := range inDegree {
        if degree == 0 {
            queue = append(queue, node)
        }
    }

    // 拓扑排序
    result := []string{}
    for len(queue) > 0 {
        node := queue[0]
        queue = queue[1:]
        result = append(result, node)

        for _, dep := range graph[node] {
            inDegree[dep]--
            if inDegree[dep] == 0 {
                queue = append(queue, dep)
            }
        }
    }

    return result
}
```

```go
// handlers/login.go

package handlers

import (
    "net/http"
    "vigo/framework/mvc"
    "vigo/framework/validate"
)

type LoginRequest struct {
    Email    string `json:"email" validate:"required,email"`
    Password string `json:"password" validate:"required,min=6"`
}

func Login(c *mvc.Context) {
    var req LoginRequest
    if err := c.Bind(&req); err != nil {
        c.Json(http.StatusBadRequest, errorResponse(err))
        return
    }

    if err := validate.Struct(&req); err != nil {
        c.Json(http.StatusBadRequest, errorResponse(err))
        return
    }

    // 调用认证服务
    authService := c.Get("auth.service").(*AuthService)
    token, err := authService.Login(req.Email, req.Password)
    if err != nil {
        c.Json(http.StatusUnauthorized, errorResponse(err))
        return
    }

    c.Json(http.StatusOK, successResponse(map[string]interface{}{
        "token": token,
    }))
}

func errorResponse(err error) map[string]interface{} {
    return map[string]interface{}{
        "code":    500,
        "message": err.Error(),
    }
}

func successResponse(data interface{}) map[string]interface{} {
    return map[string]interface{}{
        "code":    200,
        "message": "success",
        "data":    data,
    }
}
```

```go
// middleware/auth.go

package middleware

import (
    "net/http"
    "strings"
    "vigo/framework/mvc"
    "vigo/framework/jwt"
)

type AuthMiddleware struct{}

func NewAuthMiddleware() *AuthMiddleware {
    return &AuthMiddleware{}
}

func (m *AuthMiddleware) Handle(c *mvc.Context) {
    // 获取 Token
    authHeader := c.GetHeader("Authorization")
    if authHeader == "" {
        c.Json(http.StatusUnauthorized, map[string]interface{}{
            "code":    401,
            "message": "未提供认证令牌",
        })
        c.Abort()
        return
    }

    // 解析 Token
    parts := strings.SplitN(authHeader, " ", 2)
    if len(parts) != 2 || parts[0] != "Bearer" {
        c.Json(http.StatusUnauthorized, map[string]interface{}{
            "code":    401,
            "message": "令牌格式错误",
        })
        c.Abort()
        return
    }

    token := parts[1]
    claims, err := jwt.ParseToken(token)
    if err != nil {
        c.Json(http.StatusUnauthorized, map[string]interface{}{
            "code":    401,
            "message": "令牌无效",
        })
        c.Abort()
        return
    }

    // 设置用户信息到 Context
    c.Set("user_id", claims.UserID)
    c.Set("user_email", claims.Email)

    c.Next()
}
```
