### 方法二：手动上传

#### 1. 在 Gitee/GitHub 创建新仓库

**Gitee**:
1. 访问 https://gitee.com
2. 点击右上角 "+" → "新建仓库"
3. 仓库名称：`vigo`
4. 初始化选项：**不要勾选** "使用 README 初始化仓库"
5. 点击"创建"

**GitHub**:
1. 访问 https://github.com
2. 点击右上角 "+" → "New repository"
3. Repository name: `vigo`
4. 初始化选项：**不要勾选** "Add a README file"
5. 点击"Create repository"

#### 2. 配置 SSH 密钥（首次使用）

**生成 SSH 密钥**:
```bash
# 打开 Git Bash 或 PowerShell
ssh-keygen -t ed25519 -C "your_email@example.com"
# 按提示操作（可直接回车使用默认路径）
```

**添加公钥到 Gitee**:
1. 打开 `C:\Users\你的用户名\.ssh\id_ed25519.pub`
2. 复制全部内容
3. 访问 https://gitee.com/profile/sshkeys
4. 点击"添加公钥"，粘贴内容

**添加公钥到 GitHub**:
1. 打开 `C:\Users\你的用户名\.ssh\id_ed25519.pub`
2. 复制全部内容
3. 访问 https://github.com/settings/keys
4. 点击"New SSH key"，粘贴内容

#### 3. 添加远程仓库并推送

```bash
# 添加 Gitee 远程仓库
git remote add gitee git@gitee.com:你的用户名/vigo.git

# 添加 GitHub 远程仓库
git remote add github git@github.com:你的用户名/vigo.git

# 推送到 Gitee
git push -u gitee master

# 推送到 GitHub
git push -u github master
```

## 🔧 常见问题

### 1. 推送失败：Permission denied

**原因**: SSH 密钥未配置或配置错误

**解决方案**:
```bash
# 测试 SSH 连接
ssh -T git@gitee.com
ssh -T git@github.com

# 如果提示成功，重新推送
git push -u gitee master
git push -u github master
```

### 2. 远程仓库已存在

**解决方案**:
```bash
# 查看现有远程仓库
git remote -v

# 删除错误的远程仓库
git remote remove gitee
git remote remove github

# 重新添加
git remote add gitee git@gitee.com:你的用户名/vigo.git
git remote add github git@github.com:你的用户名/vigo.git
```

### 3. 推送被拒绝（非快进）

**原因**: 远程仓库有本地没有的提交

**解决方案**:
```bash
# 先拉取远程代码
git pull gitee master --allow-unrelated-histories
git pull github master --allow-unrelated-histories

# 再推送
git push -u gitee master
git push -u github master
```

### 4. 文件太大无法推送

**原因**: 提交了大文件（如二进制文件）

**解决方案**:
```bash
# 查看大文件
git rev-list --objects --all | grep "$(git verify-pack -v .git/objects/pack/*.idx | sort -k 3 -n | tail -5 | cut -f 1 -d' ')"

# 使用 git filter-branch 或 BFG Repo-Cleaner 清理
# 或检查 .gitignore 是否正确配置
```
