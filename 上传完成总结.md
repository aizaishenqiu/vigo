### 第二步：配置 SSH 密钥（首次使用）

#### 生成 SSH 密钥
```bash
# 打开 Git Bash 或 PowerShell
ssh-keygen -t ed25519 -C "your_email@example.com"
# 按回车使用默认路径
# 输入密码短语（可选，直接回车跳过）
```

#### 查看公钥内容
```bash
# Windows PowerShell
cat ~/.ssh/id_ed25519.pub

# 或打开文件
# C:\Users\你的用户名\.ssh\id_ed25519.pub
```

复制全部内容（从 `ssh-ed25519` 开头到邮箱地址）

#### 添加到 Gitee
1. 访问：https://gitee.com/profile/sshkeys
2. 点击 **"添加公钥"**
3. 标题：`My Computer`
4. 粘贴公钥内容
5. 点击 **"确定"**

#### 添加到 GitHub
1. 访问：https://github.com/settings/keys
2. 点击 **"New SSH key"**
3. Title: `My Computer`
4. 粘贴公钥内容
5. 点击 **"Add SSH key"**

### 第三步：上传代码

#### 方法一：使用自动脚本（推荐）

**PowerShell 方式**:
```powershell
# 在项目目录右键 → "在终端中打开"
# 或打开 PowerShell，cd 到项目目录
cd M:\www\vigo

# 运行上传脚本
.\upload.ps1
```

**批处理方式**:
```batch
# 双击 upload.bat
# 或在命令行运行
upload.bat
```

脚本会提示输入：
1. Gitee 仓库地址：`git@gitee.com:你的用户名/vigo.git`
2. GitHub 仓库地址：`git@github.com:你的用户名/vigo.git`

然后自动推送到两个平台。

#### 方法二：手动上传

```bash
# 1. 添加 Gitee 远程仓库
git remote add gitee git@gitee.com:你的用户名/vigo.git

# 2. 添加 GitHub 远程仓库
git remote add github git@github.com:你的用户名/vigo.git

# 3. 推送到 Gitee
git push -u gitee master

# 4. 推送到 GitHub
git push -u github master
```

## ⚠️ 注意事项

### 1. 大文件警告

如果推送时提示文件太大：
```bash
remote: error: File some-file is 200.00 MB; this exceeds GitHub's maximum file size
```

**解决方案**:
1. 检查 `.gitignore` 是否配置正确
2. 使用 `git lfs` 管理大文件
3. 或从 Git 历史中移除大文件

### 2. 认证失败

如果提示认证失败：
```
Permission denied (publickey).
fatal: Could not read from remote repository
```

**解决方案**:
1. 检查 SSH 密钥是否正确配置
2. 测试 SSH 连接：
   ```bash
   ssh -T git@gitee.com
   ssh -T git@github.com
   ```

### 3. 推送被拒绝

如果提示：
```
! [rejected] master -> master (fetch first)
```

**解决方案**:
```bash
# 先拉取远程代码
git pull gitee master --allow-unrelated-histories
git pull github master --allow-unrelated-histories

# 再推送
git push -u gitee master
git push -u github master
```
